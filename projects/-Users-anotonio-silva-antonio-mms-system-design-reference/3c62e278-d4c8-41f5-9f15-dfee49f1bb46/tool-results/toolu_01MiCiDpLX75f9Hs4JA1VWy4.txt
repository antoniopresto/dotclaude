## Performance Comparison: deck.gl vs Three.js vs Pixi.js

### General Rendering Performance

In rendering benchmark tests, the most performant was discovered to be Pixi.js, followed by Three.js and Two.js.(no content)Pixi.js achieved 47 FPS with good performance overall as a rendering engine.

### deck.gl Pan/Zoom Performance

On 2015 MacBook Pros with dual graphics cards, most basic layers (like ScatterplotLayer) render fluidly at 60 FPS during pan and zoom operations up to about 1M (one million) data items, with framerates dropping into low double digits (10-20 FPS) when the data sets approach 10M items.(no content)Even if interactivity is not an issue, browser limitations on how big chunks of contiguous memory can be allocated (e.g., Chrome caps individual allocations at 1GB) will cause most layers to crash during GPU buffer generation somewhere between 10M and 100M items. You would need to break up your data into chunks and use multiple deck.gl layers to get past this limit.(no content)Excessive overdraw (drawing many objects/pixels on top of each other) can hurt performance. For example, a Scatterplot radius of 5 pixels generates ~100 pixels per point. A Scatterplot layer with 10 million points can result in up to 1 billion fragment shader invocations per frame.

### Library Characteristics

**Pixi.js:**
PixiJS is an ultra-speedy, 2D rendering system designed primarily to deliver rich visual experiences on the web. Whether it's WebGL or Canvas, PixiJS stands as the triumphant successor to Flash in HTML5, boasting superior performance and pixel-level effects.(no content)Pixi.js is optimized for high-performance 2D rendering using WebGL, enabling smooth animations and complex visual effects. It supports sprites, textures, and filters, making it perfect for game development and interactive applications.

**Three.js:**
Three.js specializes in creating 3D animations and supports VR/AR through WebXR, focusing on GPU-accelerated 3D animations.(no content)Three.js handles 3D graphics rendering, while Pixi.js specializes in 2D. Pixi.js is celebrated for its speed and efficient rendering, beneficial for graphics-heavy applications.

**deck.gl:**
deck.gl can handle efficient WebGL2/WebGPU rendering under heavy data load. While deck.gl works standalone without a base map, it plays nicely with your favorite base map libraries such as Google Maps, Mapbox, ArcGIS, MapLibre, and more.

### Key Takeaways

- **For 2D interactive canvas with pan/zoom**: Pixi.js tends to be the fastest for pure 2D rendering
- **For large-scale data visualization with maps**: deck.gl is optimized for this use case, handling up to 1M items at 60 FPS during pan/zoom
- **For 3D scenes**: Three.js is the better choice
- Most people use Three.js or other libraries instead of directly using WebGL because it saves time not needing to 're-invent the wheel', and there are optimizations in libraries that have been carefully developed.
