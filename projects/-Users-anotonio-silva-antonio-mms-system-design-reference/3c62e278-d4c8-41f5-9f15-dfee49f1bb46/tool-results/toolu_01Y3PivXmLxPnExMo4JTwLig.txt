## Key Findings

### Basic Setup
Out of the box, deck.gl offers viewport controllers that map keyboard, mouse or touch input to camera state change. The easiest way to enable pan/zoom/rotate of the visualization is to set the controller prop on Deck or `<DeckGL>` to true along with an `initialViewState` object that defines the initial camera settings.

### OrthographicController Configuration
OrthographicController is the default controller for OrthographicView. You can configure it with various options:

There's a `zoomAxis` option to restrict zooming to the X or Y axes. It affects scroll, keyboard +/- and double tap. One of `X` (zoom along the X axis only), `Y` (zoom along the Y axis only), or `all`. Default is `all`. If this option is set to X or Y, `viewState.zoom` must be an array to enable independent zoom for each axis.

### Zoom Behavior
`zoom` (number | number[2], optional) - The zoom level of the viewport. `zoom: 0` maps one unit distance to one pixel on screen, and increasing zoom by 1 scales the same object to twice as large. For example `zoom: 1` is 2x the original size, `zoom: 2` is 4x, `zoom: 3` is 8x etc.(no content)The zoom prop of the OrthographicView now also supports an `[x, y]` Array to specify independent zoom levels. A `zoomAxis` option is added to the OrthographicController to restrict zooming to the X or Y axes.

### View State Management
If the view is interactive, every time the user pans/rotates/zooms, the view state will be updated to reflect the change. A viewState object describes the "real-time properties" of a camera. It may be updated continuously during interaction and/or transition.

### Custom Controller for Pan Events
You can extend OrthographicController to customize behavior by overriding `handleEvent()`. For example, intercepting 'pan' events while delegating others to the parent class.

### Dynamic Zoom Axis Switching
From a GitHub discussion, you can dynamically switch `zoomAxis` based on zoom level by updating controller props in `onViewStateChange`:

```javascript
let zoomMode = 'X'
const deckgl = new Deck({
  views: new OrthographicView(),
  controller: {zoomAxis: zoomMode},
  initialViewState: {target: [0, 0, 0], zoom: [0, 0]},
  onViewStateChange: ({viewState}) => {
    const nextZoomMode = viewState.zoom[0] > 10 ? 'all' : 'X'
    if (zoomMode !== nextZoomMode) {
      zoomMode = nextZoomMode
      deckgl.setProps({ controller: {zoomAxis: zoomMode} })
    }
  }
});
```

### Additional Options
OrthographicView adds a `flipY` option for controlling the Y-axis direction.

OrthographicView and OrbitView now also support 64-bit projection, with no extra code changes required. This greatly improves the visual quality when rendering very large and/or dense graphs.
