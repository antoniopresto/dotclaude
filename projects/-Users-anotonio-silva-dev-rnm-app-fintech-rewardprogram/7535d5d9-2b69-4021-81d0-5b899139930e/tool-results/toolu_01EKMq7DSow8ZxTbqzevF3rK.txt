   143→## 1. participantId Handling Pattern
   144→
   145→LEARNING: The `getParticipantIdFromToken()` function returns Promise<string | undefined>,
   146→not a synchronous value. This caused TypeScript errors initially.
   147→
   148→SOLUTION: Move participantId retrieval to the SERVICE layer, not the API layer.
   149→The API layer should accept participantId as a parameter.
   150→
   151→CORRECT PATTERN:
   152→```typescript
   153→// In service (products.service.ts):
   154→const participantId = await getParticipantIdFromToken();
   155→const response = await pointsProductsApi.getProducts(participantId);
   156→
   157→// In API (products.api.ts):
   158→async getProducts(participantId?: string): Promise<PointsProductsApiResponse> {
   159→  const response = await api.get<PointsProductsApiResponse>(
   160→    `/fintech/v1/rewardProgram/productslist`,
   161→    { headers: { ...(participantId && { participantId }) } }
   162→  );
   163→  return response.data;
   164→}
   165→```
   166→
   167→--------------------------------------------------------------------------------
   168→
   169→## 2. Status Enum Pattern vs Boolean Flags
   170→
   171→LEARNING: Redux uses isLoading/isError booleans. Zustand in this project uses Status enums.
   172→
   173→Pattern conversion:
   174→  - isLoading: true  -> status: Status.LOADING
   175→  - isError: true    -> status: Status.ERROR
   176→  - success state    -> status: Status.SUCCESS
   177→  - initial state    -> status: Status.IDLE
   178→
   179→Benefits:
   180→  - Single source of truth for state
   181→  - Cleaner conditional logic
   182→  - Easier to add new states (e.g., REFRESHING)
   183→
   184→--------------------------------------------------------------------------------

   186→## 3. Callback Handling in Rescue Domain
   187→
   188→LEARNING: Redux rescue action accepts callbacks (onSuccess, onFailed) in the action payload.
   189→These callbacks were handled by Redux-Saga middleware.
   190→
   191→SOLUTION: In Zustand, handle callbacks in the SERVICE layer, not the store.
   192→
   193→```typescript
   194→// rescue.service.ts
   195→export async function performRescue(
   196→  payload: RescuePayload,
   197→  callbacks?: RescueCallbacks
   198→): Promise<void> {
   199→  try {
   200→    rescueActions.setStatus(RescueStatus.LOADING);
   201→    const response = await rescueApi.rescue(payload);
   202→
   203→    if (response?.data?.status === 'OK') {
   204→      rescueActions.setStatus(RescueStatus.SUCCESS);
   205→      callbacks?.onSuccess?.(response.status);  // Callback here
   206→    } else {
   207→      rescueActions.setStatus(RescueStatus.ERROR);
   208→      callbacks?.onFailed?.(response?.status);   // Callback here
   209→    }
   210→  } catch (error) {
   211→    rescueActions.setStatus(RescueStatus.ERROR);
   212→    callbacks?.onFailed?.();
   213→  }
   214→}
   215→```
