# Long-Running Agent Harness Protocol

## Core Structure

Anthropic's approach uses a **two-agent system**:

1. **Initializer Agent** - Runs once on first session to establish the environment
2. **Coding Agent** - Executes in subsequent sessions, making incremental progress

## Key Components

### Environment Setup Files

The initializer creates:
- **init.sh script** - Automates development server startup and basic testing
- **claude-progress.txt** - Maintains session logs documenting prior work
- **Git repository** - Tracks changes with meaningful commit messages
- **feature_list.json** - Comprehensive feature registry (200+ items in their example)

### Feature List Format

```json
{
    "category": "functional",
    "description": "New chat button creates a fresh conversation",
    "steps": [
        "Navigate to main interface",
        "Click the 'New Chat' button",
        "Verify a new conversation is created",
        "Check that chat area shows welcome state",
        "Verify conversation appears in sidebar"
    ],
    "passes": false
}
```

Agents only modify the `passes` field, preventing accidental project structure corruption.

## Session Management Best Practices

### Startup Sequence
Each session begins with:
1. Run `pwd` to verify working directory
2. Read progress logs and git history
3. Review feature list and select next priority item
4. Execute init.sh to verify system functionality
5. Run baseline tests before implementing new features

### State Preservation
- **Git commits** capture working code states, enabling rollback
- **Progress files** provide human-readable context for the next session
- **Compaction** manages context windows, though alone it's insufficient

## Critical Failure Prevention

| Problem | Solution |
|---------|----------|
| Premature completion declarations | Structured feature list forces sequential progress |
| Undocumented work left in broken state | Git commits + progress summaries required at session end |
| Incomplete feature implementation | Mandatory end-to-end testing before marking complete |
| Time spent understanding state | Progress files + git logs provide instant orientation |

## Testing Integration

The system leverages **browser automation tools** (Puppeteer MCP) to verify features work as users experience them, not just at the code level. Testing happens before implementing new features, catching environmental issues immediately.

## Generalization Potential

While optimized for web applications, the researchers suggest these patterns could extend to scientific research and financial modeling tasks requiring multi-session workflows.
