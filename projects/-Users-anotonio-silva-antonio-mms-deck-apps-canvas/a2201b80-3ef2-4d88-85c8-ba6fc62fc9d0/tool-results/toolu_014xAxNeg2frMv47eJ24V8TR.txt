    21→## 1. Executive Summary
    22→
    23→This document establishes the visual identity, interaction patterns, and technical architecture for the MMS Fractal Canvas interface. The system enables maintenance planners to navigate seamlessly from 12-year strategic views to shift-level execution through semantic zoom—similar to Google Maps but for time-based maintenance planning.
    24→
    25→**Core Constraints:**
    26→- React 18+ with TypeScript
    27→- Storybook as development environment
    28→- External state management only (no useState/useReducer)
    29→- Permitted hooks: `useRef`, `useSyncExternalStore`, `useMemo`
    30→- Mature libraries only (8+ years of evolution)
    31→- Preference for deterministic, testable architectures (State Machines)

   259→
   260→| Theme | Description | Primary Reference |
   261→|-------|-------------|-------------------|
   262→| **Industrial Precision** | Clean, technical aesthetic for control rooms | voestalpine zentrak, Grafana |
   263→| **Information Density** | Maximum data visibility without clutter | SNCF Fleet Dashboard, Linear |

   265→| **Status at a Glance** | Color-coded health indicators | Prometheus GWOS, Railigent X |

   269→### 2.14 Anti-Patterns to Avoid
   270→
   271→| ❌ Avoid | ✅ Prefer |
   272→|----------|----------|
   273→| 3D effects, gradients on Gantt bars | Flat, monochrome rectangles |
   274→| Pure black borders | Slightly darker tone of bar background |
   275→| Multiple navigation tabs/pages | Single continuous canvas |
   276→| Modal dialogs for details | Inline expansion on zoom |
   277→| Skeuomorphic design elements | Clean, functional minimalism |
   278→| Bright saturated colors | Muted professional palette |

   282→## 3. UX Specifications
   283→
   284→### 3.1 Semantic Zoom Architecture
   285→
   286→The interface operates as a continuous zoomable canvas where zoom level determines content granularity:
   287→
   288→```
   289→ZOOM LEVEL          TIME SPAN       CONTENT DENSITY      RENDERING
   290→─────────────────────────────────────────────────────────────────────
   291→Strategic (< 0.3x)  │ 10-12 years  │ Aggregated blocks  │ WebGL
   292→                    │              │ Mileage thresholds │
   293→────────────────────┼──────────────┼────────────────────┼───────────
   294→Tactical (0.3-1.0x) │ 1-3 years    │ Campaign bars      │ WebGL/Canvas
   295→                    │              │ Capacity heatmap   │
   296→────────────────────┼──────────────┼────────────────────┼───────────
   297→Operational (1-3x)  │ 1-6 months   │ Work orders        │ Canvas/SVG
   298→                    │              │ Resource lanes     │
   299→────────────────────┼──────────────┼────────────────────┼───────────
   300→Execution (> 3x)    │ 1-4 weeks    │ Task details       │ SVG/HTML
   301→                    │              │ Bay assignments    │
   302→```
   303→
   304→### 3.2 Navigation Patterns
   305→
   306→#### Primary Navigation (Zoom + Pan)
   307→
   308→| Action | Input | Result |
   309→|--------|-------|--------|
   310→| Pan horizontally | Drag / Arrow keys / Shift+Scroll | Move through time |
   311→| Pan vertically | Drag / Arrow keys | Move through train units |
   312→| Zoom in | Scroll up / Pinch out / `+` key | Increase temporal detail |
   313→| Zoom out | Scroll down / Pinch in / `-` key | Decrease temporal detail |
   314→| Reset view | `Home` key / Reset button | Return to default strategic view |
   315→| Jump to today | `T` key / Today button | Center on current date |
   316→
   317→### 3.3 Content by Zoom Level
   318→
   319→#### Level 1: Strategic (10-12 Years)
   320→
   321→**Purpose:** Capital planning, fleet lifecycle overview, overhaul clustering
   322→
   323→**Visual Elements:**
   324→- Heatmap grid: Rows = Train units, Columns = Quarters/Years
   325→- Color intensity = Maintenance cost/hours
   326→- Horizontal threshold lines at 400K/800K/1.2M km
   327→- Major overhaul milestones as prominent blocks
   328→- Fleet segmentation by type (Kinki Sharyo / Alstom)
   329→
   330→#### Level 2: Tactical (1-3 Years)
   331→
   332→**Purpose:** Workload balancing, capacity planning, campaign scheduling
   333→
   334→**Visual Elements:**
   335→- Stacked bars showing PM types (PM1, PM2, PM3, Overhaul)
   336→- Depot capacity utilization as background heatmap
   337→- Resource allocation bands (man-hours available vs. required)
   338→- Budget period markers
   339→- Warning indicators for over-capacity periods
   340→
   341→#### Level 3: Operational (1-6 Months)
   342→
   343→**Purpose:** Work order management, resource assignment, conflict resolution
   344→
   345→**Visual Elements:**
   346→- Individual work order bars (DNA-like pattern)
   347→- Swimlanes per train unit
   348→- Resource assignment indicators
   349→- Dependency lines between related tasks
   350→- "Today" vertical line (prominent red dotted)
   351→- Mileage balance bars per train
   352→
   353→#### Level 4: Execution (1-4 Weeks)
   354→
   355→**Purpose:** Shift planning, bay assignment, real-time tracking
   356→
   357→**Visual Elements:**
   358→- Detailed task cards with task name, duration, assigned technicians, parts status
   359→- Depot bay grid overlay
   360→- Shift boundaries clearly marked
   361→- Real-time status badges
   362→
   363→### 3.4 Layout Structure
   364→
   365→```
   366→┌──────────────────────────────────────────────────────────────────────────┐
   367→│ NAVBAR                                               [Zoom: 1.00x] [LOD] │
   368→│ [☰] MMS Canvas    [<< Today >>]    [Search]    [Filters▼]    [⚙] [👤]  │
   369→├──────────┬───────────────────────────────────────────────────────────────┤
   370→│          │  TIMELINE HEADER (adaptive granularity)                       │
   371→│          │  │ 2024 │ 2025 │ 2026 │ 2027 │ ... │ or │ Jan │ Feb │ Mar │  │
   372→│  SIDEBAR ├───────────────────────────────────────────────────────────────┤
   373→│          │                                                               │
   374→│  Train   │                    MAIN CANVAS                                │
   375→│  IDs     │                                                               │
   376→│  (synced │        [Strategic | Tactical | Operational | Execution]       │
   377→│  scroll) │                                                               │
   378→│          │                    Rendered via WebGL/Canvas/SVG              │
   379→│  ───────│                                                               │
   380→│  Filters │                                                    ┌────────┐│
   381→│  Legend  │                                                    │Minimap ││
   382→│          │                                            [+][-]  └────────┘│
   383→└──────────┴───────────────────────────────────────────────────────────────┘
   384→```

   452→### 4.3 Component Specifications
   453→
   454→#### Gantt Bar Anatomy
   455→
   456→```
   457→┌─────────────────────────────────────────────────────┐
   458→│ ╭ PM2-00456                                    ⚠️ │  ← Rounded top-left corner
   459→│   4.5h  │  Track 3  │  2 technicians              │  ← Info in single line
   460→└─────────────────────────────────────────────────────┘
   461→
   462→Height: 24px (operational), 16px (tactical), 8px (strategic)
   463→Corner radius: 4px top-left only
   464→Border: None (use 10% darker background for definition)

   474→└─────────────────────────────┘
   475→```

   477→### 4.4 Animation & Motion
   478→
   479→#### Timing Functions
   480→
   481→```css
   482→--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);    /* Primary for zoom */
   483→--ease-out-cubic: cubic-bezier(0.33, 1, 0.68, 1);  /* Secondary transitions */
   484→```

   497→## 5. Frontend Architecture
   498→
   499→### 5.1 Core Principles
   500→
   501→1. **State Outside React**: All application state managed externally
   502→2. **Deterministic Updates**: State transitions via finite state machines
   503→3. **Immutable Data**: All state updates produce new references
   504→4. **Render Performance**: React only handles DOM reconciliation
   505→5. **Testability**: State logic unit-testable without React
   506→
   507→### 5.2 Allowed React Hooks
   508→
   509→| Hook | Purpose | Usage |
   510→|------|---------|-------|
   511→| `useRef` | DOM references, mutable values | Canvas refs, animation frames |
   512→| `useSyncExternalStore` | Subscribe to external stores | State machine subscriptions |
   513→| `useMemo` | Derived values from stable inputs | Computed selectors |
   514→| Library hooks | Required by external libraries | deck.gl, XState, etc. |
   515→
   516→### 5.3 Prohibited Patterns
   517→
   518→```typescript
   519→// ❌ PROHIBITED
   520→const [state, setState] = useState();
   521→const [state, dispatch] = useReducer();
   522→useEffect(() => { /* side effects */ }, []);
   523→useCallback(() => {}, []);
   524→
   525→// ✅ ALLOWED
   526→const canvasRef = useRef<HTMLCanvasElement>(null);
   527→const state = useSyncExternalStore(store.subscribe, store.getSnapshot);
   528→const derived = useMemo(() => computeExpensive(state), [state]);
   529→```

   757→### 7.4 Key Implementation Tips
   758→
   759→1. **During zoom gesture**: Use CSS transform for instant GPU-accelerated feedback
   760→2. **After zoom settles** (debounced 150ms): Redraw canvas with new LOD
   761→3. **LOD transitions**: Crossfade between layers over 200ms
   762→4. **Row virtualization**: Render only visible rows + 5-row buffer
   763→5. **Spatial indexing**: Use Quadtree/R-tree for O(log n) viewport queries
