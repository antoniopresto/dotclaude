     1→# Test Design and Risk Assessment - Validation Checklist
     2→
     3→## Prerequisites
     4→
     5→- [ ] Story markdown with clear acceptance criteria exists
     6→- [ ] PRD or epic documentation available
     7→- [ ] Architecture documents available (optional)
     8→- [ ] Requirements are testable and unambiguous
     9→
    10→## Process Steps
    11→
    12→### Step 1: Context Loading
    13→
    14→- [ ] PRD.md read and requirements extracted
    15→- [ ] Epics.md or specific epic documentation loaded
    16→- [ ] Story markdown with acceptance criteria analyzed
    17→- [ ] Architecture documents reviewed (if available)
    18→- [ ] Existing test coverage analyzed
    19→- [ ] Knowledge base fragments loaded (risk-governance, probability-impact, test-levels, test-priorities)
    20→
    21→### Step 2: Risk Assessment
    22→
    23→- [ ] Genuine risks identified (not just features)
    24→- [ ] Risks classified by category (TECH/SEC/PERF/DATA/BUS/OPS)
    25→- [ ] Probability scored (1-3 for each risk)
    26→- [ ] Impact scored (1-3 for each risk)
    27→- [ ] Risk scores calculated (probability × impact)
    28→- [ ] High-priority risks (score ≥6) flagged
    29→- [ ] Mitigation plans defined for high-priority risks
    30→- [ ] Owners assigned for each mitigation
    31→- [ ] Timelines set for mitigations
    32→- [ ] Residual risk documented
    33→
    34→### Step 3: Coverage Design
    35→
    36→- [ ] Acceptance criteria broken into atomic scenarios
    37→- [ ] Test levels selected (E2E/API/Component/Unit)
    38→- [ ] No duplicate coverage across levels
    39→- [ ] Priority levels assigned (P0/P1/P2/P3)
    40→- [ ] P0 scenarios meet strict criteria (blocks core + high risk + no workaround)
    41→- [ ] Data prerequisites identified
    42→- [ ] Tooling requirements documented
    43→- [ ] Execution order defined (smoke → P0 → P1 → P2/P3)
    44→
    45→### Step 4: Deliverables Generation
    46→
    47→- [ ] Risk assessment matrix created
    48→- [ ] Coverage matrix created
    49→- [ ] Execution order documented
    50→- [ ] Resource estimates calculated
    51→- [ ] Quality gate criteria defined
    52→- [ ] Output file written to correct location
    53→- [ ] Output file uses template structure
    54→
    55→## Output Validation
    56→
    57→### Risk Assessment Matrix
    58→
    59→- [ ] All risks have unique IDs (R-001, R-002, etc.)

    62→- [ ] Impact values are 1, 2, or 3
    63→- [ ] Scores calculated correctly (P × I)
    64→- [ ] High-priority risks (≥6) clearly marked
    65→- [ ] Mitigation strategies specific and actionable
    66→
    67→### Coverage Matrix
    68→
    69→- [ ] All requirements mapped to test levels
    70→- [ ] Priorities assigned to all scenarios
    71→- [ ] Risk linkage documented
    72→- [ ] Test counts realistic
    73→- [ ] Owners assigned where applicable
    74→- [ ] No duplicate coverage (same behavior at multiple levels)
    75→
    76→### Execution Order
    77→
    78→- [ ] Smoke tests defined (<5 min target)
    79→- [ ] P0 tests listed (<10 min target)
    80→- [ ] P1 tests listed (<30 min target)
    81→- [ ] P2/P3 tests listed (<60 min target)
    82→- [ ] Order optimizes for fast feedback

    84→### Resource Estimates
    85→
    86→- [ ] P0 hours calculated (count × 2 hours)

    90→- [ ] Total hours summed
    91→- [ ] Days estimate provided (hours / 8)
    92→- [ ] Estimates include setup time

    95→

    97→- [ ] P1 pass rate threshold defined (typically ≥95%)

   200→## Notes
   201→
   202→### Common Issues
   203→
   204→**Issue**: Too many P0 tests
   205→
   206→- **Solution**: Apply strict P0 criteria - must block core AND high risk AND no workaround
   207→
   208→**Issue**: Risk scores all high
   209→
   210→- **Solution**: Differentiate between high-impact (3) and degraded (2) impacts
   211→
   212→**Issue**: Duplicate coverage across levels
   213→
   214→- **Solution**: Use test pyramid - E2E for critical paths only
   215→
   216→**Issue**: Resource estimates too high
   217→
   218→- **Solution**: Invest in fixtures/factories to reduce per-test setup time
   219→
   220→### Best Practices
   221→
   222→- Base risk assessment on evidence, not assumptions
   223→- High-priority risks (≥6) require immediate mitigation
   224→- P0 tests should cover <10% of total scenarios
   225→- Avoid testing same behavior at multiple levels
   226→- Include smoke tests (P0 subset) for fast feedback
