    20вҶ’
    21вҶ’### 1.1 Architecture Vision
    22вҶ’
    23вҶ’MMS is a **demo-focused system** designed to win a competitive bid through a 15-minute presentation. The architecture prioritizes:
    24вҶ’
    25вҶ’1. **Demo Stability** - 15 minutes without crashes
    26вҶ’2. **Visual Impact** - Fast rendering, smooth interactions
    27вҶ’3. **Domain Credibility** - Correct railway terminology, realistic behavior
    28вҶ’4. **Technical Proof** - Working solver, real-time recommendations

   204вҶ’```sql
   205вҶ’-- Core entities
   206вҶ’CREATE TABLE fleets (
   207вҶ’    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

   226вҶ’

   228вҶ’    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

   230вҶ’    fleet_id UUID REFERENCES fleets(id),
   231вҶ’    base_depot_id UUID REFERENCES depots(id),
   232вҶ’    current_km INTEGER DEFAULT 0,
   233вҶ’    target_km INTEGER DEFAULT 0,
   234вҶ’    fitness_status VARCHAR(10) DEFAULT 'fit' CHECK (fitness_status IN ('fit', 'unfit')),
   235вҶ’    last_maintenance_date DATE,
   236вҶ’    next_maintenance_km INTEGER,
   237вҶ’    latitude DECIMAL(10, 6),
   238вҶ’    longitude DECIMAL(10, 6),
   239вҶ’    created_at TIMESTAMP DEFAULT NOW(),
   240вҶ’    updated_at TIMESTAMP DEFAULT NOW()

   242вҶ’
   243вҶ’CREATE TABLE services (
   244вҶ’    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   245вҶ’    service_code VARCHAR(20) UNIQUE NOT NULL,  -- IC 501
   246вҶ’    origin VARCHAR(100) NOT NULL,
   247вҶ’    destination VARCHAR(100) NOT NULL,
   248вҶ’    distance_km INTEGER NOT NULL,
   249вҶ’    departure_time TIME NOT NULL,
   250вҶ’    arrival_time TIME NOT NULL,
   251вҶ’    service_type VARCHAR(20) DEFAULT 'regional',  -- regional, intercity
   252вҶ’    created_at TIMESTAMP DEFAULT NOW()

   254вҶ’
   255вҶ’CREATE TABLE allocations (
   256вҶ’    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   257вҶ’    service_id UUID REFERENCES services(id),
   258вҶ’    train_id UUID REFERENCES trains(id),

   261вҶ’    score INTEGER,

   270вҶ’    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

   279вҶ’    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   280вҶ’    train_id UUID REFERENCES trains(id),
   281вҶ’    depot_id UUID REFERENCES depots(id),
   282вҶ’    maintenance_type VARCHAR(10) CHECK (maintenance_type IN ('PM1', 'PM2', 'HM', 'OVH')),
   283вҶ’    scheduled_date DATE NOT NULL,
   284вҶ’    duration_days INTEGER DEFAULT 1,
   285вҶ’    status VARCHAR(20) DEFAULT 'scheduled' CHECK (status IN ('scheduled', 'in_progress', 'completed', 'cancelled')),
   286вҶ’    created_at TIMESTAMP DEFAULT NOW()
   287вҶ’);
   288вҶ’
   289вҶ’-- Indexes for performance
   290вҶ’CREATE INDEX idx_trains_fleet ON trains(fleet_id);
   291вҶ’CREATE INDEX idx_trains_depot ON trains(base_depot_id);

   312вҶ’**Read Path (Dashboard):**
   313вҶ’1. Frontend requests `/api/dashboard/summary`

   316вҶ’4. Frontend renders immediately
   317вҶ’
   318вҶ’**Write Path (Allocation):**
   319вҶ’1. User selects service, clicks "Recommend"
   320вҶ’2. Frontend calls `POST /api/allocations/recommend`
   321вҶ’3. API calls Solver service with service details
   322вҶ’4. Solver returns ranked recommendations with justifications
   323вҶ’5. User confirms, API writes to allocations table
   324вҶ’6. Frontend updates optimistically
