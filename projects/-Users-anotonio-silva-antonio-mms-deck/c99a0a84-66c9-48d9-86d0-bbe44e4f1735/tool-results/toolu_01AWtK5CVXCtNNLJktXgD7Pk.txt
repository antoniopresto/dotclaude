# Harness para Agentes de Longa Duração

## O que é um Harness

Um harness é uma estrutura de controle que permite que agentes de IA trabalhem efetivamente em tarefas complexas que se estendem por múltiplas janelas de contexto. Como descrito no artigo, "agentes precisam de uma forma de lidar com o fato de que começam cada nova sessão sem memória do que veio antes."

## O Desafio Principal

O problema fundamental é que agentes operam em sessões discretas. Sem orientação apropriada, mesmo modelos avançados como o Claude Opus 4.5 tendem a:

1. **Tentar fazer tudo de uma vez** - resultando em contexto esgotado no meio da implementação
2. **Declarar vitória prematuramente** - abandonando tarefas incompletas

## Arquitetura de Dois Componentes

O harness proposto utiliza:

**Agente Inicializador**: Na primeira sessão, configura um ambiente estruturado que inclui:
- Script `init.sh` para inicializar o servidor
- Arquivo `claude-progress.txt` documentando progresso
- Commit git inicial

**Agente de Codificação**: Nas sessões subsequentes, trabalha incrementalmente com:
- Leitura de histórico git e progresso anterior
- Foco em uma feature por vez
- Commits descritivos após cada conclusão

## Melhores Práticas para Manter Contexto

**Lista de Features em JSON**: Um arquivo estruturado enumera todas as funcionalidades necessárias, inicialmente marcadas como "falhando". Isso oferece clareza sobre o escopo completo sem permitir que agentes declarem prematuro sucesso.

**Testes Sistemáticos**: O agente deve "verificar funcionalidades end-to-end após mudanças de código" usando ferramentas de automação como Puppeteer, evitando aprovações prematuras.

**Checklist Inicial**: Cada nova sessão segue um protocolo de orientação: verificar diretório de trabalho, ler logs git e arquivo de progresso, revisar lista de features.

Essa abordagem garante que conhecimento acumulado persista através de múltiplas sessões através de artefatos legíveis e estruturados.
