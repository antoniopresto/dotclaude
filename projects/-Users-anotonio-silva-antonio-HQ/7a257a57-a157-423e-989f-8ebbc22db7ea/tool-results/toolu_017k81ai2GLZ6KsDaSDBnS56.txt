# PatternFly Page Component Guide

## Overview

The Page component establishes the fundamental layout structure for applications, supporting both vertical and horizontal navigation patterns. It manages masthead, sidebar, and main content areas with flexible configuration options.

## Core Components

### Page
The primary container managing overall application layout. Key features include:
- Accepts a masthead for header content
- Supports optional sidebar for navigation
- Manages main content sections
- Handles responsive breakpoints automatically

**Essential Props:**
- `masthead`: Header component (typically containing Masthead)
- `sidebar`: Navigation sidebar (PageSidebar component)
- `children`: Main content sections
- `isManagedSidebar`: When true, automatically handles sidebar open/close state
- `isContentFilled`: Allows child sections to fill available vertical space

### PageSidebar
Container for vertical navigation and sidebar content.

**Key Props:**
- `isSidebarOpen`: Controls visibility (managed by Page if `isManagedSidebar={true}`)
- `children`: Accepts PageSidebarBody components

### PageSidebarBody
Individual content containers within the sidebar, offering layout flexibility.

**Configuration Options:**
- `isFilled`: Enables component to expand vertically (last body fills by default)
- `usePageInsets`: Matches page margin spacing
- `isContextSelector`: Identifies context switcher sections

### PageSection
Individual content area with customizable styling and layout behavior.

**Primary Props:**
- `type`: "default" | "subnav" | "breadcrumb" | "tabs" | "wizard"
- `variant`: "default" | "secondary" (affects background color)
- `isFilled`: Enables vertical space filling
- `isWidthLimited`: Restricts section width
- `isCenterAligned`: Centers width-limited sections
- `padding`: Object specifying padding at various breakpoints

### PageGroup
Groups related sections with shared styling and behavior options.

**Features:**
- `isFilled`: Enables space filling
- `hasOverflowScroll`: Allows internal scrolling
- `stickyOnBreakpoint`: Controls sticky positioning

## Layout Patterns

### Vertical Navigation
Structure combines a masthead with sidebar navigation:

"To add a vertical sidebar to a <Page>, pass a <PageSidebar> component into the sidebar property."

The sidebar expands/collapses via toggle in the masthead, with `isSidebarOpen` managing visibility state.

### Horizontal Navigation
Place navigation components within a ToolbarItem in the masthead's Toolbar for top-level navigation.

### Responsive Behavior
The Page component automatically applies breakpoint classes (default, sm, md, lg, xl, 2xl) based on viewport width, with customizable behavior through `getBreakpoint` prop.

## Padding Configuration

Control section spacing via the `padding` prop object with breakpoint keys:

```
padding={{ 
  default: 'noPadding', 
  md: 'padding' 
}}
```

Later breakpoints inherit previous values, allowing progressive enhancement across screen sizes.

## Best Practices

1. **Managed Sidebars**: Use `isManagedSidebar={true}` on Page to eliminate manual state handling
2. **Content Filling**: When multiple sections should fill space, available height distributes equally
3. **Width Limiting**: Combine `isWidthLimited` with `isCenterAligned` for optimal centered layouts
4. **Multiple Sidebar Bodies**: Leverage multiple PageSidebarBody components for context selectors, navigation sections, and additional controls
5. **Section Types**: Match `type` prop to content—use "breadcrumb" for breadcrumbs, "tabs" for tabbed navigation, "default" for main content

## CSS Customization

The component exposes extensive custom properties for theming, including sidebar width, spacing, colors, and animations. Default sidebar width: 18.125rem with adjustable insets at 1.5rem.
