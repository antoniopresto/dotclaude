# PatternFly Spacing System

## Core Spacer Values

PatternFly uses **rem-based sizing** (relative to 16px root) for all spacers:

| Size | Token | Value |
|------|-------|-------|
| xs | `--pf-t--global--spacer--xs` | 0.25rem (4px) |
| sm | `--pf-t--global--spacer--sm` | 0.5rem (8px) |
| md | `--pf-t--global--spacer--md` | 1rem (16px) |
| lg | `--pf-t--global--spacer--lg` | 1.5rem (24px) |
| xl | `--pf-t--global--spacer--xl` | 2rem (32px) |
| 2xl | `--pf-t--global--spacer--2xl` | 3rem (48px) |
| 3xl | `--pf-t--global--spacer--3xl` | 4rem (64px) |
| 4xl | `--pf-t--global--spacer--4xl` | 5rem (80px) |

## Semantic Spacer Categories

Beyond generic sizing, PatternFly provides **context-specific tokens**:

- **Action spacers** (`--pf-t--global--spacer--action`): Button and action padding
- **Control spacers** (`--pf-t--global--spacer--control`): Input fields and menu toggles
- **Gap spacers** (`--pf-t--global--spacer--gap`): Space between elements
- **Gutter spacers** (`--pf-t--global--spacer--gutter`): Grid layout spacing
- **Inset spacers** (`--pf-t--global--spacer--inset`): Structural padding (masthead, navigation)

## Design Best Practices

**Key consideration:** "Some components like icons, buttons, and input fields, have a fixed amount of padding built in. Remember to account for this extra space when laying the content out."

**For typography spacing:**
- Vary sizes based on text type (16px between title and body text vs. 8px between list items)
- Account for line height when spacing text-based content
- Use standard global spacer tokens for text layouts
