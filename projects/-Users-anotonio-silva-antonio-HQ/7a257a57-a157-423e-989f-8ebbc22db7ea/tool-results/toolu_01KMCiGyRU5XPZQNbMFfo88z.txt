# PatternFly Drawer Component Overview

## Purpose
The Drawer is "a sliding panel that enters from outside of the viewport, which can be configured to either overlay content or create a sidebar by pushing content."

## Key Panel Positions
- **End** (default): Panel slides in from the right
- **Start**: Panel slides in from the left
- **Bottom**: Panel slides up from the bottom
- **Inline variants**: Content and panel display side-by-side

## Core Props

**Drawer Component:**
- `isExpanded` (boolean): Controls panel visibility
- `isInline` (boolean): Displays content and panel side-by-side
- `isStatic` (boolean): Panel always visible, doesn't close
- `position`: 'start' | 'end' | 'bottom'
- `onExpand`: Callback after 250ms animation completes

**DrawerPanelContent:**
- `isResizable` (boolean): Enables manual panel resizing
- `focusTrap`: Manages focus behavior when panel opens
- `widths`: Responsive sizing at different breakpoints
- `colorVariant`: 'default' | 'secondary' | 'no-background'

## Layout Patterns

**Standard Structure:**
- DrawerSection: Optional header area
- DrawerContent: Main page content
- DrawerPanelContent: Slide-out panel with head, body, and actions

**Customization Options:**
- Adjustable padding on content and panels via `hasPadding` prop
- Multiple color variants for visual differentiation
- Resizable panels with keyboard support (arrow keys)

## Best Practices

1. **Focus Management**: Enable `focusTrap` on panels that require interaction
2. **Responsive Design**: Configure `widths` for various breakpoints
3. **Static Behavior**: Use `isStatic` at larger breakpoints to keep navigation persistent
4. **Accessibility**: Default close button includes "Close drawer panel" aria-label
