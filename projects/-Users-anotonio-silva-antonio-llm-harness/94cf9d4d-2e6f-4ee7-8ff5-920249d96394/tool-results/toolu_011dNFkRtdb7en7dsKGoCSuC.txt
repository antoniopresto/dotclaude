     1→# Run Assignment Recommendation UI Specification
     2→
     3→**Document ID:** MILEAGE-UX-005
     4→**Version:** 1.0
     5→**Created:** 2026-01-08
     6→**RFQ Reference:** Sections 4.1.2, 5, 6

     8→---
     9→
    10→## 1. Overview
    11→
    12→### 1.1 Purpose
    13→
    14→This specification defines the user experience for the **Run Assignment Recommendation** interface, enabling operations staff to allocate trains to daily routes based on AI-optimized recommendations considering fitness status, mileage balance, and PM schedules.
    15→
    16→### 1.2 Target Users
    17→
    18→| Role | Primary Tasks | Device |
    19→|------|---------------|--------|
    20→| Operations Planner | Generate daily schedules, assign trains to routes | Desktop |
    21→| Depot Supervisor | Review assignments, approve changes | Desktop/Tablet |
    22→| Control Room | Monitor real-time assignments, handle exceptions | Desktop (multi-monitor) |
    23→| Maintenance Liaison | Coordinate train release/requests with operations | Desktop |
    24→
    25→### 1.3 RFQ Requirements Addressed
    26→
    27→- **Section 4.1.2:** "Recommends the most suitable train for a given run based on fitness status"
    28→- **Section 5:** "Train run assignments" in centralised dashboard
    29→- **Section 6:** "Daily operation requirements per train run ID"
    30→- **Section 5:** "Operations-Maintenance interface"

   114→### 3.2 Component Specifications
   115→
   116→#### 3.2.1 Fleet Overview Panel (Left Sidebar)
   117→
   118→**Purpose:** Quick status summary of entire fleet
   119→
   120→| Metric | Display | Color Coding |
   121→|--------|---------|--------------|
   122→| In Service | Count + progress bar | Green #22C55E |
   123→| In Maintenance | Count + progress bar | Amber #F59E0B |
   124→| Out of Service | Count + progress bar | Red #EF4444 |
   125→| Reserved | Count + progress bar | Blue #3B82F6 |
   126→
   127→**Interactions:**
   128→- Click metric to filter train pool
   129→- Hover shows breakdown by fleet type
   130→
   131→#### 3.2.2 Train Pool Panel
   132→
   133→**Purpose:** Draggable pool of available trains for assignment
   134→
   135→**Train Card (Compact):**
   136→```
   137→┌──────────────────────┐
   138→│ 🟢 KS-012           │
   139→│ 42,350 km | PM1 due │
   140→│ ▓▓▓▓▓▓▓░░░ 72%     │
   141→└──────────────────────┘
   142→```
   143→
   144→**Data Displayed:**
   145→- Train ID with fitness indicator (🟢 FIT, 🟡 RESTRICTED, 🔴 UNFIT)
   146→- Current mileage
   147→- Next PM level and progress
   148→- Visual progress to PM threshold
   149→
   150→**Filtering:**
   151→- Fleet type: Kinki Sharyo | Alstom | All
   152→- Fitness: FIT | RESTRICTED | All
   153→- PM Status: All | Due Soon | Safe
   154→
   155→**Sorting:**
   156→- By Mileage (Low to High) - default
   157→- By PM Progress (%)
   158→- By Train ID
   159→- By Score (for selected route)

   252→## 4. AI Recommendation Patterns
   253→
   254→### 4.1 Suggestion States
   255→
   256→| State | Visual | Action Buttons |
   257→|-------|--------|----------------|
   258→| Pending Review | Yellow border, 🤖 icon | Accept, Reject |
   259→| Accepted | Green checkmark, fade animation | Undo (5s) |
   260→| Rejected | Strikethrough, gray | Select Alternative |
   261→| User Override | Blue border, 👤 icon | None (user chose) |
   262→
   263→### 4.2 Bulk Actions
   264→
   265→**Accept All Recommendations:**
   266→```
   267→┌────────────────────────────────────────────────┐
   268→│  Accept 8 AI Recommendations?                  │
   269→│                                                │
   270→│  Summary:                                      │
   271→│  • 5 Express routes assigned                   │
   272→│  • 2 Local routes assigned                     │
   273→│  • 1 Shuttle route assigned                    │
   274→│                                                │
   275→│  Fleet Balance: 87/100 (Good)                  │
   276→│  Warnings: 1 (PM approaching for AL-003)       │
   277→│                                                │
   278→│  [Review Details]  [Cancel]  [Accept All]      │
   279→└────────────────────────────────────────────────┘
   280→```
   281→
   282→### 4.3 Conflict Resolution
   283→
   284→When user input conflicts with AI recommendation:
   285→
   286→```
   287→┌────────────────────────────────────────────────┐
   288→│  ⚠️ Override AI Recommendation?                │
   289→│                                                │
   290→│  AI suggested: KS-012 (Score: 92)              │
   291→│  You selected: KS-005 (Score: 78)              │
   292→│                                                │
   293→│  Trade-offs:                                   │
   294→│  • KS-005 has higher mileage (+4,200 km)       │
   295→│  • KS-005 PM due sooner (5 days vs 15 days)    │
   296→│                                                │
   297→│  [Keep AI] [Use My Selection]                  │
   298→└────────────────────────────────────────────────┘
   299→```

   520→## 9. Gestalt Principles Applied
   521→
   522→### 9.1 Proximity
   523→
   524→- **Fleet Overview:** Metrics grouped together in left sidebar
   525→- **Train Cards:** Related data (ID, mileage, PM) tightly grouped
   526→- **Recommendations:** Suggestion and alternatives in same panel
   527→
   528→### 9.2 Similarity
   529→
   530→- **Train Status:** Consistent color coding across all views
   531→  - 🟢 Green = FIT / Safe / Good
   532→  - 🟡 Yellow = RESTRICTED / Approaching / Fair
   533→  - 🔴 Red = UNFIT / Overdue / Poor
   534→- **Card Components:** Same border radius (8px), shadow (2px blur)
   535→
   536→### 9.3 Enclosure
   537→
   538→- **Route Slots:** Clear boundary for drop zones
   539→- **Message Cards:** Distinct containers for each message
   540→- **Panels:** Sidebar and main content clearly separated
   541→
   542→### 9.4 Continuity
   543→
   544→- **Weekly Grid:** Horizontal timeline reading left-to-right
   545→- **Score Breakdown:** Vertical list with progress bars aligned
   546→- **Fleet Balance:** Left-aligned labels, right-aligned values
   547→
   548→### 9.5 Figure-Ground
   549→
   550→- **Drag Preview:** Elevated shadow distinguishes dragged item
   551→- **Active Route:** Highlighted background during assignment
   552→- **Modal Dialogs:** Dimmed backdrop focuses attention
   553→
   554→### 9.6 Common Fate
   555→
   556→- **Multi-Select:** Selected items move/highlight together
   557→- **Route Groups:** Routes in same category collapse/expand together

   735→## 14. Unknown Unknowns Discovered
   736→
   737→### 14.1 Operational Challenges
   738→
   739→1. **Multi-day assignments:** Can a train be pre-assigned for multiple days? How to handle PM interruptions?
   740→2. **Split shifts:** Can one train serve multiple routes in a day? How to sequence?
   741→3. **Depot constraints:** If train is at Depot A, can it serve route starting at Depot B?
   742→4. **Crew dependencies:** Is crew assignment coupled with train assignment?
   743→5. **Fleet mixing rules:** Can Kinki Sharyo and Alstom be assigned interchangeably to all routes?
   744→
   745→### 14.2 UX Challenges
   746→
   747→6. **Undo granularity:** Undo single assignment vs undo all changes?
   748→7. **Auto-save vs manual save:** Should changes persist immediately or require explicit save?
   749→8. **Concurrent editing:** What if two planners modify the same day?
   750→9. **Historical edits:** Can past assignments be modified? Audit trail?
   751→10. **Template schedules:** Can users save assignment patterns as templates?
   752→
   753→### 14.3 AI/Algorithm Challenges
   754→
   755→11. **Score transparency:** How much algorithm detail to expose to users?
   756→12. **Override learning:** Should the system learn from user overrides to improve future suggestions?
   757→13. **Fairness constraints:** Should mileage be strictly balanced or allow temporary imbalances?
   758→14. **Look-ahead horizon:** Should AI consider tomorrow's schedule when suggesting today?
   759→15. **Constraint conflicts:** How to prioritize when fitness and mileage balance conflict?
   760→
   761→### 14.4 Integration Challenges
   762→
   763→16. **Maximo sync timing:** When are fitness statuses updated from Maximo?
   764→17. **Real-time mileage:** Is actual mileage available in real-time or end-of-day?
   765→18. **Schedule lock time:** When does a draft schedule become immutable?
   766→19. **Notification triggers:** Which changes should trigger Ops-Maint messages automatically?
   767→20. **Export format:** What format does the external scheduling system expect?
