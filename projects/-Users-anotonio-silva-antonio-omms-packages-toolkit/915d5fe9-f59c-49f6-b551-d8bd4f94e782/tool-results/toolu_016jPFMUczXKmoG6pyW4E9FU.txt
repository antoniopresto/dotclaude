     2‚Üí
     3‚Üí## Overview
     4‚ÜíThe schema engine wraps `@sinclair/typebox` to provide runtime safety. While robust, the current implementation prioritizes developer experience (DX) over runtime performance, making it unsuitable for hot paths.
     5‚Üí
     6‚Üí## Critical Issues
     7‚Üí
     8‚Üí### 1. Interpreted vs. Compiled Validation (`lib/type.ts`)
     9‚Üí- **Location:** `createBaseType` uses `Value.Check(schema, value)`.
    10‚Üí- **Problem:** `Value.Check` is an **interpreter**. It traverses the schema definition object and the data structure recursively on every call.
    11‚Üí- **Impact:** $O(SchemaSize + DataSize)$ complexity. Extremely slow compared to JIT compilation.
    12‚Üí- **Fix:** Use `TypeCompiler` from `@sinclair/typebox/compiler`.
    13‚Üí  ```typescript
    14‚Üí  // Current (Slow)
    15‚Üí  import { Value } from 'typebox/value';
    16‚Üí  Value.Check(schema, value);
    17‚Üí
    18‚Üí  // Recommended (Fast)
    19‚Üí  import { TypeCompiler } from 'typebox/compiler';
    20‚Üí  const check = TypeCompiler.Compile(schema);
    21‚Üí  check.Check(value);
    22‚Üí  ```
    23‚Üí
    24‚Üí### 2. Assertions Re-create Schemas (`helpers.ts`)
    25‚Üí- **Location:** `invariantType`.
    26‚Üí- **Code:** `createType(definition).parse(value)` inside the function.
    27‚Üí- **Problem:** Unless the LRU cache hits, this constructs a new Schema Object and new Validator closure every time an assertion is made.
    28‚Üí- **Fix:** Accept pre-compiled validators or static schema references instead of string definitions for frequent assertions.
    29‚Üí
    30‚Üí### 3. Regex in Error Handling (`helpers.ts`)
    31‚Üí- **Location:** `invariantType` error formatting.
    32‚Üí- **Problem:** Uses `customMessage.replace(...)` with Regex.
    33‚Üí- **Impact:** Minor, but adds unnecessary overhead to what should be a fast assertion check.
    34‚Üí
    35‚Üí### 4. Limited Type Inference (`infer-type.ts`)
    36‚Üí- **Problem:** Inference depth is artificially limited to prevent recursion errors.
    37‚Üí- **Impact:** Deeply nested objects in the state might revert to `unknown` or `any` in TypeScript, losing type safety despite runtime overhead.
    38‚Üí

    42‚Üí- [ ] Remove heavy string parsing from `toTypeBox` initialization where possible.
