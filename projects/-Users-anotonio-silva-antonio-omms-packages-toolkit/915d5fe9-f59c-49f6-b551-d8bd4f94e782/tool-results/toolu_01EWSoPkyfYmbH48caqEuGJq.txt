     1→# FIXME: Helpers & Utilities Audit

     3→## Overview
     4→General utility analysis revealed performance bottlenecks and one critical stability risk.
     5→
     6→## Critical Issues
     7→
     8→### 1. CRITICAL: `devLog` Crash Risk (`logs.ts`)
     9→- **Location:** `devLog` function.
    10→- **Code:**
    11→  ```typescript
    12→  if (typeof messageOrError !== 'string') {
    13→    throw messageOrError; // <--- CRITICAL
    14→  }
    15→  ```
    16→- **Problem:** The function throws errors instead of logging them. If a developer uses `devLog(error)` inside a `catch` block, it re-throws the error, potentially crashing the app unexpectedly.
    17→- **Fix:** Change to `console.error` or rename function to `devAssert`.
    18→
    19→### 2. Performance: Square N Problem (`object-utils.ts` & `list.ts`)
    20→- **Location:** `reduceObject` (object-utils.ts) and `reduce` (list.ts).
    21→- **Code:**
    22→  ```typescript
    23→  // In both files:
    24→  return items.reduce((acc, entry) => ({
    25→      ...acc,       // <--- Copies accumulator every iteration
    26→      ...next,
    27→  }), {});
    28→  ```
    29→- **Problem:** Spreading `...acc` inside a reduce loop creates $O(N^2)$ complexity. For large lists or objects, this is disastrously slow.
    30→- **Fix:** Use mutation.
    31→  ```typescript
    32→  const result = {};
    33→  for (const [key, value] of Object.entries(items)) {
    34→     Object.assign(result, reducer([key, value]));
    35→  }
    36→  return result;
    37→  ```
    38→
    39→### 3. V8 Optimization: Hidden Classes (`function.ts`)
    40→- **Location:** `defineFunctionName`.
    41→- **Code:** `Object.defineProperties(fn, { name: ... })`.
    42→- **Problem:** Modifying the `name` property of a function changes its internal V8 structure (Hidden Class transition).
    43→- **Impact:** Doing this dynamically (e.g., inside `MapSlice`) de-optimizes the function.
    44→- **Fix:** Use this strictly for initialization/debugging, never in hot paths.
    45→
    46→### 4. "Magic" Code (`object-utils.ts`)
    47→- **Location:** `objectMap`.
    48→- **Problem:** Uses `Object.setPrototypeOf` and `Proxy` to make an array behave like an object.
    49→- **Impact:** High cognitive load, brittle, and likely slow due to prototype chain manipulation.
    50→- **Fix:** Replace with standard ES6 `Map` or plain Objects.
    51→
    52→### 5. Expensive Stack Traces (`error-utils.ts`)
    53→- **Location:** `invariant` and `moveStackTrace`.
    54→- **Problem:** `Error.captureStackTrace` is extremely slow in V8. `invariant()` calls this by default on failure, which is fine, but `moveStackTrace` might be used in control flows.
    55→- **Fix:** Ensure `invariant` is stripped in production or that `captureStackTrace` is lazy.
    56→
    57→## Action Plan

    59→- [ ] **Urgent:** Rewrite `reduceObject` to be $O(N)$.
    60→- [ ] Audit usages of `defineFunctionName` and remove from critical paths.
    61→- [ ] Deprecate `objectMap` and `createProxy` (unless strictly necessary for singleton lazy-loading).
