    32→/**

    35→ * Defines the maintenance tasks for each fleet with their intervals
    36→ * and resource requirements.
    37→ *
    38→ * Key concepts:
    39→ * - interval_km / interval_months: Whichever triggers first
    40→ * - satisfies_codes: PM hierarchy (C4 satisfies C1/C2/C3)
    41→ * - early/late tolerance: Compliance window (default ±10%)
    42→ *
    43→ * Version management: PM intervals can change over the 12-year contract.
    44→ * The effective_from/effective_to fields support versioning.

    47→  'pm_types',
    48→  {

   131→/**
