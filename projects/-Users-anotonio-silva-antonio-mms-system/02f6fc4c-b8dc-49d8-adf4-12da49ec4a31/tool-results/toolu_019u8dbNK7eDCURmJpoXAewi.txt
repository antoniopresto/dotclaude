    31→## Quick Start
    32→
    33→Get Ralph running in 5 minutes:
    34→
    35→```bash
    36→# 1. Initialize Ralph in your project
    37→ralph init --existing
    38→
    39→# 2. Set required timeouts (add to .bashrc)
    40→export BASH_DEFAULT_TIMEOUT_MS=1800000
    41→export BASH_MAX_TIMEOUT_MS=7200000
    42→
    43→# 3. Create initial plan
    44→./ralph/loop.sh plan 3
    45→
    46→# 4. Start building
    47→./ralph/loop.sh
    48→```
    49→
    50→**Essential Commands**:
    51→```bash
    52→./ralph/loop.sh              # Start building (unlimited)
    53→./ralph/loop.sh plan         # Start planning
    54→./ralph/loop.sh 50           # Build with max 50 iterations
    55→./ralph/circuit-breaker.sh status   # Check loop state
    56→./ralph/monitor.sh stats     # Quick progress check
    57→```
    58→
    59→**Steering Ralph** (mid-run):
    60→```bash
    61→# Add urgent task
    62→./ralph/scripts/request.sh "Fix login bug"
    63→
    64→# Add session focus
    65→./ralph/scripts/focus.sh "Only work on API endpoints"
    66→
    67→# Archive completed work (if plan too large)
    68→./ralph/scripts/archive-completed.sh
    69→```

    73→## 1. Introduction to the Ralph Wiggum Method
    74→
    75→### 1.1 What is Ralph Wiggum?
    76→
    77→Ralph Wiggum is an autonomous loop methodology that feeds prompts to Claude Code, enabling continuous, self-directed development over extended periods (24h+). Created by Geoffrey Huntley, the method is named after the Simpsons character and embodies the principle of being "deterministically bad in an undeterministic world."
    78→
    79→The core insight is simple but powerful: **progress persists in files and git history, NOT in the LLM's context window**. This eliminates context rot, enables infinite-length development sessions, and provides complete auditability.
    80→
    81→### 1.2 The Central Philosophy
    82→
    83→**"Disk is state, git is memory."**
    84→
    85→This foundational principle means:
    86→- Every piece of progress is written to files
    87→- Every change is committed to git
    88→- The LLM's context is transient and disposable
    89→- Recovery from any failure is possible through git history
    90→
    91→**The Playground Metaphor** (from Dev Interrupted Podcast, Jan 2026):
    92→
    93→> "Ralph is given instructions to construct a playground. Ralph is very good at making playgrounds, but he comes home bruised because he fell off the slide, so one then tunes Ralph by adding a sign next to the slide saying 'SLIDE DOWN, DON'T JUMP, LOOK AROUND.'"
    94→
    95→This illustrates the tuning process: when Ralph fails in a specific way, you add "signage" (instructions) to prevent that failure from recurring.
    96→
    97→### 1.3 The 5 Golden Rules
    98→
    99→1. **"Disk is state, git is memory"** — Never depend on context for history
   100→2. **"Plans are disposable"** — Regenerate rather than accumulate
   101→3. **"Fresh context is reliability"** — Rotate at 40% capacity, not 90%
   102→4. **"Ralph maintains Ralph"** — Use subagents for self-cleanup
   103→5. **"Deterministically bad > unpredictably good"** — Expect failures, tune prompts
   104→
   105→### 1.4 On Context and Compaction
   106→
   107→From Geoffrey Huntley:
   108→
   109→> "Compaction is like a Jenga tower - if you pull out the wrong bricks at the bottom, the tower falls over."
   110→
   111→> "It's actually a loss. It's a lossy function... If the specifications get compacted out, next thing you know, it starts making stuff up."
   112→
   113→> "The real number is like 176k tokens, but if you put in MCP servers, it gets down to 120k... So Ralph is a deliberate attempt to MINIMIZE allocation. So I never get a compaction event."
   114→
   115→### 1.5 When to Use Ralph
   116→
   117→Ralph is ideal for:
   118→- **Long-duration projects (24h+)** — Continuous autonomous development
   119→- **Complex implementations** — Multi-phase, multi-file changes
   120→- **Tasks requiring persistence** — Beyond context window limits
   121→- **Full auditability** — Complete git history of all changes
   122→- **Hands-off development** — Work while you sleep

   126→## 2. System Architecture
   127→
   128→### 2.1 The Ralph Loop
   129→
   130→```
   131→┌────────────────────────────────────────────────────────────┐
   132→│                      RALPH LOOP                             │
   133→├────────────────────────────────────────────────────────────┤
   134→│                                                            │
   135→│   ┌─────────────┐    ┌──────────────┐    ┌────────────┐   │
   136→│   │ Read Prompt │ -> │ Claude Code  │ -> │ Git Commit │   │
   137→│   │   Files     │    │  Execution   │    │   Changes  │   │
   138→│   └─────────────┘    └──────────────┘    └────────────┘   │
   139→│         ^                                      │           │
   140→│         │                                      │           │
   141→│         └──────────── Loop ────────────────────┘           │
   142→│                                                            │
   143→│   State persists in:                                       │
   144→│   - IMPLEMENTATION_PLAN.md (task state)                    │
   145→│   - Git history (code changes)                             │
   146→│   - AGENTS.md (operational knowledge)                      │
   147→│   - LEARNINGS.md (accumulated wisdom)                      │
   148→│                                                            │
   149→└────────────────────────────────────────────────────────────┘
   150→```
   151→
   152→### 2.2 State Persistence Model
   153→
   154→| Storage | Purpose | Persistence |
   155→|---------|---------|-------------|
   156→| Files (IMPLEMENTATION_PLAN.md, etc.) | Current state, active tasks | Read each iteration |
   157→| Git commits | Complete history, recovery | Permanent |
   158→| Archive files | Completed phases | Permanent |
   159→| LEARNINGS.md | Accumulated knowledge | Append-only |
   160→
   161→### 2.3 Fresh Context Principle
   162→
   163→Each iteration:
   164→1. Starts with a clean context
   165→2. Reads all state from files
   166→3. Executes one task
   167→4. Commits changes to git
   168→5. Updates state files
   169→6. Exits (context discarded)
   170→
   171→This prevents context rot and ensures reliability over extended periods.
   172→
   173→### 2.4 Directory Structure
   174→
   175→```
   176→ralph/
   177→├── loop.sh                    # Main loop script
   178→├── circuit-breaker.sh         # Infinite loop protection
   179→├── monitor.sh                 # Monitoring dashboard
   180→├── setup.sh                   # Initial setup
   181→│
   182→├── PROMPT_build.md            # Build mode instructions
   183→├── PROMPT_plan.md             # Planning mode instructions
   184→├── PROMPT_plan_work.md        # Scoped planning template
   185→├── PROMPT_discovery.md        # Discovery template
   186→│
   187→├── AGENTS.md                  # Operational quick reference (<60 lines)
   188→├── IMPLEMENTATION_PLAN.md     # Current task state (<400 lines)
   189→├── LEARNINGS.md               # Accumulated knowledge (append-only)
   190→├── @fix_plan.md               # Immediate task override (optional)
   191→│
   192→├── scripts/
   193→│   ├── request.sh             # Create task requests
   194→│   ├── focus.sh               # Manage session focus
   195→│   ├── archive-completed.sh   # Archive completed tasks
   196→│   ├── refresh-plan.sh        # Regenerate plan
   197→│   ├── check-plan-size.sh     # Validate plan size
   198→│   └── check-context.sh       # Token budget check
   199→│
   200→├── hooks/
   201→│   ├── pre-iteration.sh       # Pre-iteration checks
   202→│   ├── post-iteration.sh      # Post-iteration actions
   203→│   ├── session-start.sh       # Session initialization
   204→│   └── block-forbidden-imports.sh
   205→│
   206→└── IMPLEMENTATION_ARCHIVE/    # Archived completed work
   207→    └── YYYY-MM-DD/
   208→        ├── completed-tasks.md
   209→        ├── resolved-gaps.md
   210→        └── learnings.md
   211→```
   212→
   213→### 2.5 Key Files and Limits
   214→
   215→| File | Purpose | Max Size |
   216→|------|---------|----------|
   217→| `IMPLEMENTATION_PLAN.md` | Active tasks, blockers, progress | <400 lines |
   218→| `AGENTS.md` | Operational commands, quick reference | <60 lines |
   219→| `LEARNINGS.md` | Accumulated knowledge | Append-only |
   220→| `PROMPT_build.md` | Build mode instructions | ~200 lines |
   221→| `PROMPT_plan.md` | Planning mode instructions | ~100 lines |
   222→| `@fix_plan.md` | Immediate task override | Any |

   226→## 3. Main Loop Operation
   227→
   228→### 3.1 Synopsis
   229→
   230→```bash
   231→./ralph/loop.sh [MODE] [ITERATIONS] [OPTIONS]
   232→```
   233→
   234→### 3.2 Command Line Options
   235→
   236→| Option | Description | Default |
   237→|--------|-------------|---------|
   238→| `<number>` | Max iterations (0 = unlimited) | 0 |
   239→| `plan` | Planning mode | - |
   240→| `plan-work "desc"` | Scoped planning | - |
   241→| `-m, --model MODEL` | Model to use (opus/sonnet/haiku) | opus |
   242→| `-v, --verbose` | Verbose output | off |
   243→| `--no-push` | Don't push to remote | enabled |
   244→| `--no-stop-on-complete` | Continue after completion signal | off |
   245→| `-h, --help` | Show help | - |
   246→
   247→### 3.3 Build Mode (Default)
   248→
   249→Implements tasks from `IMPLEMENTATION_PLAN.md`:
   250→
   251→```bash
   252→./ralph/loop.sh         # Unlimited
   253→./ralph/loop.sh 50      # Max 50 iterations
   254→```
   255→
   256→**Workflow per iteration**:
   257→1. Check for `@fix_plan.md` (priority override)
   258→2. Check plan size (archive if > 400 lines)
   259→3. Read `IMPLEMENTATION_PLAN.md`
   260→4. Select highest-priority incomplete task
   261→5. Implement with validation (test → typecheck → lint → build)
   262→6. Commit changes
   263→7. Update plan, loop
   264→
   265→### 3.4 Planning Mode
   266→
   267→Generates or updates implementation plan from specs:
   268→
   269→```bash
   270→./ralph/loop.sh plan        # Unlimited planning
   271→./ralph/loop.sh plan 3      # Max 3 iterations
   272→```
   273→
   274→**Use when**:
   275→- Starting a new phase
   276→- Plan is stale or bloated
   277→- Need fresh gap analysis
   278→
   279→### 3.5 Scoped Planning Mode
   280→
   281→Plan specific work area:
   282→
   283→```bash
   284→./ralph/loop.sh plan-work "API endpoints" 5
   285→./ralph/loop.sh plan-work "database migrations" 3
   286→```
   287→
   288→**Requirements**:
   289→- Must be on a work branch (not main/master)
   290→- Sets `WORK_SCOPE` environment variable
   291→- Uses `PROMPT_plan_work.md`
   292→
   293→### 3.6 Environment Variables
   294→
   295→| Variable | Description | Default |
   296→|----------|-------------|---------|
   297→| `BASH_DEFAULT_TIMEOUT_MS` | Bash command timeout | 1800000 (30min) |
   298→| `BASH_MAX_TIMEOUT_MS` | Maximum bash timeout | 7200000 (2hr) |
   299→| `API_TIMEOUT_MS` | API request timeout | 600000 (10min) |
   300→| `MCP_TIMEOUT` | MCP timeout | 30000 (30sec) |
   301→| `MAX_CALLS_PER_HOUR` | Rate limit | 9999999 |
   302→| `MAX_RETRIES` | Retry count for transient errors | 3 |
   303→| `RETRY_DELAY` | Seconds between retries | 5 |
   304→
   305→### 3.7 Completion Signals
   306→
   307→Ralph stops when detecting these signals in output:
   308→
   309→- `ALL_TASKS_COMPLETE`
   310→- `PLAN_COMPLETE`
   311→- `EXIT_SIGNAL`
   312→- `<promise>DONE</promise>`
   313→- `<promise>COMPLETE</promise>`
   314→- `<promise>ALL_TASKS_COMPLETE</promise>`
   315→- `<promise>PLAN_COMPLETE</promise>`
   316→- `<promise>FIX_COMPLETE</promise>`
   317→- `<promise>BLOCKED</promise>`
   318→
   319→### 3.8 Examples
   320→
   321→```bash
   322→# Unlimited build iterations (default)
   323→./ralph/loop.sh
   324→
   325→# Max 50 iterations
   326→./ralph/loop.sh 50
   327→
   328→# Planning mode with 3 iterations
   329→./ralph/loop.sh plan 3
   330→
   331→# Scoped planning for specific work
   332→./ralph/loop.sh plan-work "API authentication" 5
   333→
   334→# Use sonnet model, verbose, max 20 iterations
   335→./ralph/loop.sh -m sonnet -v 20
   336→
   337→# Verbose mode only (unlimited)
   338→./ralph/loop.sh -v
   339→
   340→# Override NO_PUSH for this run
   341→NO_PUSH=false ./ralph/loop.sh
   342→
   343→# Strict mode (limited iterations and cost)
   344→CB_NO_PROGRESS_THRESHOLD=5 CB_MAX_COST_USD=50 ./ralph/loop.sh 30
   345→```

   349→## 4. Circuit Breaker
   350→
   351→### 4.1 Purpose
   352→
   353→Prevents runaway loops by monitoring:
   354→- Iterations without git commits (stuck loop)
   355→- Repeated errors
   356→- Estimated cost
   357→
   358→### 4.2 Commands
   359→
   360→```bash
   361→./ralph/circuit-breaker.sh status   # Show state
   362→./ralph/circuit-breaker.sh check    # Check if open (exit 1 = blocked)
   363→./ralph/circuit-breaker.sh reset    # Reset after stuck loop
   364→./ralph/circuit-breaker.sh errors   # Show recent errors
   365→```
   366→
   367→### 4.3 Configuration
   368→
   369→| Variable | Description | Default |
   370→|----------|-------------|---------|
   371→| `CB_NO_PROGRESS_THRESHOLD` | Iterations without commit | 10 |
   372→| `CB_SAME_ERROR_THRESHOLD` | Repeated error count | 5 |
   373→| `CB_MAX_COST_USD` | Cost limit in USD | 9999999* |
   374→| `COST_PER_ITERATION` | Estimated cost per iteration | 0.45 |
   375→
   376→*High defaults are intentional for 24h+ runs. See [Intentional Behaviors](#13-intentional-behaviors).
   377→
   378→### 4.4 Progress-Based Detection
   379→
   380→The circuit breaker uses progress-based detection rather than absolute limits:
   381→- **10 iterations without a git commit** = genuine stuck loop
   382→- Does **not** limit total iterations
   383→- Protects against stuck loops without limiting 24h+ runs
   384→
   385→### 4.5 State File
   386→
   387→Location: `/tmp/ralph-circuit-breaker/state.json`
   388→
   389→```json
   390→{
   391→  "iterations": 42,
   392→  "last_commit": "abc123",
   393→  "no_progress_count": 0,
   394→  "errors": [],
   395→  "estimated_cost": 18.90,
   396→  "start_time": 1705766400,
   397→  "session_id": "12345"
   398→}
   399→```
   400→
   401→### 4.6 Override for Stricter Limits
   402→
   403→```bash
   404→CB_NO_PROGRESS_THRESHOLD=5 CB_MAX_COST_USD=100 ./ralph/loop.sh
   405→```

   410→
   411→### 5.1 IMPLEMENTATION_PLAN.md Structure (<400 lines)
   412→
   413→```markdown
   414→# Implementation Plan
   415→
   416→> **Status**: 70% complete | **Active Phase**: 7
   417→> **Last Updated**: YYYY-MM-DD
   418→
   419→## BLOCKERS (Resolve First)
   420→- [ ] P1: [Description] - [Location]
   421→
   422→## IN PROGRESS
   423→- [ ] P2: [Current task being worked on]
   424→
   425→## NEXT UP (Max 10 items)
   426→- [ ] P2: [Next priority task]
   427→- [ ] P3: [Following task]
   428→
   429→## Quick Stats
   430→| Component | Status | Tests |
   431→|-----------|--------|-------|
   432→| Database  | Done   | N/A   |
   433→| Services  | Done   | 700+  |
   434→| API       | 98%    | 500+  |
   435→| UI        | 50%    | 7     |
   436→
   437→---
   438→*Full history: `ralph/IMPLEMENTATION_ARCHIVE/`*
   439→*Learnings: `ralph/LEARNINGS.md`*
   440→```
   441→
   442→### 5.2 Hierarchical Plan Architecture
   443→
   444→```
   445→ralph/
   446→├── IMPLEMENTATION_PLAN.md      # ONLY active tasks
   447→├── LEARNINGS.md                # Knowledge (append-only)
   448→└── IMPLEMENTATION_ARCHIVE/
   449→    ├── YYYY-MM-phase-1.md
   450→    ├── YYYY-MM-phase-2.md
   451→    └── ...
   452→```
   453→
   454→### 5.3 Archival System
   455→
   456→Automatic archival triggers:
   457→- Plan exceeds 400 lines
   458→- Phase completes
   459→- Branch changes (new feature)
   460→
   461→Manual archival:
   462→
   463→```bash
   464→./ralph/scripts/archive-completed.sh             # Archive completed
   465→./ralph/scripts/archive-completed.sh --dry-run   # Preview
   466→./ralph/scripts/archive-completed.sh --phase 3   # Archive specific phase
   467→./ralph/scripts/archive-completed.sh --all       # Archive everything
   468→```
   469→
   470→Output: `ralph/IMPLEMENTATION_ARCHIVE/YYYY-MM-DD/`
   471→
   472→### 5.4 Plan Compaction
   473→
   474→```bash
   475→./ralph plan:compact              # Smart compaction
   476→./ralph plan:compact --target 300 # Target line count
   477→./ralph plan:compact --aggressive # Maximum reduction
   478→```
   479→
   480→### 5.5 Token Budget Awareness
   481→
   482→From the community (agrimsingh/ralph-wiggum-cursor):
   483→
   484→```
   485→WARN_THRESHOLD=70000   # Send wrap-up warning
   486→ROTATE_THRESHOLD=80000 # Force context rotation
   487→```
   488→
   489→Key limits:
   490→- Claude read limit: ~25k tokens per file
   491→- IMPLEMENTATION_PLAN.md: max ~15k tokens
   492→- AGENTS.md: max ~2k tokens
   493→- **Rotate context at 40% capacity, not 90%**
   494→
   495→### 5.6 Self-Maintenance via Subagents
   496→
   497→Ralph maintains its own files using subagents:
   498→- Updates AGENTS.md when learning something new
   499→- Keeps IMPLEMENTATION_PLAN.md current
   500→- Cleans out completed items periodically
   501→- All via **SUBAGENTS** (doesn't pollute main context)
   502→
   503→From the playbook:
   504→
   505→> "When @IMPLEMENTATION_PLAN.md becomes large, periodically clean out the items that are completed from the file using a subagent."

   509→## 6. Mid-Course Corrections
   510→
   511→### 6.1 Philosophy: "Tune Ralph Like a Guitar"
   512→
   513→From Geoffrey Huntley:
   514→
   515→> "Each time Ralph does something bad, Ralph gets tuned — like a guitar."
   516→
   517→Ralph is steered through **file edits**, not runtime commands. The loop reads files fresh each iteration — edit the files, Ralph picks up changes on the next iteration.
   518→
   519→### 6.2 Method 1: @fix_plan.md (Highest Priority)
   520→
   521→Create `ralph/@fix_plan.md` for immediate tasks:
   522→
   523→```bash
   524→./ralph/scripts/request.sh "Create documentation for running the app"
   525→```
   526→
   527→Or manually create:
   528→
   529→```markdown
   530→# Immediate Task: Documentation
   531→
   532→## Objective
   533→Create comprehensive documentation for running the application locally.
   534→
   535→## Deliverables
   536→1. `docs/QUICKSTART.md` with full setup instructions
   537→2. Update `ralph/AGENTS.md` quick reference
   538→
   539→## Acceptance Criteria
   540→- Can follow docs from fresh clone to running app
   541→- Both frontend and backend accessible
   542→
   543→## Completion Signal
   544→<promise>DOCS_COMPLETE</promise>
   545→```
   546→
   547→**Behavior**: Ralph completes `@fix_plan.md` tasks before anything else, then deletes the file.
   548→
   549→### 6.3 Method 2: BLOCKER in IMPLEMENTATION_PLAN.md
   550→
   551→Add P1 blocker at top of plan:
   552→
   553→```markdown
   554→## BLOCKERS (Resolve First)
   555→
   556→- [ ] **P1 HUMAN REQUEST**: Create authentication tests
   557→  - Deliverable: `src/services/__tests__/AuthService.test.ts`
   558→  - Acceptance: All tests passing
   559→```
   560→
   561→### 6.4 Method 3: Signpost in PROMPT_build.md (Temporary)
   562→
   563→Add temporary focus to prompt file:
   564→
   565→```bash
   566→./ralph/scripts/focus.sh "Focus ONLY on Phase 7 UI components"
   567→```
   568→
   569→Or manually add to top of `PROMPT_build.md`:
   570→
   571→```markdown
   572→## SESSION FOCUS (Remove when complete)
   573→
   574→This session: Focus ONLY on Phase 7 (UI Components).
   575→- Do NOT work on backend services
   576→- Do NOT work on API endpoints
   577→- ONLY work on: React components, deck.gl layers
   578→
   579→---
   580→```
   581→
   582→### 6.5 Method 4: Signage in PROMPT_build.md (Permanent)
   583→
   584→When Ralph makes repeated mistakes, add permanent signage:
   585→
   586→```markdown
   587→## SIGNAGE
   588→- DO NOT use useState — use machine.useMachine() instead
   589→- DO NOT assume endpoints exist — search codebase first
   590→- ALWAYS run typecheck before committing
   591→```
   592→
   593→Signs are adjustments based on **observed failures**.
   594→
   595→### 6.6 Timing Considerations
   596→
   597→**When to edit files**:
   598→- During the 5-second pause between iterations (if configured)
   599→- While Claude is processing (file reads happen at iteration start)
   600→- After Ctrl+C (edit, then restart)
   601→
   602→**What gets re-read each iteration**:
   603→- `PROMPT_build.md` / `PROMPT_plan.md`
   604→- `@IMPLEMENTATION_PLAN.md`
   605→- `@AGENTS.md`
   606→- `specs/*`
   607→
   608→**Changes take effect on the next iteration.**
   609→
   610→### 6.7 Quick Reference
   611→
   612→| Need | Method | Command |
   613→|------|--------|---------|
   614→| Specific task | @fix_plan.md | `./ralph/scripts/request.sh "description"` |
   615→| Documentation | @fix_plan.md | `./ralph/scripts/request.sh --template docs "topic"` |
   616→| Specific tests | @fix_plan.md | `./ralph/scripts/request.sh --template test "component"` |
   617→| Session focus | Signpost | `./ralph/scripts/focus.sh "focus description"` |
   618→| Bug fix | BLOCKER | Edit `IMPLEMENTATION_PLAN.md` |

   710→
   711→### 8.1 PROMPT_build.md Structure
   712→
   713→```markdown
   714→## Pre-Flight (0.x)
   715→0a. CHECK @fix_plan.md — Complete if exists, then delete
   716→0b. CHECK Plan Size — Archive if > 400 lines
   717→0c. Study specs/* for context
   718→0d. Study @IMPLEMENTATION_PLAN.md
   719→
   720→## Work Phase (1.x)
   721→1. Choose highest-priority P1/P2 task
   722→2. Search codebase before implementing
   723→3. Implement completely (no stubs)
   724→4. Validate: test → typecheck → lint → build
   725→
   726→## Completion Phase (9.x)
   727→9a. Update @IMPLEMENTATION_PLAN.md with findings
   728→9b. Git commit with descriptive message
   729→9c. Check for remaining tasks
   730→
   731→## Exit Conditions
   732→- If ALL P1/P2 tasks complete: <promise>ALL_TASKS_COMPLETE</promise>
   733→- If @fix_plan.md completed: <promise>FIX_COMPLETE</promise>
   734→- If stuck 3+ attempts: <promise>BLOCKED</promise>
   735→
   736→## Guardrails (G.x)
   737→G1. PLAN SIZE: Archive if > 400 lines before starting work
   738→G2. TESTS: Required tests must pass before commit
   739→G3. SINGLE TASK: Complete ONE task per iteration
   740→G4. DOCUMENTATION: Capture the "why" in code and plan
   741→```

   744→

   749→3. Identify gaps between spec and implementation
   750→4. Create/update IMPLEMENTATION_PLAN.md
   751→5. Prioritize: P1 (blockers) > P2 (core) > P3 (nice-to-have)

   753→## Exit Condition

   806→
   807→### 9.1 Overview
   808→
   809→Hooks are scripts executed at specific points in the Ralph cycle:
   810→- **SessionStart**: Beginning of Claude session
   811→- **PreToolUse**: Before each tool use
   812→- **PostToolUse**: After each tool use
   813→- **Stop**: When Ralph stops
   814→
   815→### 9.2 Configuration Structure (.claude/settings.json)
   816→
   817→```json
   818→{
   819→  "hooks": {
   820→    "SessionStart": [{
   821→      "type": "command",
   822→      "command": "bash ralph/hooks/session-start.sh"
   823→    }],
   824→    "PreToolUse": [{
   825→      "matcher": "*",
   826→      "hooks": [{
   827→        "type": "command",
   828→        "command": "bash ralph/scripts/check-context.sh"
   829→      }]
   830→    }],
   831→    "PostToolUse": [
   832→      {
   833→        "matcher": "Write",
   834→        "hooks": [{
   835→          "type": "command",
   836→          "command": "bash .claude/hooks/block-forbidden-imports.sh"
   837→        }]
   838→      },
   839→      {
   840→        "matcher": "Edit",
   841→        "hooks": [{
   842→          "type": "command",
   843→          "command": "bash .claude/hooks/block-forbidden-imports.sh"
   844→        }]
   845→      }
   846→    ]
   847→  }
   848→}
   849→```
   850→
   851→### 9.3 Hook: session-start.sh
   852→
   853→Displays project info at session start:
   854→- Project name
   855→- Current branch
   856→- Plan status
   857→- Important alerts
   858→
   859→### 9.4 Hook: pre-iteration.sh
   860→
   861→Checks before each iteration:
   862→
   863→```bash
   864→#!/bin/bash
   865→PLAN_FILE="ralph/IMPLEMENTATION_PLAN.md"
   866→MAX_LINES=400
   867→
   868→if [ -f "$PLAN_FILE" ]; then
   869→    LINES=$(wc -l < "$PLAN_FILE")
   870→    if [ "$LINES" -gt "$MAX_LINES" ]; then
   871→        echo "Warning: IMPLEMENTATION_PLAN.md has $LINES lines (max: $MAX_LINES)"
   872→        echo "Consider running: ./ralph/scripts/archive-completed.sh"
   873→    fi
   874→fi
   875→```
   876→
   877→### 9.5 Hook: block-forbidden-imports.sh
   878→

   882→- Configurable via package list

  1244→## 13. Intentional Behaviors
  1245→
  1246→These behaviors appear to be defects but are intentional design decisions:
  1247→
  1248→### 13.1 NO_PUSH=true Default
  1249→
  1250→**Appears as**: Data loss risk (work only local)
  1251→
  1252→**Why intentional**:
  1253→- VPN interference causes git push failures unpredictably
  1254→- Push failures stop the loop, interrupting 24h+ runs
  1255→- Local commits still preserve work (manual push when convenient)
  1256→
  1257→**Override**: `NO_PUSH=false ./ralph/loop.sh`
  1258→
  1259→### 13.2 High Circuit Breaker Thresholds (9999999)
  1260→
  1261→**Appears as**: No safety limits
  1262→
  1263→**Why intentional**:
  1264→- 24h+ continuous runs required
  1265→- Human monitoring via dashboard provides oversight
  1266→- Progress-based detection (10 iterations without commit) protects against stuck loops
  1267→
  1268→**Override**: `CB_NO_PROGRESS_THRESHOLD=5 CB_MAX_COST_USD=100 ./ralph/loop.sh`
  1269→
  1270→### 13.3 Absurd Line Numbers (99, 999, 9999...)
  1271→
  1272→**Appears as**: Typos or errors
  1273→
  1274→**Why intentional**:
  1275→- Huntley's guardrail pattern
  1276→- High numbers indicate "always applicable" rules vs sequential steps
  1277→- Allows insertion of new instructions without renumbering
  1278→- Priority signaling: low numbers = do first; high numbers = always consider
  1279→
  1280→### 13.4 Case-Sensitive Completion Detection
  1281→
  1282→**Appears as**: Bug (might miss signals)
  1283→
  1284→**Why intentional**:
  1285→- Completion signals are code-defined, not user-typed
  1286→- Case-sensitivity prevents false positives
  1287→- Signals are uppercase by convention
  1288→
  1289→### 13.5 MAX_CALLS_PER_HOUR=9999999
  1290→
  1291→**Appears as**: No rate limiting
  1292→
  1293→**Why intentional**:
  1294→- Anthropic API has its own rate limits
  1295→- Retry logic handles throttling
  1296→- Artificial limits reduce productivity in 24h+ runs
  1297→
  1298→### 13.6 Override Table
  1299→
  1300→```bash

  1314→## 14. Troubleshooting
  1315→
  1316→### 14.1 Ralph Stops Unexpectedly
  1317→
  1318→**Check circuit breaker**:
  1319→```bash
  1320→./ralph/circuit-breaker.sh status
  1321→```
  1322→
  1323→**Common causes**:
  1324→- No progress threshold exceeded
  1325→- Cost limit reached
  1326→- Repeated errors
  1327→
  1328→**Fix**:
  1329→```bash
  1330→./ralph/circuit-breaker.sh reset
  1331→./ralph/loop.sh
  1332→```
  1333→
  1334→### 14.2 Timeout Errors
  1335→
  1336→**Symptom**: Operations killed after 2 minutes
  1337→
  1338→**Fix**: Set timeout environment variables:
  1339→```bash
  1340→export BASH_DEFAULT_TIMEOUT_MS=1800000
  1341→export BASH_MAX_TIMEOUT_MS=7200000
  1342→./ralph/loop.sh
  1343→```
  1344→
  1345→### 14.3 Plan Too Large
  1346→
  1347→**Symptom**: Context rot, partial reads, forgotten tasks
  1348→
  1349→**Check**:
  1350→```bash
  1351→wc -l ralph/IMPLEMENTATION_PLAN.md
  1352→./ralph/scripts/check-plan-size.sh
  1353→```
  1354→
  1355→**Fix**:
  1356→```bash
  1357→./ralph/scripts/archive-completed.sh
  1358→# Or regenerate:
  1359→./ralph/scripts/refresh-plan.sh
  1360→```
  1361→
  1362→### 14.4 Git Push Failures
  1363→
  1364→**Symptom**: Loop stops with push error (especially on VPN)
  1365→
  1366→**Workaround**: Run with NO_PUSH (default):
  1367→```bash
  1368→./ralph/loop.sh  # NO_PUSH=true by default
  1369→```
  1370→
  1371→**Manual push when convenient**:
  1372→```bash
  1373→git push origin $(git branch --show-current)
  1374→```
  1375→
  1376→### 14.5 Completion Signal Not Detected
  1377→
  1378→**Symptom**: Loop continues despite task completion
  1379→
  1380→**Check**: Signals are case-sensitive
  1381→```bash
  1382→grep -i "ALL_TASKS_COMPLETE" /tmp/ralph-outputs-*/iteration-*.json
  1383→```
  1384→
  1385→**Fix**: Ensure exact signal format in output:
  1386→```
  1387→<promise>ALL_TASKS_COMPLETE</promise>
  1388→```
  1389→
  1390→### 14.6 Stuck in Loop
  1391→
  1392→**Symptom**: Same task attempted repeatedly
  1393→
  1394→**Diagnosis**:
  1395→```bash
  1396→./ralph/circuit-breaker.sh errors
  1397→git log --oneline -10
  1398→```
  1399→
  1400→**Fixes**:
  1401→1. Add signage to prevent repeated mistake
  1402→2. Manually complete the blocking task
  1403→3. Reset and add explicit instructions
  1404→
  1405→```bash
  1406→./ralph/circuit-breaker.sh reset
  1407→# Add signage to PROMPT_build.md, then:
  1408→./ralph/loop.sh
  1409→```
  1410→
  1411→### 14.7 Subagents Not Working
  1412→
  1413→**Symptom**: Instructions mention "500 subagents"
  1414→
  1415→**Reality**: Claude Code supports ~3-5 concurrent subagents
  1416→
  1417→**Fix**: Use realistic language in prompts:
  1418→```
  1419→"Use subagents for parallel reads when beneficial"
  1420→NOT: "up to 500 parallel subagents"
  1421→```
  1422→

  1425→## 15. Best Practices
  1426→
  1427→### 15.1 From Geoffrey Huntley
  1428→
  1429→1. **Never interrupt mid-iteration** — Let it finish, then adjust
  1430→2. **Tune via file edits** — Not runtime commands
  1431→3. **Plans are disposable** — Regenerate rather than accumulate
  1432→4. **Add signage for failures** — Prevent same mistake twice
  1433→5. **Trust eventual consistency** — Ralph will get there
  1434→
  1435→### 15.2 Plan Management
  1436→
  1437→- Keep `IMPLEMENTATION_PLAN.md` under 400 lines
  1438→- Archive completed phases regularly
  1439→- Use specific, actionable task descriptions
  1440→- Include acceptance criteria
  1441→- Target: always <300 lines of active tasks
  1442→
  1443→### 15.3 Mid-Course Corrections
  1444→
  1445→- **Urgent task**: Use `@fix_plan.md`
  1446→- **Session focus**: Use signpost in `PROMPT_build.md`
  1447→- **Permanent rule**: Add to SIGNAGE section
  1448→- **Bug fix**: Add P1 BLOCKER to plan
  1449→
  1450→### 15.4 Long Runs (24h+)
  1451→
  1452→- Use `NO_PUSH=true` (default) to avoid VPN issues
  1453→- Monitor periodically: `./ralph/monitor.sh stats`
  1454→- High circuit breaker thresholds are intentional
  1455→- Trust git history for recovery
  1456→- Have backup strategy (periodic manual push)
  1457→
  1458→### 15.5 Debugging
  1459→
  1460→- Check circuit breaker status first
  1461→- Review recent commits: `git log --oneline -20`
  1462→- Check for errors: `./ralph/circuit-breaker.sh errors`
  1463→- Add verbose mode: `./ralph/loop.sh -v`
  1464→- Check state file sizes
  1465→
  1466→### 15.6 Token Awareness
  1467→
  1468→- IMPLEMENTATION_PLAN.md: max ~15k tokens
  1469→- AGENTS.md: max ~2k tokens
  1470→- Rotate context at 40% capacity, not 90%
  1471→- Compaction is lossy — avoid at all costs
  1472→- Fresh context > compacted context
  1473→
