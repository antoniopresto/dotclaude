   111→
   112→**Your Configuration is Intentional**: You want unlimited runs. Keep it.

   114→However, add **progress-based detection** instead of iteration limits:
   115→
   116→```bash

   118→CB_NO_PROGRESS_THRESHOLD=${CB_NO_PROGRESS_THRESHOLD:-9999999}
   119→
   120→# To (detects stuck loops without limiting total iterations):
   121→CB_NO_PROGRESS_THRESHOLD=${CB_NO_PROGRESS_THRESHOLD:-10}  # 10 iterations without commit = stuck

   151→## Part 2: Mid-Course Corrections (How to Steer Ralph)
   152→
   153→### 2.1 How Geoffrey Huntley Does It
   154→
   155→From the Dev Interrupted podcast (January 13, 2026):
   156→
   157→> *"Ralph is deterministically bad in an undeterministic world. You tune Ralph like a guitar."*
   158→
   159→**Methods Huntley Uses**:
   160→
   161→1. **Edit prompt files directly** (PROMPT_build.md, PROMPT_plan.md)
   162→2. **Add "signage" to prompts** - specific instructions for current issues
   163→3. **Never interrupt the loop** - let it finish, then adjust
   164→4. **Plans are disposable** - regenerate rather than accumulate
   165→
   166→### 2.2 Community Patterns for Mid-Course Corrections
   167→
   168→#### Pattern A: The "Signpost" Method (Most Common)
   169→
   170→Add temporary instructions to the TOP of `PROMPT_build.md`:
   171→
   172→```markdown

   570→1. **"Disk is state, git is memory"** — Never depend on context for history
