   134→## Codebase Patterns
   135→
   136→### Domain Services
   137→- Location: `src/services/`
   138→- Pattern: Class with injected dependencies
   139→- All services emit OpenTelemetry traces
   140→- Use Drizzle ORM for database access
   141→
   142→### API Routes
   143→- Location: `src/api/routes/`
   144→- Framework: Hono
   145→- All routes have OpenAPI spec
   146→- Use @belt/tools createSchema for validation
   147→
   148→### Workers
   149→- Location: `src/workers/`
   150→- Framework: BullMQ (Redis-backed)
   151→- Central exports: `src/workers/index.ts`
   152→- Use `registerWorker()` to create new workers
   153→- 8 queues: ingestion, tactical_scheduler, strategic_planner, optimizer, manhour_leveling, drift_monitor, notification, report
   154→
   155→**Running Workers:**
   156→```bash
   157→# Start all workers
   158→bun run src/workers/runner.ts
   159→
   160→# Start specific workers only
   161→WORKERS=optimizer,drift_monitor bun run src/workers/runner.ts
   162→
   163→# Workers require Redis to be running
   164→docker-compose up -d redis
   165→```
   166→
   167→**Implemented Processors:**
   168→- `optimizer` - Run assignment optimization
   169→- `drift_monitor` - Baseline drift analysis
   170→- `ingestion` - CSV mileage file processing
   171→- `maximo_sync` - Bidirectional sync with Maximo (push/pull/full)
   172→
   173→**Pending Processors (null placeholders):**
   174→- tactical_scheduler, strategic_planner, manhour_leveling, notification, report
   175→
   176→### Optimizer IPC
   177→- Protocol: NDJSON over stdin/stdout
   178→- Input: `{ type: "solve", data: {...} }`
   179→- Output: `{ type: "progress" | "solution" | "error", ... }`
   180→
   181→### Validation & State Management (@belt/tools)
   182→
   183→> **CRITICAL**: See README.md for complete @belt/tools API reference
   184→
   185→**Schema Validation** (replaces Zod):
   186→```typescript
   187→import { createSchema, createType, InferType } from '@belt/tools';
   188→
   189→const TrainModel = createSchema({
   190→  id: 'string',
   191→  trainNumber: 'string',
   192→  mileageKm: 'number',
   193→  fitnessStatus: { enum: ['FIT', 'RESTRICTED', 'GROUNDED'] },
   194→});
   195→type Train = InferType<typeof TrainModel>;
   196→const train = TrainModel.parse(data);
   197→```
   198→
   199→**State Management** (replaces React hooks + zustand):
   200→```typescript
   201→import { createSlice, createMachine } from '@belt/tools';
   202→
   203→const fleetSlice = createSlice({
   204→  id: 'fleet',
   205→  initial: { trains: [], selected: null },
   206→  cache: { policy: 'session' },
   207→  reducers: {
   208→    setTrains: (state, trains) => { state.trains = trains; },
   209→  },
   210→});
   211→
   212→const machine = createMachine({
   213→  id: 'mms',
   214→  slices: { fleet: fleetSlice },
   215→  trackMetadata: false, // Critical for 60fps
   216→});
   217→
   218→// In React component (NO useState/useEffect!)
   219→const { state, actions } = machine.useMachine('fleet');
   220→```
