#### Reference: @belt/tools Patterns
```typescript
// Schema validation (replaces Zod)
import { createSchema, createType, InferType } from '@belt/tools';

const TrainModel = createSchema({
  id: 'string',
  trainNumber: 'string',
  mileageKm: 'number',
  fitnessStatus: { enum: ['FIT', 'RESTRICTED', 'GROUNDED'] },
  tags: 'string[]',
  metadata: 'object?',
});
type Train = InferType<typeof TrainModel>;
const train = TrainModel.parse(data);

// State machine (replaces React hooks + zustand)
import { createSlice, createMachine } from '@belt/tools';

const fleetSlice = createSlice({
  id: 'fleet',
  initial: { trains: [], selected: null },
  cache: { policy: 'session' },
  reducers: {
    setTrains: (state, trains) => { state.trains = trains; },
    selectTrain: (state, id) => { state.selected = id; },
  },
});

const machine = createMachine({
  id: 'mms',
  slices: { fleet: fleetSlice },
  trackMetadata: false, // Critical for 60fps
});

// In React component (NO useState/useEffect!)
const { state, actions } = machine.useMachine('fleet');
```

4. **Baseline/drift**: Long-term plans stabilize via published baselines; drift thresholds prevent jitter
5. **Material constraints**: PM scheduling without parts availability is operationally useless
6. **Cold-start risk**: Historical PM data will be incomplete - explicit confidence scoring required
7. **Audit compliance**: 12-year government contract mandates immutable audit trail with full attribution
8. **Maximo Integration API (2026-01-20)**: Integration API routes registered at /api/v1/integration/*. Webhook validates event types: wo_status_changed, wo_created, wo_updated, wo_cancelled. Webhook updates shadow records and PM schedule status automatically. Status-based locking implemented (INPRG, COMP, CLOSE, CAN lock PMs from editing in MMS).
