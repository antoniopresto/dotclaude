## 6.1 Design System Overview

### 6.1.1 Technology Stack

| Component | Technology | Purpose |
|-----------|------------|---------|
| Framework | React 18 | Component architecture |
| State | Zustand | Global state management |
| Canvas | deck.gl | WebGL timeline rendering |
| Components | BlueprintJS v6 | UI component library |
| Styling | Tailwind CSS | Utility-first styling |
| Build | Vite | Development and bundling |
| Types | TypeScript | Type safety |

### 6.1.2 Design Principles

1. **Information Density**: Maximize data visibility without overwhelming
2. **Semantic Zoom**: Data representation adapts to zoom level
3. **Direct Manipulation**: Drag-and-drop with immediate feedback
4. **Progressive Disclosure**: Details on demand
5. **Constraint Visibility**: Show limits, not just violations
6. **Explainability**: Every system decision has visible rationale

### 6.3.5 Hybrid Rendering Architecture

**CRITICAL:** The canvas uses **hybrid rendering** to optimize for each zoom level:

| Zoom Level | Renderer | Rationale |
|------------|----------|-----------|
| Strategic (0-4) | **deck.gl (WebGL)** | Heatmap cells, no text needed |
| Tactical-Wide (4-8) | **deck.gl (WebGL)** | Compressed blocks, minimal text |
| Tactical (8-12) | **Konva (HTML5 Canvas)** | Gantt bars with crisp text labels |
| Operational (12-16) | **Konva (HTML5 Canvas)** | Detailed cards with readable text |
| Detail (16-20) | **React DOM** | Rich interactions, form inputs |

**Why Hybrid?**

deck.gl uses WebGL for rendering, which has limitations for text-heavy views:
- Font atlases blur during zoom operations
- Hit-testing for drag-and-drop is complex in WebGL
- Text rendering feels less crisp than native canvas/DOM

By switching to Konva (HTML5 Canvas 2D) at tactical zoom levels, we get:
- Crisp text rendering at all scales
- Native canvas hit-testing for smooth drag-and-drop
- Better performance for hundreds of labeled PM blocks

```typescript
interface CanvasLayers {
  // === DECK.GL LAYERS (Strategic/Overview) ===
  // Used for zoom levels 0-8 where text is minimal
  
  strategicHeatmapLayer: HeatmapLayer;    // PM density heatmap
  timelineAxisLayer: LineLayer;            // Time axis lines
  quarterBoundaryLayer: LineLayer;         // Quarter separators
  
  // === KONVA LAYERS (Tactical/Operational) ===
  // Used for zoom levels 8-16 where text labels are critical
  
  ganttBlockLayer: Konva.Layer;            // PM blocks with labels
  constraintWallLayer: Konva.Layer;        // Compliance windows
  trainLabelLayer: Konva.Layer;            // Train number labels
  tooltipLayer: Konva.Layer;               // Hover tooltips
  dragPreviewLayer: Konva.Layer;           // Drag-and-drop preview
  
  // === REACT DOM OVERLAY (Detail) ===
  // Used for zoom level 16+ where rich interactions needed
  
  detailCardOverlay: React.ReactNode;      // Expanded PM cards
  editFormOverlay: React.ReactNode;        // Inline editing
}

// Layer visibility by zoom level
function getActiveRenderer(zoom: number): 'deckgl' | 'konva' | 'react' {
  if (zoom < 8) return 'deckgl';
  if (zoom < 16) return 'konva';
  return 'react';
}
```

**Transition Handling:**

When crossing renderer boundaries, animate opacity to avoid jarring switches:

```typescript
function handleZoomChange(prevZoom: number, newZoom: number): void {
  const prevRenderer = getActiveRenderer(prevZoom);
  const newRenderer = getActiveRenderer(newZoom);
  
  if (prevRenderer !== newRenderer) {
    // Cross-fade over 200ms
    animateOpacity(prevRenderer, 1, 0, 200);
    animateOpacity(newRenderer, 0, 1, 200);
  }
}
```
