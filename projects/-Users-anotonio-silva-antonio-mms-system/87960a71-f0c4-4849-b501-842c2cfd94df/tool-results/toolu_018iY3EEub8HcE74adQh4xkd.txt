    44→## 3. Architecture Constraints (Non-Negotiable)
    45→
    46→| Feature | Allowed | Forbidden |
    47→|---------|---------|-----------|
    48→| **Scrolling** | Virtual Scroller (Math-based) | Native DOM Scrolling / Ghost Divs |
    49→| **Data Structures** | TypedArrays, SoA, Maps | Array of Objects (for massive data) |
    50→| **State** | Zustand (Transient & Global) | Redux, Context API (for high-freq updates) |
    51→| **Rendering** | PixiJS (WebGL), Shaders | DOM Elements (for >100 items), SVG |
    52→| **Loop** | `requestAnimationFrame` | `setInterval`, `setTimeout` (for anim) |

    56→## 4. Key Product Concepts
    57→
    58→### 4.1 Zoom Levels (8 Semantic Levels)
    59→
    60→| Temporal Scale | Default Visualization | Analysis Focus |
    61→|----------------|----------------------|----------------|
    62→| **12 Years** | Heatmap (saturation) | Total capacity, CAPEX/OPEX |
    63→| **3-5 Years** | Stacked Bars + Trends | Fleet lifecycle patterns |
    64→| **1-2 Years** | Combo Chart | Seasonality, peaks |
    65→| **Quadrimester** | Workload Equalizer | Workload vs capacity |
    66→| **1-3 Months** | Weekly Bars | Resource conflicts |
    67→| **2-4 Weeks** | Simplified Gantt | Track occupancy |
    68→| **1 Week** | Detailed Gantt | Per-train status |
    69→| **1-3 Days** | Capsules + Details | Fitness, km remaining |
    70→
    71→### 4.2 Visualization Selector
    72→
    73→Like Google Maps' `[Map] [Satellite] [Terrain]`, each zoom level offers multiple chart options:
    74→- User can switch between available visualizations for current zoom level
    75→- Each level has a default (recommended) visualization
    76→- See `docs/UX_FOR_DEMO.md` for full chart options per level
    77→
    78→### 4.3 Baseline vs Scenario Mode
    79→
    80→| Mode | Description | Editing |

    82→| **BASELINE** | Production plan | Restricted |
    83→| **SCENARIO** | Fork for "What-If" | Free editing |

    95→## 5. Coding Standards
    96→
    97→- **Strict Typing:** No `any`. Use `unknown` or Generics if unsure.
    98→- **JSDoc:** Required for all exported interfaces and complex algorithms (explain _why_, not just _what_).
    99→- **No Comments in Body:** Code should be self-documenting. Use comments only for "Why" or weird math.
   100→- **File Structure:**
   101→  - `src/engine/`: Core logic, Physics, Math, SoA
   102→  - `src/ui/`: React Components (Overlay, Sidebar, Constraint Deck)
   103→  - `src/render/`: PixiJS specific code (Shaders, Layers)
   104→  - `src/services/`: Mock API, future BFF integration
