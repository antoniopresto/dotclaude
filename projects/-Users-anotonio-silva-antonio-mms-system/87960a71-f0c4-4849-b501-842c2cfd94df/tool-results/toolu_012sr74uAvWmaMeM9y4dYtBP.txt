  3779вҶ’```typescript
  3780вҶ’interface DropValidation {
  3781вҶ’  isValid: boolean;
  3782вҶ’  reason?: string;
  3783вҶ’}
  3784вҶ’
  3785вҶ’function validateDrop(block: MaintenanceBlock, targetDate: Date, targetDepot: string): DropValidation {
  3786вҶ’  // Check depot capacity
  3787вҶ’  const capacity = getDepotCapacity(targetDepot, targetDate);
  3788вҶ’  if (capacity.utilized >= capacity.total) {
  3789вҶ’    return { isValid: false, reason: 'Depot at full capacity' };
  3790вҶ’  }
  3791вҶ’
  3792вҶ’  // Check maintenance interval
  3793вҶ’  const lastMaintenance = getLastMaintenance(block.trainId, block.type);
  3794вҶ’  const kmSinceLast = getCurrentKm(block.trainId) - lastMaintenance.km;
  3795вҶ’  if (kmSinceLast < getMinInterval(block.type)) {
  3796вҶ’    return { isValid: false, reason: 'Too early for this maintenance type' };
  3797вҶ’  }
  3798вҶ’
  3799вҶ’  // Check train availability
  3800вҶ’  const allocation = getAllocation(block.trainId, targetDate);
  3801вҶ’  if (allocation) {
  3802вҶ’    return { isValid: false, reason: `Train allocated to ${allocation.serviceCode}` };
  3803вҶ’  }
  3804вҶ’
  3805вҶ’  return { isValid: true };
  3806вҶ’}
  3807вҶ’```

  4122вҶ’#### 16.5 Key Architectural Decisions
  4123вҶ’
  4124вҶ’**ADR-001: Next.js App Router over Pages Router**
  4125вҶ’
  4126вҶ’- **Decision:** Use App Router (app/ directory)
  4127вҶ’- **Context:** Next.js 14 offers App Router with React Server Components
  4128вҶ’- **Rationale:** Better data fetching, layout nesting, streaming SSR
  4129вҶ’- **Consequences:** Some learning curve, but better long-term architecture
  4130вҶ’
  4131вҶ’**ADR-002: Fastify over Express**
  4132вҶ’
  4133вҶ’- **Decision:** Use Fastify as API framework
  4134вҶ’- **Context:** Need performant, type-safe backend
  4135вҶ’- **Rationale:** 2-3x faster than Express, built-in schema validation, TypeScript support
  4136вҶ’- **Consequences:** Slightly different plugin system, excellent performance
  4137вҶ’
  4138вҶ’**ADR-003: @belt/tools over React useState**
  4139вҶ’
  4140вҶ’- **Decision:** Use @belt/tools slice-based state management for application state
  4141вҶ’- **Context:** Complex state with animations, real-time updates, 60fps WebGL rendering
  4142вҶ’- **Rationale:** Slice-based architecture with Immer reducers avoids stale closure issues; `trackMetadata: false` option prevents GC thrashing in animation loops; auto-generated async actions simplify data fetching
  4143вҶ’- **Consequences:** Learning curve for slice/machine pattern, but eliminates common React bugs and enables 60fps performance
  4144вҶ’
  4145вҶ’**ADR-004: Blueprint.js over shadcn/ui**
  4146вҶ’
  4147вҶ’- **Decision:** Use Blueprint.js for UI components
  4148вҶ’- **Context:** Need enterprise-grade components with dark mode
  4149вҶ’- **Rationale:** Built-in dark mode, table/tree components, consistent design
  4150вҶ’- **Consequences:** Larger bundle, but comprehensive component library
  4151вҶ’
  4152вҶ’**ADR-005: Drizzle ORM over Prisma**
  4153вҶ’
  4154вҶ’- **Decision:** Use Drizzle for database access
  4155вҶ’- **Context:** Need type-safe, performant database layer
  4156вҶ’- **Rationale:** SQL-like syntax, better performance, smaller bundle
  4157вҶ’- **Consequences:** More manual work, but more control
  4158вҶ’
  4159вҶ’**ADR-006: Separate Python Solver**
  4160вҶ’
  4161вҶ’- **Decision:** Run solver as separate service, not Node.js library
  4162вҶ’- **Context:** Need OR-Tools for optimization
  4163вҶ’- **Rationale:** OR-Tools is Python-native, best performance
  4164вҶ’- **Consequences:** Extra service to deploy, HTTP overhead acceptable

  4305вҶ’#### 17.3 State Management (@belt/tools)
  4306вҶ’
  4307вҶ’MMS uses `@belt/tools` for state management, providing slice-based architecture with Immer reducers optimized for 60fps canvas rendering.
  4308вҶ’
  4309вҶ’**Overview**
  4310вҶ’
  4311вҶ’| Concept | Description | Use Case |
  4312вҶ’|---------|-------------|----------|
  4313вҶ’| **Sync Slice** | UI state with Immer reducers | Viewport, selections, UI toggles |
  4314вҶ’| **Async Slice** | Data fetching with auto-generated actions | API calls, loading states |
  4315вҶ’| **Machine** | Combines slices with computed selectors | Application-wide state |
  4316вҶ’
  4317вҶ’**Installation**
  4318вҶ’
  4319вҶ’```bash
  4320вҶ’bun add @belt/tools
  4321вҶ’```

  4323вҶ’**Sync Slice (UI State)**
  4324вҶ’
  4325вҶ’```typescript
  4326вҶ’// slices/viewportSlice.ts
  4327вҶ’import { createSlice } from '@belt/tools';
  4328вҶ’
  4329вҶ’export const viewportSlice = createSlice({
  4330вҶ’  id: 'viewport',
  4331вҶ’  initial: {
  4332вҶ’    camera: { x: 0, y: 0, zoom: 1 },
  4333вҶ’    lod: 'MESO' as const,
  4334вҶ’    selectedTrainId: null as string | null,
  4335вҶ’  },
  4336вҶ’  reducers: {
  4337вҶ’    setCamera: (state, camera: { x: number; y: number; zoom: number }) => {
  4338вҶ’      state.camera = camera;
  4339вҶ’      // Compute LOD based on zoom
  4340вҶ’      if (camera.zoom < 0.5) state.lod = 'MACRO';
  4341вҶ’      else if (camera.zoom > 2.0) state.lod = 'MICRO';
  4342вҶ’      else state.lod = 'MESO';
  4343вҶ’    },
  4344вҶ’    selectTrain: (state, trainId: string | null) => {
  4345вҶ’      state.selectedTrainId = trainId;
  4346вҶ’    },
  4347вҶ’  },
  4348вҶ’  cache: { policy: 'session' },
  4349вҶ’});
  4350вҶ’```

  4382вҶ’**Machine (Combining Slices)**
  4383вҶ’
  4384вҶ’```typescript
  4385вҶ’// state/canvasMachine.ts
  4386вҶ’import { createMachine } from '@belt/tools';
  4387вҶ’import { viewportSlice } from './viewportSlice';
  4388вҶ’import { trainsSlice } from './trainsSlice';
  4389вҶ’import { tasksSlice } from './tasksSlice';
  4390вҶ’
  4391вҶ’export const canvasMachine = createMachine({
  4392вҶ’  id: 'canvas',
  4393вҶ’  slices: {
  4394вҶ’    viewport: viewportSlice,
  4395вҶ’    trains: trainsSlice,
  4396вҶ’    tasks: tasksSlice,
  4397вҶ’  },
  4398вҶ’  selectors: {
  4399вҶ’    // Computed values (memoized)
  4400вҶ’    visibleTasks: (state) => {
  4401вҶ’      const { trains, tasks, viewport } = state;
  4402вҶ’      if (!tasks.value) return [];
  4403вҶ’      return filterByViewport(tasks.value, viewport.camera);
  4404вҶ’    },
  4405вҶ’    hasConflicts: (state) => {
  4406вҶ’      return state.tasks.value?.some(t => t.hasConflict) ?? false;
  4407вҶ’    },
  4408вҶ’    selectedTrain: (state) => {
  4409вҶ’      const id = state.viewport.selectedTrainId;
  4410вҶ’      return state.trains.value?.find(t => t.id === id);
  4411вҶ’    },
  4412вҶ’  },
  4413вҶ’  // CRITICAL for 60fps rendering - disables metadata tracking
  4414вҶ’  trackMetadata: false,
  4415вҶ’});
  4416вҶ’
  4417вҶ’// Generated type-safe hooks:
  4418вҶ’// - canvasMachine.useCanvas('viewport')  вҶ’ { state, actions }
  4419вҶ’// - canvasMachine.useCanvasSelector(s => s.visibleTasks)
  4420вҶ’```

  4465вҶ’**Performance: trackMetadata**
  4466вҶ’
  4467вҶ’For 60fps applications (WebGL, games, animations):
  4468вҶ’
  4469вҶ’```typescript
  4470вҶ’createMachine({

  4479вҶ’- No object allocation per dispatch
  4480вҶ’- No timestamp tracking
  4481вҶ’- Suitable for high-frequency updates (pan, zoom, animation frames)

  4484вҶ’
  4485вҶ’| XState (FSM) | @belt/tools (Slices) |
