    91вҶ’в”Ӯ   Persistent    в”Ӯ  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ>  в”Ӯ     Runtime     в”Ӯ  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ>  в”Ӯ    Rendering    в”Ӯ
    92вҶ’в”Ӯ   (JSON/API)    в”Ӯ               в”Ӯ   (in memory)   в”Ӯ               в”Ӯ    (canvas)     в”Ӯ

    99вҶ’**Why this separation?**
   100вҶ’- `Date` objects cannot be serialized to JSON (become `{}`)
   101вҶ’- Canvas needs numeric offsets for position calculations
   102вҶ’- Domain logic needs semantic dates for business rules

   109вҶ’
   110вҶ’```typescript
   111вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   112вҶ’// ENUMS & CONSTANTS
   113вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   114вҶ’
   115вҶ’type AssetType = 'ALSTOM' | 'KINKI';
   116вҶ’type TaskType = 'maintenance' | 'overhaul' | 'inspection';
   117вҶ’type MaintenanceLevel = 'A' | 'B' | 'C' | 'D';
   118вҶ’
   119вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   120вҶ’// TRAIN (Persistent)
   121вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

   123вҶ’interface Train {

   126вҶ’  assetType: AssetType;    // Fleet type

   128вҶ’
   129вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   130вҶ’// TASK (Persistent)
   131вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

   135вҶ’  trainId: string;         // Reference to Train.id

   138вҶ’  type: TaskType;          // Visual category
   139вҶ’  hasConflict: boolean;    // Conflict indicator

   141вҶ’
   142вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   143вҶ’// DATASET METADATA (Persistent)
   144вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   145вҶ’
   146вҶ’interface DatasetMeta {
   147вҶ’  startDate: string;       // ISO 8601: "2024-01-01"
   148вҶ’  trainCount: number;      // Number of trains
   149вҶ’  years: number;           // Time range in years
   150вҶ’}
   151вҶ’
   152вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   153вҶ’// GENERATED DATA (Complete dataset)
   154вҶ’// в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   155вҶ’
   156вҶ’interface GeneratedData {

   299вҶ’## Data Generation
   300вҶ’
   301вҶ’### Mock Data Generator
   302вҶ’
   303вҶ’Implementation: `apps/canvas/src/data/generator.ts`
   304вҶ’
   305вҶ’The generator creates:
   306вҶ’
   307вҶ’1. **50 Trains** with unique IDs (TRAIN-001 to TRAIN-050)
   308вҶ’2. **10 Years of Tasks** (~10-12 tasks per train per year)
   309вҶ’3. **~5% Conflict Rate** (hasConflict flag for testing)
   310вҶ’
   311вҶ’### Generation Algorithm
   312вҶ’
   313вҶ’```typescript
   314вҶ’function generateData(meta: DatasetMeta): GeneratedData {
   315вҶ’  const trains = generateTrains(meta.trainCount);
   316вҶ’  const tasks = generateTasks(trains, meta);
   317вҶ’
   318вҶ’  return { trains, tasks, meta };
   319вҶ’}
   320вҶ’
   321вҶ’function generateTrains(count: number): Train[] {
   322вҶ’  return Array.from({ length: count }, (_, i) => ({
   323вҶ’    id: `TRAIN-${String(i + 1).padStart(3, '0')}`,
   324вҶ’    name: `Train ${i + 1}`,
   325вҶ’    assetType: i < 30 ? 'ALSTOM' : 'KINKI',
   326вҶ’  }));
   327вҶ’}
   328вҶ’
   329вҶ’function generateTasks(trains: Train[], meta: DatasetMeta): Task[] {
   330вҶ’  const tasks: Task[] = [];
   331вҶ’  const types: TaskType[] = ['maintenance', 'overhaul', 'inspection'];
   332вҶ’
   333вҶ’  trains.forEach((train) => {
   334вҶ’    for (let year = 0; year < meta.years; year++) {
   335вҶ’      const tasksPerYear = 8 + Math.floor(Math.random() * 5);
   336вҶ’      for (let t = 0; t < tasksPerYear; t++) {
   337вҶ’        const dayOffset = year * 365 + Math.floor(Math.random() * 360);
   338вҶ’        const duration = 2 + Math.floor(Math.random() * 10);
   339вҶ’        const startDate = addDays(meta.startDate, dayOffset);
   340вҶ’        const endDate = addDays(startDate, duration);
   341вҶ’
   342вҶ’        tasks.push({
   343вҶ’          id: `${train.id}-${year}-${t}`,
   344вҶ’          trainId: train.id,
   345вҶ’          startDate: formatISO(startDate),
   346вҶ’          endDate: formatISO(endDate),
   347вҶ’          type: types[Math.floor(Math.random() * types.length)],
   348вҶ’          hasConflict: Math.random() < 0.05,  // 5% conflict rate
   349вҶ’        });
   350вҶ’      }
   351вҶ’    }
   352вҶ’  });
   353вҶ’
   354вҶ’  return tasks;
   355вҶ’}
   356вҶ’```

   437вҶ’## Business Rules
   438вҶ’
   439вҶ’### Mileage Management
   440вҶ’
   441вҶ’1. **Overrun Risk**: Train within 5% of maintenance interval вҶ’ Yellow warning
   442вҶ’2. **Overrun**: Train exceeds interval вҶ’ Red alert, must schedule immediately
   443вҶ’3. **Underrun**: Train significantly below target mileage вҶ’ Efficiency warning
   444вҶ’
   445вҶ’### Scheduling Constraints
   446вҶ’
   447вҶ’1. **Track Availability**: Max 2 trains per track per day
   448вҶ’2. **Man-Hours**: Cannot exceed weekly depot capacity
   449вҶ’3. **Dependencies**: Child tasks cannot start before parent completes
   450вҶ’4. **Tolerance Window**: Tasks must be scheduled within Вұtolerance of ideal date
   451вҶ’
   452вҶ’### Conflict Resolution Priority
   453вҶ’
   454вҶ’1. Safety-critical (D-Overhaul) вҶ’ Highest priority
   455вҶ’2. Regulatory compliance (C-Check) вҶ’ High priority
   456вҶ’3. Preventive maintenance (A/B-Check) вҶ’ Normal priority
   457вҶ’4. Optimization (equalizing) вҶ’ Low priority
