     1→# Prompts para Iniciar Sessões com AI
     2→
     3→
     4→## Sessões Seguintes (Desenvolvimento)
     5→
     6→```
     7→Leia claude-progress.txt para ver o estado atual.
     8→Continue de onde parou seguindo features.json.
     9→Commit a cada feature concluída.
    10→Atualize claude-progress.txt ao final.
    11→Pare.
    12→```
    13→
    14→## Primeira Sessão (Setup do Projeto)
    15→
    16→```
    17→Leia CLAUDE.md e claude-progress.txt para entender o projeto.
    18→Execute o setup inicial seguindo features.json (setup-001 a setup-008).
    19→Commit a cada etapa concluída.
    20→Pare.
    21→```
    22→
    23→## Sessão de Correção (Bugs/Problemas)
    24→
    25→```
    26→Leia claude-progress.txt.
    27→Execute pnpm test e pnpm lint para ver o estado.
    28→Corrija os problemas encontrados.
    29→Commit as correções.
    30→Pare.
    31→```
    32→
    33→## Sessão de Review (Verificação)
    34→
    35→```
    36→Leia claude-progress.txt.
    37→Execute pnpm test:e2e e pnpm test:visual.
    38→Verifique se a UI está conforme docs/UX_FOR_DEMO.md.
    39→Liste problemas encontrados.
    40→Pare.
    41→```
    42→
    43→## Sessão de Sincronização (Código ↔ Documentação)
    44→
    45→```
    46→Leia CLAUDE.md e claude-progress.txt.
    47→Compare o estado atual do código com o documentado.
    48→Se houver divergências, atualize CLAUDE.md e/ou claude-progress.txt.
    49→Commit as atualizações de documentação.
    50→Pare.
    51→```
    52→
    53→---
    54→
    55→## Dicas
    56→
    57→- **Seja específico**: "Implemente canvas-001 e canvas-002" é melhor que "continue o projeto"
    58→- **Uma feature por vez**: Evita commits grandes e facilita rollback
    59→- **Peça commits frequentes**: "Commit após cada feature"
    60→- **Peça atualização do progresso**: "Atualize claude-progress.txt ao final"
    61→
