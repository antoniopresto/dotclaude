     1→# UX Specification: Fractal Canvas Maintenance Planner
     2→
     3→## 1. Executive Vision
     4→
     5→**Objective:** Replace disjointed spreadsheets and legacy ERP interfaces with a "Single Pane of Glass" solution for Rolling Stock Maintenance.
     6→
     7→**Core Concept:** A continuous, zoomable timeline—like Google Maps but for time—that unifies Strategic Planning (12 Years), Tactical Balancing (Monthly), and Operational Execution (Daily).
     8→
     9→**Key Differentiator:** The "Fractal" nature of data—strategic decisions instantly propagate to daily operations, and operational realities ripple up to strategic KPIs.
    10→

    13→## 2. Core UX Principles
    14→
    15→### 2.1 Interface Philosophy
    16→
    17→- **Single Pane of Glass:** No navigation tabs or page reloads. Context changes solely via **Panning (Time)** and **Zooming (Granularity)**.
    18→
    19→- **Semantic Zoom:** Data representation morphs based on zoom level:
    20→  - *Zoomed out (12 years):* Heatmaps showing capacity saturation
    21→  - *Mid zoom (months):* Task blocks showing resource loading
    22→  - *Zoomed in (days):* Detailed cards showing individual trains & dependencies
    23→
    24→- **Direct Manipulation:** Users drag and drop visual elements to reschedule (future feature).
    25→
    26→### 2.2 The Google Earth Metaphor
    27→
    28→Just as Google Earth allows seamless navigation from planet view to street level:
    29→
    30→| Google Earth | MMS System |
    31→|--------------|------------|
    32→| Planet view | 12-year strategic heatmap |
    33→| Country view | Annual/quarterly workload |
    34→| City view | Monthly task distribution |
    35→| Street view | Daily train schedules |

   141→│            │  TIMELINE HEADER (adaptive granularity)        │
   142→│  SIDEBAR   ├────────────────────────────────────────────────┤
   143→│ (Blueprint │                                                 │
   144→│  Card)     │              R3F CANVAS                        │
   145→│            │                                                │
   146→│  Train     │    [MacroLayer | MesoLayer | MicroLayer]       │
   147→│  IDs       │                                                │
   148→│  (Synced   │                          [+]  ← Zoom Controls  │
   149→│   with     │                          [-]  (ButtonGroup)    │
   150→│   panY)    │  [LOD Legend]                                  │
   151→└────────────┴────────────────────────────────────────────────┘
   152→```

   154→### 6.2 Component Responsibilities
   155→
   156→| Component | Location | BlueprintJS | Function |
   157→|-----------|----------|-------------|----------|
   158→| Navbar | Top | `<Navbar>` | Title, zoom indicator, LOD badge, reset button |
   159→| Sidebar | Left | `<Card>` | Train IDs, syncs with panY |
   160→| Timeline Header | Top of canvas | Custom | Date labels with adaptive granularity |
   161→| Canvas | Center | R3F `<Canvas>` | WebGL rendering of LOD layers |
   162→| Zoom Controls | Bottom-right | `<ButtonGroup>` | +/- buttons for zoom |
   163→| LOD Indicator | Navbar | `<Tag>` | Shows MACRO/MESO/MICRO with Intent color |
   164→| Instructions | Top-center | `<Toaster>` | Auto-hiding onboarding message |

   168→## 7. Synchronization Requirements
   169→
   170→### 7.1 Virtual Scroll Sync
   171→
   172→The sidebar scroll MUST be mathematically locked to the canvas camera:
   173→
   174→```typescript
   175→// Constants
   176→const PX_PER_TRAIN = 24; // pixels per train row
   177→
   178→// Sidebar scroll → Camera Y
   179→const cameraY = -sidebarScrollTop / PX_PER_TRAIN;
   180→
   181→// Camera Y → Sidebar scroll
   182→const sidebarScrollTop = -cameraY * PX_PER_TRAIN;
   183→```
   184→
   185→### 7.2 Time Scrubber Sync
   186→
   187→The footer minimap controls the camera X position:
   188→
   189→```typescript
   190→// Scrubber position → Camera X
   191→const cameraX = scrubberValue * totalTimeRange;
   192→
   193→// Camera X → Scrubber position
   194→const scrubberValue = cameraX / totalTimeRange;
   195→```
