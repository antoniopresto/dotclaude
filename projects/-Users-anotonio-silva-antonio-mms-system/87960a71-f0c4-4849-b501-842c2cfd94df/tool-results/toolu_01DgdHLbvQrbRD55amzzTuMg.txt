    11→## 2. Architecture: "Sandwich" Layers
    12→
    13→```
    14→┌─────────────────────────────────────────────────────────────┐
    15→│ LAYER 3: HUD (HTML/React)                                   │
    16→│ - Header, Tooltips, Sidebar, Axes                          │
    17→│ - pointer-events: none (except buttons)                    │
    18→│ - Crisp text, accessibility                                │
    19→├─────────────────────────────────────────────────────────────┤
    20→│ LAYER 2: CANVAS (WebGL/PixiJS)                             │
    21→│ - Heatmap, DNA Strip, Gantt blocks                         │
    22→│ - 90% of visual content                                    │
    23→│ - 60fps with 50k+ objects                                  │
    24→├─────────────────────────────────────────────────────────────┤
    25→│ LAYER 1: ENGINE (Logic/Workers)                            │
    26→│ - Zustand stores, D3 scales                                │
    27→│ - Constraint solver, spatial index                         │
    28→│ - Runs in Web Workers (no UI blocking)                     │
    29→└─────────────────────────────────────────────────────────────┘
    30→```
    31→
    32→### 2.1 Layer Responsibilities
    33→
    34→| Layer | Technology | What It Renders |
    35→|-------|------------|-----------------|
    36→| **HUD** | React/MUI | Header, Tooltips, Axes (text), Sidebar, Modals |
    37→| **Canvas** | PixiJS v8 | Heatmap cells, DNA stripes, Task blocks, Arrows |
    38→| **Engine** | Zustand/D3 | State, Time↔Pixel conversion, Collision detection |

    94→## 6. Technical Constraints
    95→
    96→| Aspect | Use | Avoid |
    97→|--------|-----|-------|
    98→| **Navigation** | pixi-viewport (pan/zoom/inertia) | Native scroll |
    99→| **Rendering** | PixiJS batch rendering | DOM for >100 items |
   100→| **Text on Canvas** | BitmapFont/MSDF | Canvas fillText (blurry) |
   101→| **State** | Zustand transient updates | React state for animations |
   102→| **Spatial queries** | RBush R-Tree | Linear search |
   103→| **Time scales** | D3-scale/D3-time | Manual calculation |

   136→┌─────────────────────────────────────────────────────────────┐
   137→│  1. CODE      │ Implement the feature                       │
   138→├───────────────┼─────────────────────────────────────────────┤
   139→│  2. TYPECHECK │ pnpm typecheck                              │
   140→├───────────────┼─────────────────────────────────────────────┤
   141→│  3. LINT      │ pnpm lint                                   │
   142→├───────────────┼─────────────────────────────────────────────┤
   143→│  4. TEST      │ pnpm test --run                             │
   144→├───────────────┼─────────────────────────────────────────────┤
   145→│  5. VISUAL ★  │ Playwright MCP QA (SEE section 9.1)        │
   146→├───────────────┼─────────────────────────────────────────────┤
   147→│  6. COMMIT    │ git commit (only after visual confirmation) │
   148→├───────────────┼─────────────────────────────────────────────┤
   149→│  7. UPDATE    │ Mark task as "done" in features.json        │
   150→└───────────────┴─────────────────────────────────────────────┘
   151→```
   152→
   153→### 9.1 Visual QA with Playwright CLI - MANDATORY
   154→
   155→**CRITICAL:** The AI MUST visually verify renders. Never trust "tests passed" alone.
   156→
   157→#### 9.1.1 Storybook URLs
   158→
   159→**Always use the IFRAME URL for visual verification:**
   160→```
   161→http://localhost:6006/iframe.html?id=[story-id]&viewMode=story
   162→```
   163→
   164→Story ID format: `[folder]-[component]--[story-name]` (lowercase, dashes)
   165→- Example: `render-pixicanvas--default`
   166→- Example: `render-pixicanvas--with-debug`
   167→- Example: `components-hud-header--default`
   168→
   169→**DO NOT use the main Storybook URL** (`/?path=/story/...`) - it has nested iframes that complicate verification.
   170→
   171→#### 9.1.2 Playwright CLI Commands
   172→
   173→Use `npx playwright` CLI to capture screenshots and verify visually:
   174→
   175→```bash
   176→# Take screenshot (saves to ./qa-screenshots/)
   177→npx playwright screenshot "<URL>" ./qa-screenshots/<name>.png --wait-for-timeout=3000
   178→
   179→# Open browser for interactive testing
   180→npx playwright open "<URL>"
   181→```
   182→
   183→**Screenshot location:** All QA screenshots MUST be saved to `./qa-screenshots/` directory (NOT /tmp/).
   184→
   185→**Naming convention:** `<story-id>-<timestamp>.png` or `<feature>-<step>.png`
   186→
   187→#### 9.1.3 Visual Verification Checklist
   188→
   189→**The AI MUST check these on EVERY screenshot using the Read tool:**
   190→
   191→| Check | Pass | Fail |
   192→|-------|------|------|
   193→| Content renders | Visible UI elements | Blank/white screen |
   194→| No errors | Clean render | Error messages visible |
   195→| Colors correct | Match design tokens | Wrong colors |
   196→| Layout correct | Match mockup | Misaligned elements |
   197→| Interactions work | Responds to input | Frozen/unresponsive |
   198→
   199→**Common failure patterns to detect:**
   200→- ❌ White/blank screen = JavaScript error
   201→- ❌ Blue screen = CSS/styling missing
   202→- ❌ Spinner forever = Component stuck loading
   203→- ❌ Error boundary = React crash
   204→- ❌ Console errors = Runtime issues
   205→
   206→#### 9.1.4 QA Workflow Example
   207→
   208→```bash
   209→# 1. Start Storybook (if not running)
   210→pnpm storybook
   211→
   212→# 2. Take screenshot with Playwright CLI
   213→npx playwright screenshot "http://localhost:6006/iframe.html?id=render-pixicanvas--default&viewMode=story" ./qa-screenshots/pixicanvas-default.png --wait-for-timeout=3000
   214→
   215→# 3. Read and VISUALLY ANALYZE the screenshot
   216→# Use the Read tool on ./qa-screenshots/pixicanvas-default.png
   217→# AI must describe what it sees:
   218→# - "I see a dark canvas with colored rectangles..."
   219→# - "I see a white screen - ERROR, component not rendering"
   220→
   221→# 4. For interaction testing, use Playwright open:
   222→npx playwright open "http://localhost:6006/iframe.html?id=render-pixicanvas--default&viewMode=story"
   223→# Then take new screenshot after interactions
   224→
   225→# 5. Only proceed if ALL checks pass
   226→```
