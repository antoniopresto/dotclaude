     7→## 1. Industry Leader Patterns
     8→
     9→### 1.1 Figma
    10→
    11→| Gesture | Action | Notes |
    12→|---------|--------|-------|
    13→| Scroll wheel | Pan vertically | Default behavior (browser compatibility) |
    14→| Ctrl/Cmd + Scroll | Zoom in/out | Standard modifier pattern |
    15→| Middle-click drag | Pan in any direction | Alternative pan gesture |
    16→| Two-finger trackpad | Pan (both axes) | Natural for laptop users |
    17→| Pinch trackpad | Zoom | Native gesture support |
    18→| Spacebar + drag | Pan | Classic design tool pattern |
    19→| Z key + click | Zoom to point | Precision zoom gesture |
    20→
    21→**Key Insight:** Figma made a deliberate compromise - scroll wheel pans vertically by default (browser-native behavior) rather than zooming. This prevents scroll hijacking on pages with embedded Figma content.
    22→
    23→**Customization:** Users can enable "Scroll wheel zoom" in Preferences, making scroll = zoom (without Ctrl).
    24→
    25→### 1.2 Google Maps
    26→
    27→| Gesture | Action | Notes |
    28→|---------|--------|-------|
    29→| Drag | Pan with momentum | Velocity-based inertia |
    30→| Scroll wheel | Zoom | Centers on cursor |
    31→| Double-click | Zoom in 2x | At click location |
    32→| Pinch | Zoom | Touch devices |
    33→| Two-finger drag | Pan | Touch devices |
    34→
    35→**Gesture Handling Modes:**
    36→- `cooperative`: Scroll pans page; Ctrl+scroll zooms map
    37→- `greedy`: Scroll always zooms map
    38→- `none`: Disable interactions
    39→
    40→**Camera Easing:** Google uses `Quadratic.Out` easing with tween.js for smooth camera transitions. Typical duration: 300-500ms for zoom, 15s for animated tours.
    41→
    42→**Momentum Physics:**
    43→- Pan has momentum/inertia after release
    44→- Zoom does NOT have momentum (intentional UX decision)
    45→- Friction coefficient creates natural deceleration
    46→
    47→### 1.3 Apple Maps / iOS
    48→
    49→| Gesture | Action | Notes |
    50→|---------|--------|-------|
    51→| One-finger drag | Pan with strong momentum | "Buttery" feel |
    52→| Pinch | Zoom with momentum | Unlike Google Maps |
    53→| Double-tap | Zoom in | Fixed 2x increment |
    54→| Two-finger tap | Zoom out | Fixed 0.5x increment |
    55→| Rotate gesture | Rotate map | Optional feature |
    56→
    57→**Key Differentiator:** iOS implements momentum for BOTH pan AND zoom gestures, creating a more fluid feel than Google Maps.
    58→
    59→### 1.4 Canva
    60→
    61→| Gesture | Action | Notes |
    62→|---------|--------|-------|
    63→| Drag | Pan canvas | Infinite canvas |
    64→| Scroll wheel | Zoom | Centers on cursor |
    65→| Pinch | Zoom | Touch devices |
    66→| Pan gesture | Pan | Two-finger drag |
    67→
    68→**Infinite Canvas:** Canva's whiteboard feature supports unlimited pan in all directions with zoom/pan/scroll navigation.

    72→## 2. Mouse Events Reference
    73→
    74→### 2.1 Event Types
    75→
    76→| Event | Triggers | Properties |
    77→|-------|----------|------------|
    78→| `mousedown` | Button pressed | `button`, `buttons`, `clientX/Y` |
    79→| `mouseup` | Button released | `button`, `buttons`, `clientX/Y` |
    80→| `mousemove` | Cursor moved | `clientX/Y`, `movementX/Y` |
    81→| `click` | Press + release (same element) | Fires after `mouseup` |
    82→| `dblclick` | Two clicks within threshold | ~300ms default |
    83→| `contextmenu` | Right-click | Prevent for custom menu |
    84→| `wheel` | Scroll wheel rotated | `deltaX/Y/Z`, `deltaMode` |
    85→| `mouseenter` | Cursor enters element | Does NOT bubble |
    86→| `mouseleave` | Cursor leaves element | Does NOT bubble |
    87→| `mouseover` | Cursor over element | Bubbles (includes children) |
    88→| `mouseout` | Cursor leaves element | Bubbles (includes children) |
    89→
    90→### 2.2 Button Values
    91→
    92→| `event.button` | Description |
    93→|----------------|-------------|
    94→| 0 | Left button (primary) |
    95→| 1 | Middle button (wheel click) |
    96→| 2 | Right button (secondary) |
    97→| 3 | Back button (browser) |
    98→| 4 | Forward button (browser) |
    99→
   100→### 2.3 Drag Implementation Pattern
   101→
   102→```typescript
   103→interface DragState {
   104→  isDragging: boolean;
   105→  startX: number;
   106→  startY: number;
   107→  lastX: number;
   108→  lastY: number;
   109→  velocityX: number;
   110→  velocityY: number;
   111→  lastTime: number;
   112→}
   113→
   114→// Calculate velocity during drag for momentum
   115→function onMouseMove(e: MouseEvent, state: DragState) {
   116→  const now = performance.now();
   117→  const dt = now - state.lastTime;
   118→
   119→  if (dt > 0) {
   120→    state.velocityX = (e.clientX - state.lastX) / dt;
   121→    state.velocityY = (e.clientY - state.lastY) / dt;
   122→  }
   123→
   124→  state.lastX = e.clientX;
   125→  state.lastY = e.clientY;
   126→  state.lastTime = now;
   127→}
   128→```

   130→### 2.4 Double-Click Detection
   131→
   132→```typescript
   133→const DOUBLE_CLICK_THRESHOLD = 300; // ms
   134→let lastClickTime = 0;
   135→
   136→function onClick(e: MouseEvent) {
   137→  const now = Date.now();
   138→  if (now - lastClickTime < DOUBLE_CLICK_THRESHOLD) {

   140→  }

   147→## 3. Touch Events Reference
   148→
   149→### 3.1 Touch Event Types
   150→
   151→| Event | Triggers | Notes |
   152→|-------|----------|-------|
   153→| `touchstart` | Finger touches screen | One or more touches |
   154→| `touchmove` | Finger moves on screen | Fires frequently |
   155→| `touchend` | Finger lifts from screen | |
   156→| `touchcancel` | Touch interrupted | System event, incoming call |
   157→
   158→### 3.2 Pointer Events (Modern, Unified)
   159→
   160→Pointer Events unify mouse, touch, and pen input:
   161→
   162→| Event | Equivalent Touch/Mouse |
   163→|-------|------------------------|
   164→| `pointerdown` | `touchstart` / `mousedown` |
   165→| `pointermove` | `touchmove` / `mousemove` |
   166→| `pointerup` | `touchend` / `mouseup` |
   167→| `pointercancel` | `touchcancel` |
   168→| `pointerenter` | `mouseenter` |
   169→| `pointerleave` | `mouseleave` |
   170→
   171→**Key Properties:**
   172→- `pointerId`: Unique identifier for each contact
   173→- `pointerType`: "mouse", "touch", "pen"
   174→- `isPrimary`: True for first finger/primary button
   175→- `pressure`: 0.0 to 1.0 (pen pressure)
   176→- `width`, `height`: Contact area size
   177→
   178→### 3.3 Pinch-to-Zoom Algorithm
   179→
   180→```typescript
   181→// Cache active pointers
   182→const evCache: PointerEvent[] = [];
   183→let prevDist = -1;
   184→
   185→function onPointerDown(ev: PointerEvent) {
   186→  evCache.push(ev);
   187→}
   188→
   189→function onPointerMove(ev: PointerEvent) {
   190→  // Update cached pointer
   191→  const idx = evCache.findIndex(e => e.pointerId === ev.pointerId);
   192→  if (idx >= 0) evCache[idx] = ev;
   193→
   194→  // Pinch gesture requires exactly 2 pointers
   195→  if (evCache.length === 2) {
   196→    const dx = evCache[0].clientX - evCache[1].clientX;
   197→    const dy = evCache[0].clientY - evCache[1].clientY;
   198→    const curDist = Math.sqrt(dx * dx + dy * dy);
   199→
   200→    if (prevDist > 0) {
   201→      const scale = curDist / prevDist;
   202→      // scale > 1 = zoom in, scale < 1 = zoom out
   203→      handlePinchZoom(scale, getMidpoint(evCache[0], evCache[1]));
   204→    }
   205→
   206→    prevDist = curDist;
   207→  }
   208→}
   209→
   210→function onPointerUp(ev: PointerEvent) {
   211→  const idx = evCache.findIndex(e => e.pointerId === ev.pointerId);
   212→  if (idx >= 0) evCache.splice(idx, 1);
   213→  if (evCache.length < 2) prevDist = -1;
   214→}
   215→
   216→function getMidpoint(a: PointerEvent, b: PointerEvent) {
   217→  return {
   218→    x: (a.clientX + b.clientX) / 2,
   219→    y: (a.clientY + b.clientY) / 2
   220→  };
   221→}
   222→```

   237→## 4. Wheel Event Normalization
   238→
   239→### 4.1 The Problem
   240→
   241→Different browsers/platforms report wildly different `deltaY` values:
   242→
   243→| Environment | deltaY per "tick" |
   244→|-------------|-------------------|
   245→| Chrome (Windows) | 100-120 |
   246→| Chrome (macOS trackpad) | 1-5 (many events) |
   247→| Firefox (Windows) | 3 (lines mode) |
   248→| Firefox (macOS) | 1-2 |
   249→| Safari | 1-12 |
   250→
   251→### 4.2 deltaMode Values
   252→
   253→```typescript
   254→enum WheelDeltaMode {
   255→  DOM_DELTA_PIXEL = 0,  // deltaY is in pixels
   256→  DOM_DELTA_LINE = 1,   // deltaY is in lines
   257→  DOM_DELTA_PAGE = 2    // deltaY is in pages
   258→}
   259→```
   260→
   261→### 4.3 Facebook's Normalization Algorithm
   262→
   263→```typescript
   264→// Constants for normalization
   265→const PIXEL_STEP = 10;
   266→const LINE_HEIGHT = 40;
   267→const PAGE_HEIGHT = 800;
   268→
   269→interface NormalizedWheel {
   270→  spinX: number;   // Normalized spin (-1 to 1)
   271→  spinY: number;   // Normalized spin (-1 to 1)
   272→  pixelX: number;  // Normalized pixels
   273→  pixelY: number;  // Normalized pixels
   274→}
   275→
   276→function normalizeWheel(event: WheelEvent): NormalizedWheel {
   277→  let sX = 0, sY = 0;   // spin
   278→  let pX = 0, pY = 0;   // pixel
   279→
   280→  // Modern wheel event
   281→  if ('deltaY' in event) {
   282→    pX = event.deltaX;
   283→    pY = event.deltaY;
   284→
   285→    // Convert based on deltaMode
   286→    if (event.deltaMode === 1) {        // LINE mode
   287→      pX *= LINE_HEIGHT;
   288→      pY *= LINE_HEIGHT;
   289→    } else if (event.deltaMode === 2) { // PAGE mode
   290→      pX *= PAGE_HEIGHT;
   291→      pY *= PAGE_HEIGHT;
   292→    }
   293→  }
   294→
   295→  // Legacy fallback
   296→  if ('wheelDelta' in event) {
   297→    const wd = (event as any).wheelDelta;
   298→    if ('wheelDeltaY' in event) {
   299→      sY = -(event as any).wheelDeltaY / 120;
   300→      sX = -(event as any).wheelDeltaX / 120;
   301→    } else {
   302→      sY = -wd / 120;
   303→    }
   304→  }
   305→
   306→  // Calculate spin from pixel if not set
   307→  if (!sX && pX) sX = pX < 0 ? -1 : pX > 0 ? 1 : 0;
   308→  if (!sY && pY) sY = pY < 0 ? -1 : pY > 0 ? 1 : 0;
   309→
   310→  return { spinX: sX, spinY: sY, pixelX: pX, pixelY: pY };
   311→}
   312→```
   313→
   314→### 4.4 Trackpad vs Mouse Detection
   315→
   316→Trackpads produce many small events rapidly; mice produce fewer large events:
   317→
   318→```typescript
   319→let lastWheelTime = 0;
   320→let wheelEventCount = 0;
   321→
   322→function detectInputType(event: WheelEvent): 'trackpad' | 'mouse' {
   323→  const now = performance.now();
   324→  const timeDelta = now - lastWheelTime;
   325→
   326→  if (timeDelta < 50) {
   327→    wheelEventCount++;
   328→  } else {
   329→    wheelEventCount = 1;
   330→  }
   331→
   332→  lastWheelTime = now;
   333→
   334→  // Trackpads fire many events rapidly (<20ms apart)
   335→  // Mice fire discrete events with gaps
   336→  if (wheelEventCount > 3 && timeDelta < 20) {
   337→    return 'trackpad';
   338→  }
   339→
   340→  // Also check delta magnitude
   341→  if (Math.abs(event.deltaY) < 10 && event.deltaMode === 0) {
   342→    return 'trackpad'; // Small pixel deltas = trackpad
   343→  }
   344→
   345→  return 'mouse';
   346→}
   347→```

   351→## 5. Momentum/Inertia Physics
   352→
   353→### 5.1 Basic Momentum Implementation
   354→
   355→```typescript
   356→interface MomentumState {
   357→  velocityX: number;
   358→  velocityY: number;
   359→  friction: number;
   360→  minVelocity: number;
   361→  animationId: number | null;
   362→}
   363→
   364→const momentum: MomentumState = {
   365→  velocityX: 0,
   366→  velocityY: 0,
   367→  friction: 0.95,      // Per-frame velocity multiplier
   368→  minVelocity: 0.1,    // Stop threshold
   369→  animationId: null
   370→};
   371→
   372→function startMomentum(vx: number, vy: number) {
   373→  momentum.velocityX = vx;
   374→  momentum.velocityY = vy;
   375→
   376→  if (momentum.animationId) {
   377→    cancelAnimationFrame(momentum.animationId);
   378→  }
   379→
   380→  momentum.animationId = requestAnimationFrame(animateMomentum);
   381→}
   382→
   383→function animateMomentum() {
   384→  // Apply velocity to pan position
   385→  panX += momentum.velocityX;
   386→  panY += momentum.velocityY;
   387→
   388→  // Apply friction
   389→  momentum.velocityX *= momentum.friction;
   390→  momentum.velocityY *= momentum.friction;
   391→
   392→  // Stop when velocity is negligible
   393→  const speed = Math.sqrt(
   394→    momentum.velocityX ** 2 + momentum.velocityY ** 2
   395→  );
   396→
   397→  if (speed > momentum.minVelocity) {
   398→    momentum.animationId = requestAnimationFrame(animateMomentum);
   399→  } else {
   400→    momentum.animationId = null;
   401→  }
   402→}
   403→
   404→function stopMomentum() {
   405→  if (momentum.animationId) {
   406→    cancelAnimationFrame(momentum.animationId);
   407→    momentum.animationId = null;
   408→  }
   409→  momentum.velocityX = 0;
   410→  momentum.velocityY = 0;
   411→}
   412→```
   413→
   414→### 5.2 Easing Functions
   415→
   416→```typescript
   417→// Common easing functions for zoom animations
   418→const Easing = {
   419→  // Quadratic ease out (Google Maps style)
   420→  quadOut: (t: number) => t * (2 - t),
   421→
   422→  // Cubic ease out (smoother)
   423→  cubicOut: (t: number) => 1 - Math.pow(1 - t, 3),
   424→
   425→  // Expo ease out (fast start, slow end)
   426→  expoOut: (t: number) => t === 1 ? 1 : 1 - Math.pow(2, -10 * t),
   427→
   428→  // Circ ease out (natural feel)
   429→  circOut: (t: number) => Math.sqrt(1 - Math.pow(t - 1, 2))
   430→};
   431→
   432→function animateZoom(
   433→  fromZoom: number,
   434→  toZoom: number,
   435→  duration: number,
   436→  easing = Easing.quadOut
   437→) {
   438→  const startTime = performance.now();
   439→
   440→  function tick(currentTime: number) {
   441→    const elapsed = currentTime - startTime;
   442→    const progress = Math.min(elapsed / duration, 1);
   443→    const easedProgress = easing(progress);
   444→
   445→    const currentZoom = fromZoom + (toZoom - fromZoom) * easedProgress;
   446→    setZoom(currentZoom);
   447→
   448→    if (progress < 1) {
   449→      requestAnimationFrame(tick);
   450→    }
   451→  }
   452→
   453→  requestAnimationFrame(tick);
   454→}
   455→```
   456→
   457→### 5.3 Recommended Durations
   458→
   459→| Animation | Duration | Easing |
   460→|-----------|----------|--------|
   461→| Zoom step (+/-) | 200-300ms | quadOut |
   462→| Double-click zoom | 300-400ms | cubicOut |
   463→| Reset view | 400-500ms | expoOut |
   464→| Pan deceleration | Physics-based | Friction 0.92-0.95 |

   468→## 6. Cross-Platform Considerations
   469→
   470→### 6.1 macOS vs Windows Scroll Direction
   471→
   472→| Platform | Default Scroll Direction |
   473→|----------|--------------------------|
   474→| macOS (trackpad) | "Natural" (content follows fingers) |
   475→| macOS (mouse) | Same as trackpad (unless changed) |
   476→| Windows | "Traditional" (opposite of macOS) |
   477→| Linux | Varies by desktop environment |
   478→
   479→**Recommendation:** Use the raw `deltaY` value directly. Browser vendors have already normalized for user preferences. Do NOT attempt to detect or reverse scroll direction.
   480→
   481→### 6.2 Platform-Specific deltaY Magnitude
   482→
   483→| Platform | Typical deltaY (mouse) | Typical deltaY (trackpad) |
   484→|----------|------------------------|---------------------------|
   485→| Windows Chrome | 100 | 1-10 (many events) |
   486→| macOS Chrome | 100 | 0.5-3 (many events) |
   487→| Windows Firefox | 3 (LINE mode) | 1-2 |
   488→| macOS Safari | 12-30 | 0.5-2 |
   489→
   490→### 6.3 Recommended Normalization Strategy
   491→
   492→```typescript
   493→const ZOOM_SENSITIVITY = 0.001;  // Base sensitivity
   494→const MAX_ZOOM_DELTA = 0.5;      // Cap per event
   495→
   496→function calculateZoomDelta(event: WheelEvent): number {
   497→  let delta = event.deltaY;
   498→
   499→  // Normalize LINE mode to pixels
   500→  if (event.deltaMode === 1) {
   501→    delta *= 16; // Approximate line height
   502→  }
   503→
   504→  // Apply sensitivity and clamp
   505→  delta *= ZOOM_SENSITIVITY;
   506→  delta = Math.max(-MAX_ZOOM_DELTA, Math.min(MAX_ZOOM_DELTA, delta));
   507→
   508→  return delta;
   509→}
   510→```

   566→## 8. Implementation Recommendations for MMS
   567→
   568→### 8.1 F118: Momentum Panning
   569→
   570→1. Track velocity during drag (store last 3-5 positions with timestamps)
   571→2. On mouse up, calculate average velocity
   572→3. Apply friction (0.93-0.95 per frame) in animation loop
   573→4. Stop when velocity < 0.1 px/frame
   574→5. Cancel momentum on any new interaction
   575→
   576→### 8.2 F119: Smooth Zoom
   577→
   578→1. Use `requestAnimationFrame` for zoom animations
   579→2. Apply `Easing.quadOut` over 250ms
   580→3. For pinch-zoom, apply zoom changes immediately (no animation)
   581→4. Clamp zoom rate to prevent jumps
   582→
   583→### 8.3 F120: Double-Click Zoom
   584→
   585→1. Detect double-click with 300ms threshold
   586→2. Convert click to world coordinates
   587→3. Animate zoom 2x towards click point
   588→4. Use 350ms duration with `cubicOut` easing
   589→5. Respect MAX_ZOOM limit
   590→
   591→### 8.4 F121: Cross-Platform Scroll Normalization
   592→
   593→1. Use Facebook's normalization pattern (simplified)
   594→2. Detect trackpad vs mouse by event frequency
   595→3. Apply different sensitivity multipliers:
   596→   - Mouse: `ZOOM_SENSITIVITY * 1.0`
   597→   - Trackpad: `ZOOM_SENSITIVITY * 0.3`
   598→4. Clamp zoom delta per event to ±0.5
   599→5. Test on macOS, Windows, Linux (if available)
