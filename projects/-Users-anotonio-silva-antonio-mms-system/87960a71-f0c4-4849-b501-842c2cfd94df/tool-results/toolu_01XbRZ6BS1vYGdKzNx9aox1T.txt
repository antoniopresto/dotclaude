    71→**Commit early, commit often.** Do NOT wait for a complete feature.
    72→
    73→### Rules
    74→
    75→1. **Commit after each logical change** (1-3 files max per commit)
    76→2. **Never batch multiple unrelated changes** into one commit
    77→3. **Commit before switching context** to another task
    78→4. **Atomic commits** - each commit should be independently meaningful
    79→
    80→### Commit Frequency Guidelines
    81→
    82→| Change Type               | Commit When                |
    83→|---------------------------|----------------------------|
    84→| Config file update        | Immediately after change   |
    85→| Documentation update      | Immediately after change   |
    86→| Single component/function | After it compiles/works    |
    87→| Bug fix                   | After fix is verified      |
    88→| Refactor                  | After each file refactored |

   260→1. **Navigate** to page:
   261→```
   262→mcp__playwright__browser_navigate url="http://localhost:3000"
   263→```
   264→
   265→2. **Check console** (JS errors):
   266→```
   267→mcp__playwright__browser_console_messages level="error"
   268→```
   269→
   270→3. **Capture screenshot**:
   271→```
   272→mcp__playwright__browser_take_screenshot filename="test-name.png"
   273→```
   274→
   275→4. **Analyze visually**:
   276→- [ ] Content visible?
   277→- [ ] Layout correct?
   278→- [ ] No errors in console?
   279→
   280→5. **Close browser**:
   281→```
   282→mcp__playwright__browser_close
   283→```
