    57‚Üí## 2. Core Concepts
    58‚Üí
    59‚Üí### The Ralph Loop
    60‚Üí
    61‚ÜíRalph is an infinite loop that feeds prompts to Claude Code, with progress persisting in files and git, not in context.

    84‚Üí### The 5 Golden Rules
    85‚Üí
    86‚Üí1. **"Disk is state, git is memory"** ‚Äî Progress lives in files and commits, not context
    87‚Üí2. **"Plans are disposable"** ‚Äî Regenerate rather than accumulate
    88‚Üí3. **"Fresh context is reliability"** ‚Äî Each iteration starts clean
    89‚Üí4. **"Ralph maintains Ralph"** ‚Äî Use subagents for self-cleanup
    90‚Üí5. **"Tune like a guitar"** ‚Äî Add signage for observed failures
    91‚Üí
    92‚Üí### Key Files
    93‚Üí
    94‚Üí| File | Purpose | Max Size |
    95‚Üí|------|---------|----------|
    96‚Üí| `IMPLEMENTATION_PLAN.md` | Active tasks, blockers, progress | <400 lines |
    97‚Üí| `AGENTS.md` | Operational commands, quick reference | <60 lines |
    98‚Üí| `PROMPT_build.md` | Build mode instructions | ~200 lines |
    99‚Üí| `PROMPT_plan.md` | Planning mode instructions | ~100 lines |
   100‚Üí| `@fix_plan.md` | Immediate task override (optional) | Any |

   104‚Üí## 3. Main Loop (loop.sh)
   105‚Üí
   106‚Üí### Synopsis
   107‚Üí```bash
   108‚Üí./ralph/loop.sh [MODE] [ITERATIONS] [OPTIONS]
   109‚Üí```
   110‚Üí
   111‚Üí### Options
   112‚Üí
   113‚Üí| Option | Description | Default |
   114‚Üí|--------|-------------|---------|
   115‚Üí| `<number>` | Max iterations (0 = unlimited) | 0 |
   116‚Üí| `plan` | Planning mode | - |
   117‚Üí| `plan-work "desc"` | Scoped planning | - |
   118‚Üí| `-m, --model MODEL` | Model to use (opus/sonnet/haiku) | opus |
   119‚Üí| `-v, --verbose` | Verbose output | off |
   120‚Üí| `--no-push` | Don't push to remote | enabled |
   121‚Üí| `--no-stop-on-complete` | Continue after completion signal | off |
   122‚Üí| `-h, --help` | Show help | - |
   123‚Üí
   124‚Üí### Examples
   125‚Üí
   126‚Üí```bash
   127‚Üí# Unlimited build iterations (default)
   128‚Üí./ralph/loop.sh
   129‚Üí
   130‚Üí# Max 50 iterations
   131‚Üí./ralph/loop.sh 50
   132‚Üí
   133‚Üí# Planning mode with 3 iterations
   134‚Üí./ralph/loop.sh plan 3
   135‚Üí
   136‚Üí# Scoped planning for specific work
   137‚Üí./ralph/loop.sh plan-work "API authentication" 5
   138‚Üí
   139‚Üí# Use sonnet model with verbose output
   140‚Üí./ralph/loop.sh -m sonnet -v 20
   141‚Üí
   142‚Üí# Override NO_PUSH for this run
   143‚ÜíNO_PUSH=false ./ralph/loop.sh
   144‚Üí```
   145‚Üí
   146‚Üí### Environment Variables
   147‚Üí
   148‚Üí| Variable | Description | Default |
   149‚Üí|----------|-------------|---------|
   150‚Üí| `BASH_DEFAULT_TIMEOUT_MS` | Bash command timeout | 1800000 (30min) |
   151‚Üí| `BASH_MAX_TIMEOUT_MS` | Maximum bash timeout | 7200000 (2hr) |
   152‚Üí| `API_TIMEOUT_MS` | API request timeout | 600000 (10min) |
   153‚Üí| `MAX_CALLS_PER_HOUR` | Rate limit | 9999999 |
   154‚Üí| `MAX_RETRIES` | Retry count for transient errors | 3 |
   155‚Üí| `RETRY_DELAY` | Seconds between retries | 5 |
   156‚Üí
   157‚Üí### Completion Signals
   158‚Üí
   159‚ÜíRalph stops when detecting these signals in output:
   160‚Üí
   161‚Üí- `ALL_TASKS_COMPLETE`
   162‚Üí- `PLAN_COMPLETE`
   163‚Üí- `EXIT_SIGNAL`
   164‚Üí- `<promise>DONE</promise>`
   165‚Üí- `<promise>COMPLETE</promise>`
   166‚Üí- `<promise>ALL_TASKS_COMPLETE</promise>`
   167‚Üí- `<promise>PLAN_COMPLETE</promise>`

   171‚Üí## 4. Modes of Operation
   172‚Üí
   173‚Üí### Build Mode (Default)
   174‚Üí
   175‚ÜíImplements tasks from `IMPLEMENTATION_PLAN.md`:
   176‚Üí
   177‚Üí```bash
   178‚Üí./ralph/loop.sh         # Unlimited
   179‚Üí./ralph/loop.sh 50      # Max 50 iterations
   180‚Üí```
   181‚Üí
   182‚Üí**Workflow per iteration**:
   183‚Üí1. Check for `@fix_plan.md` (priority override)
   184‚Üí2. Read `IMPLEMENTATION_PLAN.md`
   185‚Üí3. Select highest-priority incomplete task
   186‚Üí4. Implement with validation (test ‚Üí typecheck ‚Üí lint ‚Üí build)
   187‚Üí5. Commit changes
   188‚Üí6. Update plan, loop
   189‚Üí
   190‚Üí### Planning Mode
   191‚Üí
   192‚ÜíGenerates or updates implementation plan from specs:
   193‚Üí
   194‚Üí```bash
   195‚Üí./ralph/loop.sh plan        # Unlimited planning
   196‚Üí./ralph/loop.sh plan 3      # Max 3 iterations
   197‚Üí```
   198‚Üí
   199‚Üí**Use when**:
   200‚Üí- Starting a new phase
   201‚Üí- Plan is stale or bloated
   202‚Üí- Need fresh gap analysis
   203‚Üí
   204‚Üí### Scoped Planning Mode
   205‚Üí
   206‚ÜíPlan specific work area:
   207‚Üí
   208‚Üí```bash
   209‚Üí./ralph/loop.sh plan-work "API endpoints" 5
   210‚Üí./ralph/loop.sh plan-work "database migrations" 3
   211‚Üí```
   212‚Üí
   213‚Üí**Requirements**:
   214‚Üí- Must be on a work branch (not main/master)
   215‚Üí- Sets `WORK_SCOPE` environment variable
   216‚Üí- Uses `PROMPT_plan_work.md`
   217‚Üí

   220‚Üí## 5. Mid-Course Corrections
   221‚Üí
   222‚Üí### How It Works
   223‚Üí
   224‚ÜíRalph reads files fresh each iteration. Edit files ‚Üí changes take effect next iteration.
   225‚Üí
   226‚Üí**Geoffrey Huntley's Method**:
   227‚Üí> *"Tune Ralph like a guitar. When Ralph fails a specific way, add a sign to help him next time."*
   228‚Üí
   229‚Üí### Method 1: @fix_plan.md (Highest Priority)
   230‚Üí
   231‚ÜíCreate `ralph/@fix_plan.md` for immediate tasks:
   232‚Üí
   233‚Üí```bash
   234‚Üí./ralph/scripts/request.sh "Create documentation for running the app"
   235‚Üí```
   236‚Üí
   237‚ÜíOr manually:
   238‚Üí
   239‚Üí```markdown
   240‚Üí# Immediate Task: Documentation
   241‚Üí
   242‚Üí## Objective
   243‚ÜíCreate comprehensive documentation for running MMS locally.
   244‚Üí
   245‚Üí## Deliverables
   246‚Üí1. `docs/QUICKSTART.md` with full setup instructions
   247‚Üí2. Update `ralph/AGENTS.md` quick reference
   248‚Üí
   249‚Üí## Completion Signal
   250‚Üí<promise>DOCS_COMPLETE</promise>
   251‚Üí```
   252‚Üí
   253‚Üí**Behavior**: Ralph completes `@fix_plan.md` tasks before anything else, then deletes the file.
   254‚Üí
   255‚Üí### Method 2: BLOCKER in IMPLEMENTATION_PLAN.md
   256‚Üí
   257‚ÜíAdd P1 blocker at top of plan:
   258‚Üí
   259‚Üí```markdown
   260‚Üí## BLOCKERS (Resolve First)
   261‚Üí
   262‚Üí- [ ] **P1 HUMAN REQUEST**: Create authentication tests
   263‚Üí  - Deliverable: `src/services/__tests__/AuthService.test.ts`
   264‚Üí  - Acceptance: All tests passing
   265‚Üí```
   266‚Üí
   267‚Üí### Method 3: Signpost in PROMPT_build.md
   268‚Üí
   269‚ÜíAdd temporary focus to prompt file:
   270‚Üí
   271‚Üí```bash
   272‚Üí./ralph/scripts/focus.sh "Focus ONLY on Phase 7 UI components"
   273‚Üí```
   274‚Üí
   275‚ÜíOr manually add to top of `PROMPT_build.md`:
   276‚Üí
   277‚Üí```markdown
   278‚Üí## SESSION FOCUS (Remove when complete)
   279‚Üí
   280‚ÜíThis session: Focus ONLY on Phase 7 (UI Components).
   281‚Üí- Do NOT work on backend services
   282‚Üí- Do NOT work on API endpoints
   283‚Üí- ONLY work on: React components, deck.gl layers
   284‚Üí
   285‚Üí---
   286‚Üí```
   287‚Üí
   288‚Üí### Method 4: Signage (Permanent Corrections)
   289‚Üí
   290‚ÜíWhen Ralph makes repeated mistakes, add permanent signage:
   291‚Üí
   292‚Üí```markdown
   293‚Üí## SIGNAGE
   294‚Üí- DO NOT use useState ‚Äî use machine.useMachine() instead
   295‚Üí- DO NOT assume endpoints exist ‚Äî search codebase first
   296‚Üí- ALWAYS run typecheck before committing
   297‚Üí```
   298‚Üí
   299‚Üí### Quick Reference
   300‚Üí
   301‚Üí| Need | Method | Command |
   302‚Üí|------|--------|---------|
   303‚Üí| Specific task | @fix_plan.md | `./ralph/scripts/request.sh "description"` |
   304‚Üí| Documentation | @fix_plan.md | `./ralph/scripts/request.sh --template docs "topic"` |
   305‚Üí| Specific tests | @fix_plan.md | `./ralph/scripts/request.sh --template test "component"` |

   307‚Üí| Bug fix | BLOCKER | Edit `IMPLEMENTATION_PLAN.md` |
   308‚Üí| Stop for review | Checkpoint | Add task with `<promise>REVIEW_NEEDED</promise>` |
   309‚Üí| Prevent behavior | Signage | Edit `PROMPT_build.md` |

   399‚Üí## 7. Circuit Breaker
   400‚Üí
   401‚Üí### Overview
   402‚Üí
   403‚ÜíPrevents runaway loops by monitoring:
   404‚Üí- Iterations without git commits (stuck loop)
   405‚Üí- Repeated errors
   406‚Üí- Estimated cost
   407‚Üí
   408‚Üí### Commands
   409‚Üí
   410‚Üí```bash
   411‚Üí# Show status
   412‚Üí./ralph/circuit-breaker.sh status
   413‚Üí
   414‚Üí# Check if circuit is open (exit 1 = blocked)
   415‚Üí./ralph/circuit-breaker.sh check
   416‚Üí
   417‚Üí# Reset after stuck loop
   418‚Üí./ralph/circuit-breaker.sh reset
   419‚Üí
   420‚Üí# Show recent errors
   421‚Üí./ralph/circuit-breaker.sh errors
   422‚Üí```
   423‚Üí
   424‚Üí### Configuration
   425‚Üí
   426‚Üí| Variable | Description | Default |

   642‚Üí## 11. Troubleshooting
   643‚Üí
   644‚Üí### Ralph Stops Unexpectedly
   645‚Üí
   646‚Üí**Check circuit breaker**:
   647‚Üí```bash
   648‚Üí./ralph/circuit-breaker.sh status
   649‚Üí```
   650‚Üí
   651‚Üí**Common causes**:
   652‚Üí- No progress threshold exceeded
   653‚Üí- Cost limit reached
   654‚Üí- Repeated errors
   655‚Üí
   656‚Üí**Fix**:
   657‚Üí```bash
   658‚Üí./ralph/circuit-breaker.sh reset
   659‚Üí./ralph/loop.sh
   660‚Üí```
   661‚Üí
   662‚Üí### Timeout Errors
   663‚Üí
   664‚Üí**Symptom**: Operations killed after 2 minutes
   665‚Üí
   666‚Üí**Fix**: Set timeout environment variables:
   667‚Üí```bash
   668‚Üíexport BASH_DEFAULT_TIMEOUT_MS=1800000
   669‚Üíexport BASH_MAX_TIMEOUT_MS=7200000
   670‚Üí./ralph/loop.sh
   671‚Üí```
   672‚Üí
   673‚Üí### Plan Too Large
   674‚Üí
   675‚Üí**Symptom**: Context rot, partial reads, forgotten tasks
   676‚Üí
   677‚Üí**Check**:
   678‚Üí```bash
   679‚Üíwc -l ralph/IMPLEMENTATION_PLAN.md
   680‚Üí./ralph/scripts/check-plan-size.sh
   681‚Üí```
   682‚Üí
   683‚Üí**Fix**:
   684‚Üí```bash
   685‚Üí./ralph/scripts/archive-completed.sh
   686‚Üí# Or regenerate:
   687‚Üí./ralph/scripts/refresh-plan.sh
   688‚Üí```
   689‚Üí
   690‚Üí### Git Push Failures
   691‚Üí
   692‚Üí**Symptom**: Loop stops with push error (especially on VPN)
   693‚Üí
   694‚Üí**Workaround**: Run with NO_PUSH (default):
   695‚Üí```bash
   696‚Üí./ralph/loop.sh  # NO_PUSH=true by default
   697‚Üí```
   698‚Üí
   699‚Üí**Manual push when convenient**:
   700‚Üí```bash
   701‚Üígit push origin $(git branch --show-current)
   702‚Üí```
   703‚Üí
   704‚Üí### Completion Signal Not Detected
   705‚Üí
   706‚Üí**Symptom**: Loop continues despite task completion
   707‚Üí
   708‚Üí**Check**: Signals are case-sensitive by default
   709‚Üí```bash
   710‚Üígrep -i "ALL_TASKS_COMPLETE" /tmp/ralph-outputs-*/iteration-*.json
   711‚Üí```
   712‚Üí
   713‚Üí**Fix**: Ensure exact signal format in output:
   714‚Üí```
   715‚Üí<promise>ALL_TASKS_COMPLETE</promise>
   716‚Üí```
   717‚Üí
   718‚Üí### Stuck in Loop
   719‚Üí
   720‚Üí**Symptom**: Same task attempted repeatedly
   721‚Üí
   722‚Üí**Diagnosis**:
   723‚Üí```bash
   724‚Üí./ralph/circuit-breaker.sh errors
   725‚Üígit log --oneline -10
   726‚Üí```
   727‚Üí
   728‚Üí**Fixes**:
   729‚Üí1. Add signage to prevent repeated mistake
   730‚Üí2. Manually complete the blocking task
   731‚Üí3. Reset and add explicit instructions
   732‚Üí
   733‚Üí```bash
   734‚Üí./ralph/circuit-breaker.sh reset
   735‚Üí# Add signage to PROMPT_build.md, then:
   736‚Üí./ralph/loop.sh
   737‚Üí```

   741‚Üí## 12. Best Practices
   742‚Üí
   743‚Üí### From Geoffrey Huntley
   744‚Üí
   745‚Üí1. **Never interrupt mid-iteration** ‚Äî Let it finish, then adjust
   746‚Üí2. **Tune via file edits** ‚Äî Not runtime commands
   747‚Üí3. **Plans are disposable** ‚Äî Regenerate rather than accumulate
   748‚Üí4. **Add signage for failures** ‚Äî Prevent same mistake twice
   749‚Üí5. **Trust eventual consistency** ‚Äî Ralph will get there
   750‚Üí
   751‚Üí### Plan Management
   752‚Üí
   753‚Üí- Keep `IMPLEMENTATION_PLAN.md` under 400 lines
   754‚Üí- Archive completed phases regularly
   755‚Üí- Use specific, actionable task descriptions
   756‚Üí- Include acceptance criteria
   757‚Üí
   758‚Üí### Mid-Course Corrections
   759‚Üí
   760‚Üí- **Urgent task**: Use `@fix_plan.md`
   761‚Üí- **Session focus**: Use signpost in `PROMPT_build.md`
   762‚Üí- **Permanent rule**: Add to SIGNAGE section
   763‚Üí- **Bug fix**: Add P1 BLOCKER to plan
   764‚Üí
   765‚Üí### Long Runs (24h+)
   766‚Üí
   767‚Üí- Use `NO_PUSH=true` (default) to avoid VPN issues
   768‚Üí- Monitor periodically: `./ralph/monitor.sh stats`
   769‚Üí- High circuit breaker thresholds are intentional
   770‚Üí- Trust git history for recovery
   771‚Üí
   772‚Üí### Debugging
   773‚Üí
   774‚Üí- Check circuit breaker status first
   775‚Üí- Review recent commits: `git log --oneline -20`
   776‚Üí- Check for errors: `./ralph/circuit-breaker.sh errors`
   777‚Üí- Add verbose mode: `./ralph/loop.sh -v`
   778‚Üí

   781‚Üí## 13. Intentional Behaviors
   782‚Üí
   783‚ÜíThese behaviors appear to be defects but are intentional design decisions:
   784‚Üí
   785‚Üí### NO_PUSH=true Default
   786‚Üí
   787‚Üí**Appears as**: Data loss risk (work only local)
   788‚Üí
   789‚Üí**Why intentional**: VPN interference causes git push failures, which stop the loop. Local commits still preserve work.
   790‚Üí
   791‚Üí**Override**: `NO_PUSH=false ./ralph/loop.sh`
   792‚Üí
   793‚Üí### High Circuit Breaker Thresholds (9999999)
   794‚Üí
   795‚Üí**Appears as**: No safety limits
   796‚Üí
   797‚Üí**Why intentional**: 24h+ continuous runs required. Human monitoring via dashboard provides oversight.
   798‚Üí
   799‚Üí**Override**: `CB_NO_PROGRESS_THRESHOLD=10 CB_MAX_COST_USD=100 ./ralph/loop.sh`
   800‚Üí
   801‚Üí### Absurd Line Numbers (99, 999, 9999...)
   802‚Üí
   803‚Üí**Appears as**: Typos or errors
   804‚Üí
   805‚Üí**Why intentional**: Huntley's guardrail pattern ‚Äî high numbers indicate "always applicable" rules. Allows insertion without renumbering.
   806‚Üí
   807‚Üí### Case-Sensitive Completion Detection
   808‚Üí
   809‚Üí**Appears as**: Bug (might miss signals)
   810‚Üí
   811‚Üí**Why intentional**: Completion signals are code-defined, not user-typed. Case-sensitivity prevents false positives.
   812‚Üí
