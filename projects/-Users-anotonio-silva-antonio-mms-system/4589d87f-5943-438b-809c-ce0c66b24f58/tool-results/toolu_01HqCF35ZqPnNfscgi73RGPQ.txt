    58→## Playwright MCP Verification Workflow
    59→
    60→> **MANDATORY** for all UI work (new features AND verification of completed features)
    61→
    62→### Step-by-step E2E verification using Playwright MCP:
    63→
    64→**1. Start dev server** (in separate terminal):
    65→```bash
    66→bun run dev
    67→```
    68→Server runs at http://localhost:5173
    69→
    70→**2. Navigate with Playwright MCP**:
    71→- Use `mcp__playwright__browser_navigate` to `http://localhost:5173`
    72→
    73→**3. Capture page state**:
    74→- Use `mcp__playwright__browser_snapshot` to get accessibility tree
    75→- Review snapshot for expected elements
    76→
    77→**4. Verify interactions**:
    78→- Use `mcp__playwright__browser_click` for clicks
    79→- Use `mcp__playwright__browser_type` for text input
    80→- Use `mcp__playwright__browser_press_key` for keyboard shortcuts
    81→
    82→**5. Validate expected state**:
    83→- Take another snapshot after interactions
    84→- Verify expected changes occurred
    85→
    86→**6. Cleanup**:
    87→- Use `mcp__playwright__browser_close`
    88→
    89→### If Playwright MCP verification fails:
    90→- Task is **NOT** complete
    91→- Document issue in @IMPLEMENTATION_PLAN.md
    92→- Fix before proceeding
    93→
    94→### Key Playwright MCP Tools:
    95→| Tool | Purpose |
    96→|------|---------|
    97→| `mcp__playwright__browser_navigate` | Navigate to URL |
    98→| `mcp__playwright__browser_snapshot` | Capture accessibility tree |
    99→| `mcp__playwright__browser_click` | Click element by ref |
   100→| `mcp__playwright__browser_type` | Type text into element |
   101→| `mcp__playwright__browser_press_key` | Press keyboard key |
   102→| `mcp__playwright__browser_take_screenshot` | Capture visual screenshot |
   103→| `mcp__playwright__browser_close` | Close browser |
   104→

   106→- Completed UI features must be verified with Playwright MCP
   107→- New UI features must pass Playwright MCP before being marked complete

   214→## Codebase Patterns
   215→
   216→### Domain Services
   217→- Location: `src/services/`
   218→- Pattern: Class with injected dependencies
   219→- All services emit OpenTelemetry traces
   220→- Use Drizzle ORM for database access
   221→
   222→### API Routes
   223→- Location: `src/api/routes/`
   224→- Framework: Hono
   225→- All routes have OpenAPI spec
   226→- Use @belt/tools createSchema for validation
   227→
   228→### Workers
   229→- Location: `src/workers/`
   230→- Framework: BullMQ (Redis-backed)
   231→- Central exports: `src/workers/index.ts`
   232→- Use `registerWorker()` to create new workers
   233→- 8 queues: ingestion, tactical_scheduler, strategic_planner, optimizer, manhour_leveling, drift_monitor, notification, report
   234→
   235→**Running Workers:**
   236→```bash
   237→# Start all workers
   238→bun run src/workers/runner.ts
   239→
   240→# Start specific workers only
   241→WORKERS=optimizer,drift_monitor bun run src/workers/runner.ts
   242→
   243→# Workers require Redis to be running
   244→docker-compose up -d redis
   245→```
   246→
   247→**Implemented Processors (All 9 production-ready):**
   248→- `optimizer` - Run assignment optimization (C++ OR-Tools or TS fallback)
   249→- `drift_monitor` - Baseline drift analysis
   250→- `ingestion` - CSV mileage file processing
   251→- `maximo_sync` - Bidirectional sync with Maximo (push/pull/full)
   252→- `tactical_scheduler` - PM scheduling for 12-week horizon
   253→- `strategic_planner` - Long-term PM planning (genetic algorithm)
   254→- `manhour_leveling` - Workforce optimization across depots
   255→- `notification` - Email notification delivery
   256→- `report` - Excel/PDF report generation
   257→
   258→### Optimizer IPC
   259→- Protocol: NDJSON over stdin/stdout
   260→- Input: `{ type: "solve", data: {...} }`
   261→- Output: `{ type: "progress" | "solution" | "error", ... }`

   263→### Validation & State Management (@belt/tools)
   264→
   265→> **CRITICAL**: See README.md for complete @belt/tools API reference
   266→
   267→**Schema Validation** (replaces Zod):
   268→```typescript
   269→import { createSchema, createType, InferType } from '@belt/tools';
   270→
   271→const TrainModel = createSchema({
   272→  id: 'string',
   273→  trainNumber: 'string',
   274→  mileageKm: 'number',
   275→  fitnessStatus: { enum: ['FIT', 'RESTRICTED', 'GROUNDED'] },
   276→});
   277→type Train = InferType<typeof TrainModel>;
   278→const train = TrainModel.parse(data);
   279→```
   280→
   281→**State Management** (replaces React hooks + zustand):
   282→```typescript
   283→import { createSlice, createMachine } from '@belt/tools';
   284→
   285→const fleetSlice = createSlice({
   286→  id: 'fleet',
   287→  initial: { trains: [], selected: null },
   288→  cache: { policy: 'session' },
   289→  reducers: {
   290→    setTrains: (state, trains) => { state.trains = trains; },
   291→  },
   292→});
   293→
   294→const machine = createMachine({
   295→  id: 'mms',
   296→  slices: { fleet: fleetSlice },
   297→  trackMetadata: false, // Critical for 60fps
   298→});
   299→
   300→// In React component (NO useState/useEffect!)
   301→const { state, actions } = machine.useMachine('fleet');
   302→```
