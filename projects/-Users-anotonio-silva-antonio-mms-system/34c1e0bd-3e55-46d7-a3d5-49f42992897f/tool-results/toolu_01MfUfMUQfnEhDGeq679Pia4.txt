Mathematical Formulas\n\n### 4.1 Km Variance (Coefficient of Variation)\n\nPrimary metric for fleet mileage balance. Lower variance = better distribution.\n\n```\nKm Variance % = (Standard Deviation / Mean) * 100\n\nWhere:\n  mean = sum(all train km) / n\n  stddev = sqrt(sum((x_i - mean)^2) / n)\n```\n\n```typescript\nfunction calculateKmVariance(kmValues: number[]): number {\n  if (kmValues.length < 2) return 0;\n\n  const mean = kmValues.reduce((a, b) => a + b, 0) / kmValues.length;\n  if (mean === 0) return 0;\n\n  const squaredDiffs = kmValues.map(km => Math.pow(km - mean, 2));\n  const variance = squaredDiffs.reduce((a, b) => a + b, 0) / kmValues.length;\n  const stdDev = Math.sqrt(variance);\n\n  return (stdDev / mean) * 100;\n}\n```\n\n| Status | Variance | Indicator |\n|--------|----------|-----------|\n| Green (Healthy) | <= 10% | Green badge |\n| Yellow (Warning) | 10% - 15% | Yellow badge |\n| Red (Critical) | > 15% | Red badge |\n\n---\n\n### 4.2 Fleet Availability\n\n```\nFleet Availability % = (Fit Trains / Total Trains) * 100\n\nTargets:\n  Target: >= 97%\n  Critical: < 95%\n```\n\n```typescript\nfunction calculateFleetAvailability(fitTrains: number, totalTrains: number): number {\n  if (totalTrains === 0) return 0;\n  return (fitTrains / totalTrains) * 100;\n}\n```\n\n---\n\n### 4.3 MTBF / MTTR / MDBF\n\n```\nMTBF = Total Operating Hours / Number of Failures (Hours)\nMDBF = Total Operating km / Number of Failures (Kilometers)\nMTTR = Total Downtime Hours / Number of Repairs (Hours)\n       Target: <= 24 hours, Critical: > 48 hours\n```\n\n```typescript\ninterface FailureMetrics {\n  mtbf: number;\n  mdbf: number;\n  mttr: number;\n}\n\nfunction calculateFailureMetrics(\n  operatingHours: number,\n  operatingKm: number,\n  downtimeHours: number,\n  failureCount: number,\n  repairCount: number\n): FailureMetrics {\n  return {\n    mtbf: failureCount > 0 ? operatingHours / failureCount : Infinity,\n    mdbf: failureCount > 0 ? operatingKm / failureCount : Infinity,\n    mttr: repairCount > 0 ? downtimeHours / repairCount : 0,\n  };\n}\n```\n\n---\n\n### 4.4 Coordinate System\n\n#### Constants\n\n| Constant | Value | Description |\n|----------|-------|-------------|\n| `PX_PER_DAY` | 10 pixels | 1 day = 10 pixels on X axis |\n| `PX_PER_TRAIN` | 24 pixels | 1 train row = 24 pixels on Y axis |\n| `minZoom` | 0.1 | Minimum zoom (zoomed out) |\n| `maxZoom` | 5.0 | Maximum zoom (zoomed in) |\n\n#### Coordinate Conversions\n\n```typescript\nconst PX_PER_DAY = 10;\nconst PX_PER_TRAIN = 24;\n\nfunction dayToX(day: number): number {\n  return day * PX_PER_DAY;\n}\n\nfunction trainIndexToY(index: number): number {\n  return -index * PX_PER_TRAIN;\n}\n\nfunction screenToWorld(\n  screenX: number, screenY: number,\n  cameraX: number, cameraY: number,\n  zoom: number,\n  viewportWidth: number, viewportHeight: number\n): { worldX: number; worldY: number } {\n  return {\n    worldX: cameraX + (screenX - viewportWidth / 2) / zoom,\n    worldY: cameraY + (screenY - viewportHeight / 2) / zoom,\n  };\n}\n```\n\n#### LOD Thresholds\n\n| LOD Level | Zoom Range | Time Scale | Rendering |\n|-----------|------------|------------|-----------|\n| MACRO | < 0.5 | 10-20 years | Heatmap (aggregated) |\n| MESO | 0.5 - 2.0 | 2-5 years | Task blocks (instanced) |\n| MICRO | >= 2.0 | 4-8 weeks | Detailed cards (HTML) |\n\n#### Blend Zone\n\n```typescript\nconst BLEND_ZONE_SIZE = 0.15;\n\nfunction calculateOpacity(\n  zoom: number,\n  threshold: number,\n  direction: 'in' | 'out'\n): number {\n  const t = (zoom - threshold) / BLEND_ZONE_SIZE;\n  const clamped = Math.max(0, Math.min(1, t));\n  return direction === 'in' ? clamped : 1 - clamped;\n}\n```\n\n---\n\n### 4.5 Km Penalty Calculation (Solver)\n\nUsed by allocation solver to prioritize under-utilized trains.\n\n```\ndeviation_pct = |current_km - target_km| / max(target_km, 1) * 100\n\nIf current_km < target_km:\n  penalty = -deviation_pct * 10  (BONUS)\nIf current_km >= target_km:\n  penalty = +deviation_pct * 10  (PENALTY)\n```\n\n```typescript\nfunction calculateKmPenalty(\n  currentKm: number,\n  targetKm: number,\n  fleetAvgKm: number\n): number {\n  const effectiveTarget = targetKm || fleetAvgKm;\n  if (effectiveTarget === 0) return 0;\n\n  const deviationPct = Math.abs(currentKm - effectiveTarget) / effectiveTarget * 100;\n\n  if (currentKm < effectiveTarget) {\n    return -Math.round(deviationPct * 10);\n  } else {\n    return Math.round(deviationPct * 10);\n  }\n}\n```\n\n---\n\n### 4.6 Haversine Distance (Dead Running)\n\nGreat circle distance between two geographic points.\n\n```\na = sin^2(delta_lat/2) + cos(lat1) * cos(lat2) * sin^2(delta_lon/2)\nc = 2 * atan2(sqrt(a), sqrt(1-a))\ndistance = R * c\n\nWhere: R = 6371 km (Earth's radius)\n```\n\n```typescript\nfunction haversineDistance(\n  lat1: number, lon1: number,\n  lat2: number, lon2: number\n): number {\n  const R = 6371;\n  const toRad = (deg: number) => deg * Math.PI / 180;\n\n  const lat1Rad = toRad(lat1);\n  const lat2Rad = toRad(lat2);\n  const deltaLat = toRad(lat2 - lat1);\n  const deltaLon = toRad(lon2 - lon1);\n\n  const a = Math.sin(deltaLat / 2) ** 2 +\n            Math.cos(lat1Rad) * Math.cos(lat2Rad) *\n            Math.sin(deltaLon / 2) ** 2;\n\n  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));\n\n  return R * c;\n}\n```\n\n---\n\n## 5.
