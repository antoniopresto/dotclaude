    23→### 6.1.2 Design Principles
    24→
    25→1. **Information Density**: Maximize data visibility without overwhelming
    26→2. **Semantic Zoom**: Data representation adapts to zoom level
    27→3. **Direct Manipulation**: Drag-and-drop with immediate feedback
    28→4. **Progressive Disclosure**: Details on demand
    29→5. **Constraint Visibility**: Show limits, not just violations
    30→6. **Explainability**: Every system decision has visible rationale

   224→### 6.3.5 Hybrid Rendering Architecture
   225→
   226→**CRITICAL:** The canvas uses **hybrid rendering** to optimize for each zoom level:
   227→
   228→| Zoom Level | Renderer | Rationale |
   229→|------------|----------|-----------|
   230→| Strategic (0-4) | **deck.gl (WebGL)** | Heatmap cells, no text needed |
   231→| Tactical-Wide (4-8) | **deck.gl (WebGL)** | Compressed blocks, minimal text |
   232→| Tactical (8-12) | **Konva (HTML5 Canvas)** | Gantt bars with crisp text labels |
   233→| Operational (12-16) | **Konva (HTML5 Canvas)** | Detailed cards with readable text |
   234→| Detail (16-20) | **React DOM** | Rich interactions, form inputs |
   235→
   236→**Why Hybrid?**
   237→
   238→deck.gl uses WebGL for rendering, which has limitations for text-heavy views:
   239→- Font atlases blur during zoom operations
   240→- Hit-testing for drag-and-drop is complex in WebGL
   241→- Text rendering feels less crisp than native canvas/DOM
   242→
   243→By switching to Konva (HTML5 Canvas 2D) at tactical zoom levels, we get:
   244→- Crisp text rendering at all scales
   245→- Native canvas hit-testing for smooth drag-and-drop
   246→- Better performance for hundreds of labeled PM blocks
   247→
   248→```typescript
   249→interface CanvasLayers {
   250→  // === DECK.GL LAYERS (Strategic/Overview) ===
   251→  // Used for zoom levels 0-8 where text is minimal
   252→  
   253→  strategicHeatmapLayer: HeatmapLayer;    // PM density heatmap
   254→  timelineAxisLayer: LineLayer;            // Time axis lines
   255→  quarterBoundaryLayer: LineLayer;         // Quarter separators
   256→  
   257→  // === KONVA LAYERS (Tactical/Operational) ===
   258→  // Used for zoom levels 8-16 where text labels are critical
   259→  
   260→  ganttBlockLayer: Konva.Layer;            // PM blocks with labels
   261→  constraintWallLayer: Konva.Layer;        // Compliance windows
   262→  trainLabelLayer: Konva.Layer;            // Train number labels
   263→  tooltipLayer: Konva.Layer;               // Hover tooltips
   264→  dragPreviewLayer: Konva.Layer;           // Drag-and-drop preview
   265→  
   266→  // === REACT DOM OVERLAY (Detail) ===
   267→  // Used for zoom level 16+ where rich interactions needed
   268→  
   269→  detailCardOverlay: React.ReactNode;      // Expanded PM cards
   270→  editFormOverlay: React.ReactNode;        // Inline editing
   271→}
   272→
   273→// Layer visibility by zoom level
   274→function getActiveRenderer(zoom: number): 'deckgl' | 'konva' | 'react' {
   275→  if (zoom < 8) return 'deckgl';
   276→  if (zoom < 16) return 'konva';
   277→  return 'react';
   278→}
   279→```
   280→
   281→**Transition Handling:**
   282→
   283→When crossing renderer boundaries, animate opacity to avoid jarring switches:
   284→
   285→```typescript
   286→function handleZoomChange(prevZoom: number, newZoom: number): void {
   287→  const prevRenderer = getActiveRenderer(prevZoom);
   288→  const newRenderer = getActiveRenderer(newZoom);
   289→  
   290→  if (prevRenderer !== newRenderer) {
   291→    // Cross-fade over 200ms
   292→    animateOpacity(prevRenderer, 1, 0, 200);
   293→    animateOpacity(newRenderer, 0, 1, 200);
   294→  }
   295→}
