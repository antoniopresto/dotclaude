   210→## Known Test Issues

   212→### Bun mock.module Test Isolation (4 tests)
   213→
   214→**Status**: Known limitation of Bun test runner
   215→**Impact**: 4 tests fail when run in full suite, pass when run individually
   216→
   217→When `export.test.ts` runs before `integration.test.ts` or `scenario.test.ts`, the `mock.module` calls persist globally, affecting subsequent test files. This causes 4 tests to return 500 instead of 404:
   218→
   219→1. `Scenario Routes > POST /scenarios/cleanup > should trigger cleanup (integration)`
   220→2. `Integration Routes > POST /integration/conflicts/:id/resolve > should return 404`
   221→3. `Integration Routes > POST /integration/shadow-records > should return 404`
   222→4. `Integration Routes > POST /integration/validate-edit > should return 404`
   223→
   224→**Root Cause**: Bun's `mock.module` does not reset between test files. The logger/tracing/queue mocks from `export.test.ts` replace the real modules globally.
   225→
   226→**Workaround**: Tests pass when run individually:
   227→```bash
   228→bun test src/api/routes/integration.test.ts  # ✅ All pass
   229→bun test src/api/routes/scenario.test.ts     # ✅ All pass
   230→```
   231→
   232→**Affected Test Count**: 2025 pass, 21 skip, 4 fail (99.8% pass rate)
