## What is Ralph Wiggum?

The Ralph Wiggum technique is an iterative AI development methodology. In its purest form, it's a simple while loop that repeatedly feeds an AI agent a prompt until completion. Named after The Simpsons character, it embodies the philosophy of persistent iteration despite setbacks.(no content)The technique was created by Geoffrey Huntley back in July 2025.(no content)Anthropic added an official plugin to Claude Code in December.

## How It Works

You give Claude Code a task, it works on it, and when it tries to exit, a Stop hook blocks the exit and feeds the same prompt back in. The files it just changed are still there, so each iteration can build on the last one.

## Key Best Practices & Tips

### 1. Always Set Iteration Limits
Always set `—max-iterations`. The `—completion-promise` flag uses exact string matching, which is unreliable. Iteration limits are your real safety net.

### 2. Clear Completion Criteria
The most important thing is a clear completion signal. Without it, Ralph just keeps running forever. Your prompt needs explicit criteria.

### 3. One Task Per Iteration
Each iteration should do one thing because context windows have a quality curve. Early in the window, Claude is sharp. As tokens accumulate, quality degrades. If you try to cram multiple features into one iteration, you're working in the degraded part of the curve.

### 4. Start Small
Don't run overnight loops immediately. Start with single iterations manually, review the output, then run again. Get a feel for how Claude interprets your prompt and whether it's making progress.

### 5. Use a Progress File
One technique from Anthropic's research is the progress file - a simple text file where Claude logs what it did each iteration. When a new iteration starts, Claude reads this file to understand the current state. It's like leaving notes for yourself.

### 6. Pick the Right Tasks
Ralph Wiggum works best for tasks with clear completion criteria and mechanical execution: large refactors, batch operations, test coverage, greenfield builds. If you can describe "done" precisely, Ralph can iterate toward it.
