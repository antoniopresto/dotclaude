     1‚Üí# MMS Run Assignment Optimizer
     2‚Üí
     3‚ÜíC++ OR-Tools CP-SAT solver for train-to-route assignment optimization.
     4‚Üí
     5‚Üí## Overview
     6‚Üí
     7‚ÜíThis optimizer uses Google OR-Tools Constraint Programming SAT solver to find optimal train-to-route assignments while respecting hard constraints and minimizing soft constraint penalties.
     8‚Üí
     9‚Üí## Communication Protocol
    10‚Üí
    11‚ÜíThe optimizer uses NDJSON (newline-delimited JSON) for IPC with the TypeScript wrapper:
    12‚Üí
    13‚Üí### Input (stdin)
    14‚ÜíSingle JSON object with:
    15‚Üí- `request_id`: Correlation ID
    16‚Üí- `trains`: Array of train objects
    17‚Üí- `routes`: Array of route objects
    18‚Üí- `hard_constraints`: Constraints that must not be violated
    19‚Üí- `soft_constraints`: Constraints with penalty weights
    20‚Üí- `config`: Solver configuration
    21‚Üí
    22‚Üí### Output (stdout)
    23‚ÜíNDJSON stream with message types:
    24‚Üí- `HEARTBEAT`: Periodic liveness signal (every 5s)
    25‚Üí- `PROGRESS`: Solver progress updates
    26‚Üí- `ERROR`: Fatal error with recovery suggestion
    27‚Üí- `RESULT`: Final optimization result
    28‚Üí

    70‚Üí## Constraints
    71‚Üí
    72‚Üí### Hard Constraints (NEVER violated)
    73‚Üí- Each train assigned to at most one route
    74‚Üí- Route requirements (exact N trains needed)
    75‚Üí- Fleet compatibility (train fleet must be allowed on route)
    76‚Üí- Fitness status (GROUNDED trains cannot be assigned)
    77‚Üí- Unavailability events (blocked trains)
    78‚Üí- Route compatibility matrix
    79‚Üí
    80‚Üí### Soft Constraints (minimized with penalties)
    81‚Üí- Mileage imbalance (under/over-utilized trains)
    82‚Üí- Preference violations (depot, route preferences)
    83‚Üí
    84‚Üí## Integration
    85‚Üí
    86‚ÜíThe TypeScript wrapper at `src/optimizer/index.ts` handles:
    87‚Üí- Binary spawning and lifecycle management
    88‚Üí- Heartbeat monitoring (15s timeout = 3 missed beats)
    89‚Üí- Progress callback streaming
    90‚Üí- Automatic fallback to TypeScript greedy algorithm
    91‚Üí- Error recovery and logging
    92‚Üí
