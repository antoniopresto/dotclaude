     1→# MMS
     2→
     3→**Metro Fleet Mileage Management System**
     4→
     5→Constraint-based fleet management platform for train deployments and preventive maintenance scheduling over a 12-year government contract period.
     6→
     7→## Architecture
     8→
     9→```
    10→┌─────────────────────────────────────────────────────────────────────────────┐
    11→│                              Frontend (Vite + React 18)                     │
    12→│   deck.gl Canvas │ BlueprintJS Components │ @belt/tools State Machines     │
    13→└────────────────────────────────┬────────────────────────────────────────────┘
    14→                                 │ /api/*
    15→                                 ▼
    16→┌─────────────────────────────────────────────────────────────────────────────┐
    17→│                           API Server (Bun + Hono)                           │
    18→│              REST Endpoints │ OpenAPI Spec │ OpenTelemetry Traces           │
    19→└───────┬────────────────────────────┬───────────────────────────┬────────────┘
    20→        │                            │                           │
    21→        ▼                            ▼                           ▼
    22→┌───────────────┐          ┌─────────────────┐          ┌─────────────────┐
    23→│  PostgreSQL   │          │      Redis      │          │     Workers     │
    24→│  + TimescaleDB│          │  Queues + Cache │          │     (BullMQ)    │
    25→└───────────────┘          └─────────────────┘          └────────┬────────┘
    26→                                                                 │
    27→                                                                 ▼
    28→                                                        ┌─────────────────┐
    29→                                                        │  C++ Optimizer  │
    30→                                                        │   (OR-Tools)    │
    31→                                                        └─────────────────┘
    32→```
    33→
    34→**Stack:**
    35→- **Runtime**: Bun 1.x
    36→- **Database**: PostgreSQL 16 + TimescaleDB (time-series)
    37→- **Queue**: BullMQ (Redis-backed job processing)
    38→- **API**: Hono framework with OpenAPI/Zod validation
    39→- **Frontend**: React 18 + deck.gl (WebGL) + BlueprintJS v6
    40→- **State**: @belt/tools machines (NOT React hooks/zustand)
    41→- **Optimizer**: C++ with OR-Tools CP-SAT solver

   198→## Domain Concepts
   199→
   200→- **PM (Preventive Maintenance)**: C1, C2, C3, C4, Overhaul cycles
   201→- **Run Assignment**: Daily train-to-route allocation
   202→- **Compliance Window**: Valid date range for PM scheduling
   203→- **Bow Wave**: Accumulation of deferred maintenance (risk indicator)
   204→- **TCO**: Total Cost of Ownership (12-year projection)
   205→
   206→**Constraint Hierarchy:**
   207→1. **Hard constraints**: Safety, physical limits - NEVER violate
   208→2. **Soft constraints**: Compliance windows, preferences - violate with penalty
   209→3. **Material constraints**: Check inventory before scheduling PM
   210→4. **Track constraints**: Validate track capability for PM type

   215→
   216→The `@belt/tools` package provides comprehensive utilities for state management, runtime validation, and common operations. All utilities are already available in this project.

   218→### State Machine
   219→
   220→Immer-based reactive state management with React hooks and automatic cache management.
   221→
   222→#### Creating Slices
   223→
   224→```typescript
   225→import { createSlice, createAsyncSlice, createMachine } from '@belt/tools';
   226→
   227→// Synchronous slice for local state
   228→const fleetSlice = createSlice({
   229→  id: 'fleet',
   230→  initial: {
   231→    trains: [],
   232→    pmHistory: [],
   233→    selectedTrain: null as string | null,
   234→  },
   235→  cache: { policy: 'session' }, // or 'public', 'private'
   236→  reducers: {
   237→    importFleet: (state, payload: Train[]) => {
   238→      state.trains = payload; // Immer allows direct mutations
   239→    },
   240→    selectTrain: (state, trainId: string) => {
   241→      state.selectedTrain = trainId;
   242→    },
   243→    clearSelection: (state) => {
   244→      state.selectedTrain = null;
   245→    },
   246→  },
   247→});
   248→
   249→// Async slice for data fetching
   250→const analyticsSlice = createAsyncSlice({
   251→  id: 'analytics',
   252→  cache: {
   253→    policy: 'public',
   254→    ttl: '5m', // Time-to-live using ms() format
   255→    swr: true, // Stale-while-revalidate
   256→  },
   257→  context: {},
   258→  load: async () => {
   259→    const response = await fetch('/api/analytics');
   260→    return response.json();
   261→  },
   262→});
   263→```
   264→
   265→#### Creating Machines
   266→
   267→```tsx
   268→import { createMachine } from '@belt/tools';
   269→
   270→const appMachine = createMachine({
   271→  id: 'mms',
   272→  slices: {
   273→    fleet: fleetSlice,
   274→    analytics: analyticsSlice,
   275→  },
   276→  context: { userId: null }, // Global context
   277→});
   278→
   279→// Use in React components
   280→function FleetView() {
   281→  const { state, actions } = appMachine.useMachine('fleet');
   282→
   283→  return (
   284→    <div>
   285→      <button onClick={() => actions.importFleet(trains)}>
   286→        Import Fleet
   287→      </button>
   288→      <div>{state.trains.length} trains loaded</div>
   289→    </div>
   290→  );
   291→}
   292→```
   293→
   294→**Cache Policies:**
   295→- `public` - Shared cache, TTL-based invalidation
   296→- `private` - User-specific cache, invalidated on user change
   297→- `session` - Session-specific cache, invalidated on app restart

   598→### Type Utilities
   599→
   600→Advanced TypeScript type helpers.
   601→
   602→```typescript
   603→import type {
   604→  Paths,          // All dot-notation paths of object
   605→  PathType,       // Type at specific path
   606→  Compute,        // Flatten intersection types
   607→  Writable,       // Remove readonly modifiers
   608→  Falsy,          // Union of falsy values
   609→  InferType,      // Infer type from schema
   610→  UnknownRecord,  // { [key: string]: unknown }
   611→  AnyRecord,      // { [key: string]: any }
   612→  MaybePromise,   // T | Promise<T>
   613→  MaybeArray,     // T | T[]
   614→  PromiseValue,   // Extract Promise<T> → T
   615→  Simplify,       // Flatten complex types
   616→  Merge,          // Deep merge two types
   617→  PartialFields,  // Make specific fields optional
   618→  NullableToPartial, // Convert nullable to optional
   619→  UnionToIntersection, // Union → Intersection
   620→  CastUnknown,    // Cast if unknown
   621→  SerializableOnly, // Only serializable fields
   622→} from '@belt/tools';
   623→
   624→// Example: Paths
   625→type User = {
   626→  id: string;
   627→  profile: {
   628→    address: {
   629→      street: string;
   630→    };
   631→  };
   632→};
   633→
   634→type UserPaths = Paths<User>;
   635→// 'id' | 'profile' | 'profile.address' | 'profile.address.street'
   636→
   637→type StreetType = PathType<User, 'profile.address.street'>;
   638→// string
   639→
   640→// Example: Compute (flatten intersections)
   641→type A = { a: number };
   642→type B = { b: string };
   643→type Flat = Compute<A & B>; // { a: number; b: string }
   644→
   645→// Example: Writable
   646→type ReadOnly = { readonly id: string; readonly name: string };
   647→type Mutable = Writable<ReadOnly>; // { id: string; name: string }
   648→
   649→// Example: PartialFields
   650→type Train = { id: string; name: string; metadata: object };
   651→type Optional = PartialFields<Train, ['metadata']>;
   652→// { id: string; name: string; metadata?: object }
   653→```
