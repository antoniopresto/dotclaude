    23вҶ’### 6.1.2 Design Principles
    24вҶ’
    25вҶ’1. **Information Density**: Maximize data visibility without overwhelming
    26вҶ’2. **Semantic Zoom**: Data representation adapts to zoom level
    27вҶ’3. **Direct Manipulation**: Drag-and-drop with immediate feedback
    28вҶ’4. **Progressive Disclosure**: Details on demand
    29вҶ’5. **Constraint Visibility**: Show limits, not just violations
    30вҶ’6. **Explainability**: Every system decision has visible rationale

   224вҶ’### 6.3.5 Hybrid Rendering Architecture
   225вҶ’
   226вҶ’**CRITICAL:** The canvas uses **hybrid rendering** to optimize for each zoom level:
   227вҶ’
   228вҶ’| Zoom Level | Renderer | Rationale |
   229вҶ’|------------|----------|-----------|
   230вҶ’| Strategic (0-4) | **deck.gl (WebGL)** | Heatmap cells, no text needed |
   231вҶ’| Tactical-Wide (4-8) | **deck.gl (WebGL)** | Compressed blocks, minimal text |
   232вҶ’| Tactical (8-12) | **Konva (HTML5 Canvas)** | Gantt bars with crisp text labels |
   233вҶ’| Operational (12-16) | **Konva (HTML5 Canvas)** | Detailed cards with readable text |
   234вҶ’| Detail (16-20) | **React DOM** | Rich interactions, form inputs |
   235вҶ’
   236вҶ’**Why Hybrid?**
   237вҶ’
   238вҶ’deck.gl uses WebGL for rendering, which has limitations for text-heavy views:
   239вҶ’- Font atlases blur during zoom operations
   240вҶ’- Hit-testing for drag-and-drop is complex in WebGL
   241вҶ’- Text rendering feels less crisp than native canvas/DOM
   242вҶ’
   243вҶ’By switching to Konva (HTML5 Canvas 2D) at tactical zoom levels, we get:
   244вҶ’- Crisp text rendering at all scales
   245вҶ’- Native canvas hit-testing for smooth drag-and-drop
   246вҶ’- Better performance for hundreds of labeled PM blocks
   247вҶ’
   248вҶ’```typescript
   249вҶ’interface CanvasLayers {
   250вҶ’  // === DECK.GL LAYERS (Strategic/Overview) ===
   251вҶ’  // Used for zoom levels 0-8 where text is minimal
   252вҶ’  
   253вҶ’  strategicHeatmapLayer: HeatmapLayer;    // PM density heatmap
   254вҶ’  timelineAxisLayer: LineLayer;            // Time axis lines
   255вҶ’  quarterBoundaryLayer: LineLayer;         // Quarter separators
   256вҶ’  
   257вҶ’  // === KONVA LAYERS (Tactical/Operational) ===
   258вҶ’  // Used for zoom levels 8-16 where text labels are critical
   259вҶ’  
   260вҶ’  ganttBlockLayer: Konva.Layer;            // PM blocks with labels
   261вҶ’  constraintWallLayer: Konva.Layer;        // Compliance windows
   262вҶ’  trainLabelLayer: Konva.Layer;            // Train number labels
   263вҶ’  tooltipLayer: Konva.Layer;               // Hover tooltips
   264вҶ’  dragPreviewLayer: Konva.Layer;           // Drag-and-drop preview
   265вҶ’  
   266вҶ’  // === REACT DOM OVERLAY (Detail) ===
   267вҶ’  // Used for zoom level 16+ where rich interactions needed
   268вҶ’  
   269вҶ’  detailCardOverlay: React.ReactNode;      // Expanded PM cards
   270вҶ’  editFormOverlay: React.ReactNode;        // Inline editing
   271вҶ’}
   272вҶ’
   273вҶ’// Layer visibility by zoom level
   274вҶ’function getActiveRenderer(zoom: number): 'deckgl' | 'konva' | 'react' {
   275вҶ’  if (zoom < 8) return 'deckgl';
   276вҶ’  if (zoom < 16) return 'konva';
   277вҶ’  return 'react';
   278вҶ’}
   279вҶ’```
   280вҶ’
   281вҶ’**Transition Handling:**
   282вҶ’
   283вҶ’When crossing renderer boundaries, animate opacity to avoid jarring switches:
   284вҶ’
   285вҶ’```typescript
   286вҶ’function handleZoomChange(prevZoom: number, newZoom: number): void {
   287вҶ’  const prevRenderer = getActiveRenderer(prevZoom);
   288вҶ’  const newRenderer = getActiveRenderer(newZoom);
   289вҶ’  
   290вҶ’  if (prevRenderer !== newRenderer) {
   291вҶ’    // Cross-fade over 200ms
   292вҶ’    animateOpacity(prevRenderer, 1, 0, 200);
   293вҶ’    animateOpacity(newRenderer, 0, 1, 200);
   294вҶ’  }
   295вҶ’}
   296вҶ’```
