    53→### BUG-009: Trackpad Zoom Performance - IDENTIFIED
    54→
    55→**Objective:**

    57→
    58→**Research Conducted:**
    59→- Web searches: Three.js frustum culling, R3F viewport culling, RBush spatial indexing, deck.gl massive rendering, Figma WebGL architecture

    62→- Read: `apps/canvas/src/canvas/InfiniteCanvas.tsx` (current LOD implementation)
    63→
    64→**Key Findings:**
    65→
    66→1. **InstancedMesh Problem:** Three.js sends ALL instances to GPU regardless of viewport
    67→   - 60k instances = 60k transforms uploaded every frame
    68→   - No built-in per-instance frustum culling
    69→
    70→2. **RBush Spatial Index:**
    71→   - Bulk load 1M items ~1600ms (one-time)
    72→   - Viewport query for 1000 results ~4ms
    73→   - Perfect for viewport culling queries
    74→
    75→3. **Culling Strategy:**
    76→   - Reorder visible instances to front of buffer
    77→   - Set `mesh.count = visibleCount` (only visible rendered)
    78→   - No new allocations during pan/zoom
    79→
    80→4. **Alternative Libraries:**
    81→   - `@three.ez/instanced-mesh` (InstancedMesh2) - has BVH + per-instance culling
    82→   - `three-mesh-bvh` - accelerated raycasting
    83→   - `flatbush` - 5-8x faster than RBush for static data
    84→

    86→- `docs/VIEWPORT_CULLING_RESEARCH.md` - Comprehensive research consolidating all findings

  1144→### F122: Fix elastic/jump effect when dragging - COMPLETE
  1145→
  1146→**Problem:**
  1147→Canvas felt elastic or jumped when starting a new drag during/after momentum animation.
  1148→
  1149→**Root Cause Analysis:**
  1150→React state updates are batched/async. When user starts a new drag while momentum is running:
  1151→1. `handlePointerDown` captures `cameraPos` from React state
  1152→2. But React state hasn't been updated yet with latest momentum position
  1153→3. New drag starts from stale position → canvas "jumps" back
  1154→
  1155→**Solution Implemented:**
  1156→1. Added `cameraPosRef` to track camera position synchronously via useRef
  1157→2. Created `updateCameraPos()` helper that updates BOTH ref and state
  1158→3. Updated all `setCameraPos()` calls to use `updateCameraPos()`:
  1159→   - `handleCameraInitialized`
  1160→   - `animateZoomTo` (zoom animation loop)
  1161→   - `startMomentum` (momentum animation loop)
  1162→   - `handlePinchZoom`
  1163→   - `handlePointerMove` (pan movement)
  1164→   - `handleDoubleClick` (double-click zoom animation)
  1165→4. Changed `handlePointerDown` to read from `cameraPosRef.current` instead of `cameraPos` state
  1166→

  1168→- `apps/canvas/src/canvas/InfiniteCanvas.tsx` (cameraPosRef, updateCameraPos helper)

  1216→### F121: Cross-platform scroll/input normalization - COMPLETE
  1217→
  1218→**Objective:**
  1219→Normalize scroll sensitivity across macOS/Windows/Linux and trackpad/mouse.
  1220→
  1221→**Implementation:**
  1222→
  1223→1. **deltaMode Normalization:**
  1224→   - LINE mode (deltaMode=1): multiply by LINE_HEIGHT (40px)
  1225→   - PAGE mode (deltaMode=2): multiply by PAGE_HEIGHT (800px)
  1226→   - PIXEL mode (deltaMode=0): use directly
  1227→
  1228→2. **Trackpad vs Mouse Detection:**
  1229→   - Track event frequency with timestamps
  1230→   - Events < 20ms apart = trackpad behavior
  1231→   - After 3+ rapid events, mark as trackpad
  1232→   - Large deltas (>50px) reset to mouse mode
  1233→
  1234→3. **Adaptive Sensitivity:**
  1235→   - Mouse: MOUSE_ZOOM_SENSITIVITY = 0.003
  1236→   - Trackpad: TRACKPAD_ZOOM_SENSITIVITY = 0.001 (lower for many small events)
  1237→
  1238→4. **Zoom Delta Clamping:**
  1239→   - MAX_ZOOM_DELTA_PER_EVENT = 0.5
  1240→   - Prevents jump zoom from large scroll events
  1241→
  1242→**Constants Added:**
  1243→```typescript
  1244→LINE_HEIGHT = 40                    // Pixels per line
  1245→PAGE_HEIGHT = 800                   // Pixels per page
  1246→MOUSE_ZOOM_SENSITIVITY = 0.003      // Base zoom sensitivity
  1247→TRACKPAD_ZOOM_SENSITIVITY = 0.001   // Lower for trackpad
